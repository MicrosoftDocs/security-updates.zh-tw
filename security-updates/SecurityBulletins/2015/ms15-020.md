Microsoft 資訊安全公告 MS15-020 - 重大
======================================

Microsoft Windows 中的資訊安全風險可能會允許遠端執行程式碼 (3041836)
--------------------------------------------------------------------

發行日期：2015 年 3 月 10 日 | 更新日期：2015 年 3 月 10 日

**版本：** 1.1

提要
----

此安全性更新可解決 Microsoft Windows 中的資訊安全風險。如果攻擊者成功引誘使用者瀏覽蓄意製作的網站、開啟蓄意製作的檔案，或在包含蓄意製作的 DLL 檔案的工作目錄中開啟檔案，此資訊安全風險可能會允許遠端執行程式碼。

對於所有受支援版本的 Microsoft Windows，此資訊安全更新的等級為「重大」。如需更多資訊，請參閱**＜受影響的軟體＞**一節。

此安全性更新會修正 Microsoft 文字服務處理記憶體中物件的方式和 Microsoft Windows 處理 DLL 檔案的載入方式，藉以解決資訊安全風險。如需有關資訊安全風險的詳細資訊，請參閱**＜資訊安全風險資訊＞**一節。

如需有關此更新的詳細資訊，請參閱 [Microsoft 知識庫文章 3041836](https://support.microsoft.com/kb/3041836/zh-tw)。

受影響的軟體
------------

下列軟體版號或版本會受到影響。未列出的版號或版本不是超出支援週期就是不受影響。若要瞭解您的軟體版本的支援週期，請參閱 [Microsoft 產品技術支援週期網站](http://go.microsoft.com/fwlink/?linkid=21742)。

資訊安全風險資訊
----------------

WTS 遠端執行程式碼資訊安全風險 - CVE-2015-0081
----------------------------------------------

當 Windows 文字服務不當處理記憶體中物件時，會存在遠端執行程式碼弱點。一旦攻擊得逞，攻擊者可在登入使用者的內容中執行任意程式碼。

若要利用此資訊安全風險，攻擊者必須引誘使用者瀏覽蓄意製作的網站或開啟蓄意製作的檔案。此更新可修正 Microsoft 文字服務處理記憶體中物件的方式，進而解決此資訊安全風險。

Microsoft 是經由協同合作的來源接獲有關此資訊安全風險的訊息。本資訊安全公告初次發行時，Microsoft 尚未接到任何有關本資訊安全風險已公開用來攻擊客戶的消息。

### 緩和因素

Microsoft 尚未找到此項資訊安全風險的任何[緩和因素](https://technet.microsoft.com/zh-tw/library/security/dn848375.aspx)。

### 因應措施

Microsoft 尚未找到此項資訊安全風險的任何[因應措施](https://technet.microsoft.com/zh-tw/library/security/dn848375.aspx)。

 

DLL 設置遠端程式碼執行資訊安全風險 - CVE-2015-0096
--------------------------------------------------

當 Microsoft Windows 不當處理 DLL 檔案載入時，會存在遠端執行程式碼弱點。成功利用此資訊安全風險的攻擊者可以取得受影響系統的完整控制權。攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。設定為具有較少使用者權限的使用者帳戶所受到的影響，可能會比利用系統管理使用者權限進行操作的使用者帳戶小。

形成這個資訊安全風險在於 Windows 未正確剖析捷徑，導致蓄意製作的捷徑圖示顯示時，便可能會允許執行惡意程式碼。使用者必須使用 Windows 檔案總管瀏覽惡意網站、遠端網路共用或本機工作目錄 (請注意，其他瀏覽工作目錄的方法，例如透過 cmd.exe 或 powershell.exe，並不會觸發攻擊)，攻擊者才有機會利用此資訊安全風險。此外，此資訊安全風險可以透過 USB 卸除式磁碟機攻擊，尤其是在啟用「自動播放」的系統上。

使用者瀏覽至惡意目錄後，其中包含的蓄意製作 DLL 將會被載入記憶體中，而使攻擊者在已登入使用者的資訊安全內容中取得受影響系統的控制權。更新會修正 Microsoft Windows 處理載入 DLL 檔案的方式，從而消除這項資訊安全風險。

Microsoft 是經由協同合作的來源接獲有關此資訊安全風險的訊息。本資訊安全公告初次發行時，Microsoft 尚未接到任何有關本資訊安全風險已公開用來攻擊客戶的消息。

### 緩和因素

Microsoft 尚未找到此項資訊安全風險的任何[緩和因素](https://technet.microsoft.com/zh-tw/library/security/dn848375.aspx)。

### 因應措施

下列[因應措施](https://technet.microsoft.com/zh-tw/library/security/dn848375.aspx)可能對您的狀況有所助益：

-   **停用捷徑圖示的顯示**  
    **注意：** 請參閱 [Microsoft 知識庫文件編號 2286198](http://support.microsoft.com/kb/2286198/zh-tw)，瞭解如何使用自動化的 **Microsoft Fix It** 解決方案來啟用或停用本因應措施。這個 Fix it 解決方案需要在完成時重新啟動電腦，才會生效。這個 Fix it 解決方案會部署因應措施，因此對使用者具有相同的影響。我們建議系統管理員先詳閱 KB 文件，再部署這個 Fix it 解決方案。
    **注意：**不當使用 \[登錄編輯程式\] 可能會造成嚴重的問題，甚至可能需要重新安裝您的作業系統。Microsoft 不保證可以解決您不當使用「登錄編輯程式」所導致的問題。請自行承擔使用「登錄編輯程式」的一切風險。如需有關如何編輯登錄的資訊，請檢視「登錄編輯程式」(Regedit.exe) 中的＜變更機碼及數值＞說明主題，或是 Regedt32.exe 中的＜新增及刪除登錄中的資訊＞與＜編輯登錄資料＞說明主題。

    2.  找出並選取下列登錄機碼： 

        ```
            HKEY_CLASSES_ROOT\lnkfile\shellex\IconHandler
        ```

    3.  按一下 **\[檔案\]** 功能表，然後按一下 **\[匯出\]**。
    4.  在 **\[匯出登錄檔案\]** 對話方塊中，輸入 **LNK\_Icon\_Backup.reg**，然後按一下 **\[儲存\]**。
        **注意：**依預設，這個動作會在 \[我的文件\] 資料夾中建立這個登錄機碼的備份。
    5.  在 \[登錄編輯程式\] 的右窗格中選取值 **(預設)**。按下 **ENTER** 編輯機碼的值。刪除數值，使其為空白，然後按下 **ENTER**。
    6.  找出並選取下列登錄機碼： 

        ```
            HKEY_CLASSES_ROOT\piffile\shellex\IconHandler
        ```

    8.  在 **\[匯出登錄檔案\]** 對話方塊中，輸入 **PIF\_Icon\_Backup.reg**，然後按一下 **\[儲存\]**。
        **注意：**依預設，這個動作會在 \[我的文件\] 資料夾中建立這個登錄機碼的備份。
    9.  在 \[登錄編輯程式\] 的右窗格中選取值 **(預設)**。按下 **ENTER** 編輯機碼的值。刪除數值，使其為空白，然後按下 **ENTER**。
    10. 將所有使用者登出再登入，或重新啟動電腦。

    **因應措施的影響。**停止顯示捷徑的圖示，可防止此問題在受影響的系統上遭到利用。實作此因應措施後，系統可能會將大多數的圖示顯示為「白色」的預設物件圖示，而造成使用上的影響。我們建議系統管理員仔細測試此因應措施，再進行部署。復原此因應措施後，所有圖示將會再次出現。
    **如何復原因應措施。**

    使用互動方法：

    1.  按一下 **\[開始\]**，按一下 **\[執行\]**，在 **\[開啟\]** 方塊中輸入 **Regedit**，然後按一下 **\[確定\]**。
    2.  按一下 **\[檔案\]** 功能表，然後按一下 **\[匯入\]**。
    3.  在 **\[匯入登錄檔案\]** 對話方塊中，選取 **\[LNK\_Icon\_Backup.reg\]**，然後按一下 **\[開啟\]**。
    4.  按一下 **\[檔案\]** 功能表，然後按一下 **\[匯入\]**。
    5.  在 **\[匯入登錄檔案\]** 對話方塊中，選取 **\[PIF\_Icon\_Backup.reg\]**，然後按一下 **\[開啟\]**。
    6.  結束登錄編輯程式，然後將電腦重新開機。

-   **停用 WebClient 服務**  
    停用 WebClient 服務可封鎖最可能透過 Web Distributed Authoring and Versioning (WebDAV) 用戶端服務進行的遠端攻擊模式，藉以協助保護受影響的系統，使不會遭到利用此資訊安全風險的攻擊。採取此因應措施後，成功地利用此資訊安全風險的遠端攻擊者仍可能導致 Microsoft Office Outlook 執行目標使用者的電腦或區域網路 (LAN) 上的程式，但系統會先提示使用者確認，才會從網際網路開啟任意的程式。
    若要停用 WebClient 服務，請執行這些步驟：

    1.  按一下 **\[開始\]**，按一下 **\[執行\]**，輸入 **Services.msc**，然後按一下 **\[確定\]**。
    2.  在 **\[WebClient\]** 服務上按一下滑鼠右鍵，然後選取 **\[內容\]**。
    3.  將 \[啟動類型\] 變更為 **\[停用\]**。若正在執行服務，請按一下 **\[停止\]**。
    4.  按一下 **\[確定\]** 並結束管理應用程式。

    **因應措施的影響。**停用 WebClient 服務時，不會傳送 Web Distributed Authoring and Versioning (WebDAV) 要求。此外，任何明確依賴網路用戶端服務的服務將不能啟動，系統記錄檔也會記錄錯誤訊息。例如，用戶端電腦將無法存取 WebDAV 共用。
    **如何復原因應措施：**

    若要重新啟用 WebClient 服務，請依照下列步驟進行：

    1.  按一下 **\[開始\]**，按一下 **\[執行\]**，輸入 **Services.msc**，然後按一下 **\[確定\]**。
    2.  在 **\[WebClient\]** 服務上按一下滑鼠右鍵，然後選取 **\[內容\]**。
    3.  將 \[啟動類型\] 變更為 **\[自動\]**。服務如未執行，請按一下 **\[啟動\]**。
    4.  按一下 **\[確定\]** 並結束管理應用程式。
