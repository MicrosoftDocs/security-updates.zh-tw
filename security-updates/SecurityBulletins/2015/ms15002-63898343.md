Microsoft 資訊安全佈告欄 MS15-002 - 重大
========================================

Windows Telnet 服務中的資訊安全風險可能會允許遠端執行程式碼 (3020393)
---------------------------------------------------------------------

發行日期：2015 年 1 月 13 日

**版本：** 1.0

提要
----

<span id="sectionToggle0"></span>
此資訊安全更新可解決 Microsoft Windows 中一項未公開報告的資訊安全風險。如果攻擊者傳送蓄意製作的封包到受影響的 Windows 伺服器，這項資訊安全風險可允許遠端執行程式碼。依照預設，Telnet 並未安裝於任何受影響的作業系統版本上。唯有手動安裝此項服務的客戶，才有可能受影響。

對於所有受支援版本的 Microsoft Windows，此資訊安全更新的等級為「重大」。如需更多資訊，請參閱**＜受影響的軟體＞**一節。

此安全性更新藉由修正 Telnet 驗證使用者輸入的方式解決此資訊安全風險。如需有關此資訊安全風險的詳細資訊，請參閱**＜資訊安全風險資訊＞**一節。

<span id="KBArticle"></span>
如需有關此更新的詳細資訊，請參閱 [Microsoft 知識庫文件編號 3020393](https://support.microsoft.com/kb/3020393/zh-tw)。

資訊安全風險資訊
----------------

<span id="sectionToggle3"></span>
Windows Telnet 服務緩衝區滿溢的資訊安全風險 - CVE-2015-0014
-----------------------------------------------------------

Windows [Telnet](https://technet.microsoft.com/zh-tw/library/security/dn848375.aspx) 服務中存在緩衝區滿溢的資訊安全風險，該風險可能會允許遠端程式碼執行。Telnet 服務未正確驗證使用者輸入時，會造成此資訊安全風險。攻擊者可以透過傳送蓄意製作的 Telnet 封包至 Windows 伺服器，如果成功，便可在伺服器上執行任意程式碼。此更新藉由修正 Telnet 驗證使用者輸入的方式解決此資訊安全風險。

Microsoft 是經由協同合作的來源接獲有關此資訊安全風險的訊息。本資訊安全佈告欄初次發行時，Microsoft 尚未接到任何有關本資訊安全風險已公開用來攻擊客戶的消息。

### 緩和因素

下列[緩和因素](https://technet.microsoft.com/zh-tw/library/security/dn848375.aspx)可能對您的狀況有所助益：

-   根據預設，Telnet 已安裝，但尚未在 Windows Server 2003 上啟用。唯有手動啟用此項服務的客戶，才有可能受此問題影響。
-   根據預設，Telnet 未安裝於受影響的 Windows Vista 及更新版本。唯有手動安裝並啟用此項服務的客戶，才有可能受此問題的影響。

### 因應措施

Microsoft 尚未找到此項資訊安全風險的任何[因應措施](https://technet.microsoft.com/zh-tw/library/security/dn848375.aspx)。
