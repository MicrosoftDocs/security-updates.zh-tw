Microsoft 資訊安全公告 MS15-041 - 重要
======================================

.NET Framework 中的資訊安全風險可能會導致資訊洩漏 (3048010)
-----------------------------------------------------------

發行日期：2015 年 4 月 14 日 | 更新日期：2015 年 5 月 12 日

**版本：** 2.0

提要
----

<span id="sectionToggle0"></span>
這個安全性更新可解決 Microsoft .NET Framework 中一項資訊安全風險。如果攻擊者將蓄意製作的 Web 要求傳送至已自訂錯誤訊息的受影響伺服器，則此資訊安全風險可能會導致資訊洩漏。成功利用這個資訊安全風險的攻擊者可檢視組態檔的部分，從而得知敏感資訊。

對於受影響版本的 Microsoft Windows 上的 Microsoft .NET Framework 1.1 Service Pack 1、Microsoft .NET Framework 2.0 Service Pack 2、Microsoft .NET Framework 3.5、Microsoft .NET Framework 3.5.1、Microsoft .NET Framework 4、Microsoft .NET Framework 4.5、Microsoft .NET Framework 4.5.1 和 Microsoft .NET Framework 4.5.2，此資訊安全更新的等級為「重要」。如需更多資訊，請參閱**＜受影響的軟體＞**一節。

此安全性更新可從已洩露資訊的錯誤訊息中移除檔案內容詳細資料，進而解決這項資訊安全風險。如需更多有關此資訊安全風險的資訊，請參閱特定資訊安全風險的**＜常見問題集 (FAQ)＞**小節。

<span id="KBArticle"></span>
如需有關此更新的詳細資訊，請參閱 [Microsoft 知識庫文章 3048010](https://support.microsoft.com/zh-tw/kb/3048010)。

更新常見問題集
--------------

<span id="sectionToggle2"></span>
**我該如何分辨已安裝的 Microsoft .NET Framework 版本？**  
您可以在一個系統上安裝和執行多個版本的 .NET Framework，且能以任何順序安裝各版本。如需更多資訊，請參閱 [Microsoft 知識庫文件編號 318785](https://support.microsoft.com/zh-tw/kb/318785)。

**.NET Framework 4 與 .NET Framework 4 Client Profile 之間有何差異？**  
可在兩個設定檔中使用 .NET Framework 第 4 版可轉散發套件：.NET Framework 4 和 .NET Framework 4 Client Profile。.NET Framework 4 Client Profile 是 .NET Framework 4 設定檔的子集，其已針對用戶端應用程式進行最佳化。它會為多數用戶端應用程式提供功能，包括 Windows Presentation Foundation (WPF)、Windows Forms、Windows Communication Foundation (WCF) 和 ClickOnce 功能。這可為以 .NET Framework 4 Client Profile 為目標的應用程式進行更快速的部署及使用更小的安裝套件。如需詳細資訊，請參閱 MSDN 文章：[.NET Framework Client Profile](https://msdn.microsoft.com/zh-tw/library/cc656912) (英文)。

**部分受影響軟體有多個更新套件。我需要安裝「受影響的軟體」表中列出的更新的軟體嗎?**  
是的。客戶應安裝針對其系統上之軟體所提供的所有更新。

**我是否必須按照特定順序來安裝這些資訊安全更新？**  
否。特定的作業系統如有多項更新，套用時無需按照特定順序。

資訊安全風險資訊
----------------

<span id="sectionToggle4"></span>
ASP.NET 資訊洩漏資訊安全風險 -CVE-2015-1648
-------------------------------------------

ASP.NET 中存在資訊洩露資訊安全風險，原因是 ASP.NET 不當處理已停用自訂錯誤訊息之系統的特定要求。成功利用這個資訊安全風險的攻擊者可檢視組態檔的部分，從而得知敏感資訊。

若要利用此資訊安全風險，攻擊者可將蓄意製作的 Web 要求傳送給包含錯誤訊息的受影響伺服器，該訊息則可能洩露發生例外狀況的原始程式碼行的專屬資訊。最後，這可能透露不想供存取的資訊。此安全性更新可從已洩露資訊的錯誤訊息中移除檔案內容詳細資料，進而解決這項資訊安全風險。

依預設，ASP.NET 應用程式不受此資訊安全風險影響，因為它們是設定不會向遠端使用者顯示詳細錯誤訊息。開發人員有時會開啟詳細的錯誤訊息來收集資訊，接著卻無法關閉，應用程式隨後即曝露在此資訊安全風險中。

        
```
    <customErrors mode="remoteOnly" />
```    

在生產環境中，最佳作法是將此項目變更如下：

```   
    <customErrors mode="On" defaultRedirect="ErrorPage.htm" /> 
```    
一般而言，生產環境開發人員應該永遠不會使用下列方法：

```
    <customErrors mode="off" />
```    

請注意，可根據錯誤碼自訂錯誤訊息。如需詳細資訊，請參閱 [customErrors 元素 (ASP.NET 設定結構描述)](https://msdn.microsoft.com/zh-tw/library/h0hfz6fc(v=vs.85).aspx) (英文)。

為避免意外停用自訂錯誤以加強保護，可在零售模式中開啟機器管理。請參閱本公告的[因應措施](#configretail)以取得更多資訊。

Microsoft 是經由協同合作的來源接獲有關此資訊安全風險的訊息。本資訊安全公告發行時，Microsoft 尚未接到任何有關本資訊安全風險已公開用來攻擊客戶的消息。

### 緩和因素

下列[緩和因素](https://technet.microsoft.com/zh-tw/library/security/dn848375.aspx)可能對您的狀況有所助益：

-   **僅出現詳細錯誤訊息的 IIS 伺服器才會受到影響；生產伺服器不太可能會受到影響。**

### 因應措施

<span id="configretail"></span>
下列[因應措施](https://technet.microsoft.com/zh-tw/library/security/dn848375.aspx)可能對您的狀況有所助益：

-   **在零售模式中於所有網頁伺服器上設定 .NET**

    在所有網頁伺服器的 machine.config 檔案中，將「&lt;deployment retail="true" /&gt;」設定新增至「configuration」區段的「system.web」區段。

    在 32 位元系統上，可在 %windir%\\Microsoft.NET\\Framework\\\[version\]\\config\\machine.config 中找到 machine.config。

    在 64 位元系統上，可在 %windir%\\Microsoft.NET\\Framework64\\\[version\]\\config\\machine.config 中找到 machine.config。

    如需關於此設定的詳細資訊，請參閱 [deployment 元素 (ASP.NET 設定結構描述)](https://msdn.microsoft.com/zh-tw/library/vstudio/ms228298%28v=vs.100%29.aspx?f=255&mspperror=-2147217396) (英文)。

    **因應措施的影響。**會抑制每部伺服器上所有網站的所有 ASP.NET 詳細錯誤訊息。

    **如何復原因應措施。**

    若要復原因應措施，請透過此程序移除已加入的設定。

     

-   **啟用所有網站的自訂錯誤**

    在所有網站的 web.config 檔案，確定不要將「customErrors」設定 (位於「configuration」區段的「system.web」區段) 設為「off」。安全值為「on」、「remoteonly」，或完全不要設定。

    如需 customErrors 設定的詳細資訊，請參閱 [customErrors 元素 (ASP.NET 設定結構描述)](https://msdn.microsoft.com/zh-tw/library/h0hfz6fc(v=vs.85).aspx) (英文)。

    **因應措施的影響。**會抑制所有網站的所有 ASP.NET 詳細錯誤訊息。

    **如何復原因應措施。**

    若要復原因應措施，請透過此程序反轉已建立的設定。
