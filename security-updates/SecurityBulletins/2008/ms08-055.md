Microsoft Security Bulletin MS08-055 - 重大
===========================================

Microsoft Office 中的弱點可能會允許遠端執行程式碼 (955047)
----------------------------------------------------------

發行: 2008年9月10日

**版本:** 1.0

### 一般資訊

#### 提要

這個安全性更新可解決 Microsoft Office 中一項未公開報告的弱點。 如果使用者按一下蓄意製作的 OneNote URL，此弱點便可能允許遠端執行程式碼。 成功利用此弱點的攻擊者可以取得受影響系統的完整控制權。 攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。 系統上帳戶使用者權限較低的使用者，其受影響的程度比擁有系統管理權限的使用者要小。

對於所有受支援版本的 Microsoft Office OneNote 2007 而言，此安全性更新的等級為「重大」；對於所有受支援版本的 Microsoft Office XP、Microsoft Office 2003 和 2007 Microsoft Office System 則為「重要」。 如需更多資訊，請參閱本節中的＜受影響及不受影響的軟體＞小節。

此安全性更新是藉由修改 Microsoft Office 驗證統一資源定位器的方式，來解決這個弱點。 如需更多有關弱點的資訊，請參閱下節＜弱點資訊＞下的特定弱點項目的＜常見問題集 (FAQ)＞小節。

**建議。** Microsoft 建議客戶立即套用此更新。

**已知問題。** 無

下列軟體已經過測試判斷哪些版號或版本會受到影響。 其他版本超過它們的支援週期或不受影響。 若要瞭解您的軟體版本的支援週期，請造訪 [Microsoft 產品技術支援週期網站](http://go.microsoft.com/fwlink/?linkid=21742)。

與本安全性更新相關的常見問題集 (FAQ)
------------------------------------

 
**何處可找到檔案資訊詳細資料？**      
檔案資訊詳細資料請參閱 [Microsoft 知識庫文件編號 955047](http://support.microsoft.com/kb/955047)。

**MS08-052 也有描述 Microsoft Office XP Service Pack 3 的弱點嗎？ MS08-052 與此公告 (MS08-055) 有何關聯？**  
這個 Microsoft Office XP Service Pack 3 (KB953405) 安全性更新是 Microsoft Office XP 累計服務模型的一部分，它也可以解決 MS08-052 所描述的弱點。已經安裝 Microsoft Office XP Service Pack 3 的使用者，也必須安裝此安全性更新，但只要安裝一次即可。

**我已經安裝了 Visual Studio 2008、Expression Web 或 Expression Web 2，為什麼還會收到這個更新？**  
Visual Studio 2008、Expression Web 和 Expression Web 2 不受此弱點的影響，您無需安裝此更新。 Visual Studio 2008、Expression Web 和 Expression Web 2 雖然包含受影響的共用 Office 元件，卻沒有存取受影響的程式碼。 不過，由於受影響的程式碼確實存在，所以還是會提供此更新。

**為什麼 Microsoft Office 被列為此更新的影響軟體？**  
Microsoft Office XP、Microsoft Office 2003 和 2007 Microsoft Office System 包含一組由共用 Office 功能使用的共用元件。 此安全性更新是藉由更新共用 Office 元件來解決弱點。

**為什麼此安全性更新對於 OneNote 2007 和 OneNote 2007 Service Pack 1 的等級是「重大」，但是對於其他受影響的軟體來說卻是「重要」呢？**  
雖然此安全性更新是藉由更新共用 Office 功能所用的檔案來解決弱點，但是要解決這個受到攻擊的弱點，必須安裝 OneNote 2007，而且使用者仍必須按一下蓄意製作的 OneNote URL。

統一資源定位器驗證錯誤弱點 - CVE-2008-3007
------------------------------------------

 
遠端執行程式碼弱點是存在於 Microsoft Office 使用 OneNote 通訊協定處理常式 (onenote://)，來處理蓄意製作的 URL 的方式。 如果使用者按一下蓄意製作的 OneNote URL，此弱點便可能允許遠端執行程式碼。 成功利用此弱點的攻擊者可以取得受影響系統的完整控制權。 攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。

若要以一般性弱點清單中的標準項目來檢視此弱點，請參閱 [CVE-2008-3007](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-3007)。

#### 統一資源定位器驗證錯誤弱點的緩和因素 - CVE-2008-3007

緩和因素是指存在於預設狀態中的設定、共用設定或一般最佳作法，可能會減少弱點影響的嚴重性。 下列緩和因素可能對您的狀況有所助益：

-   在網頁攻擊案例中，網站可能含有為了攻擊這個弱點而蓄意製作的連結 (onenote://)。 攻擊者必須說服使用者造訪網站，並開啟蓄意製作的 OneNote URL，一般的做法是設法讓使用者按一下電子郵件或即時訊息中通往攻擊者網站的連結，然後說服他們按一下蓄意製作的連結。
-   成功利用此弱點的攻擊者可以取得與本機使用者相同的使用者權限。 系統上帳戶使用者權限較低的使用者，其受影響的程度比擁有系統管理權限的使用者要小。
-   此弱點不會透過預覽電子郵件而自動受到攻擊。 使用者必須按一下在電子郵件中傳送的蓄意製作之連結，攻擊才能進行。

#### 統一資源定位器驗證錯誤弱點的因應措施 - CVE-2008-3007

因應措施指的是無法徹底修正弱點，但有助於在套用更新之前封鎖已知攻擊模式的設定變更。 Microsoft 測試了下列因應措施和狀態，討論因應措施是否會降低功能：

-   **停用 OneNote 通訊協定處理常式**

    **注意：** 修復 Office 或安裝 Office 安全性更新，可能會復原這個因應措施。

    **警告：** 如果使用「登錄編輯程式」的方式錯誤，可能造成嚴重問題，以致於您必須重新安裝作業系統。 Microsoft 無法保證您可以解決因為不正確使用 \[登錄編輯程式\] 所造成的問題。 請自行承擔使用「登錄編輯程式」的風險。 如需有關如何編輯登錄的資訊，請檢視「登錄編輯程式」(Regedit.exe) 中的＜變更機碼及數值＞說明主題，或是 Regedt32.exe 中的＜新增及刪除登錄中的資訊＞與＜編輯登錄資料＞說明主題。

    **注意：** 我們建議您在編輯登錄之前先將之備份。

    若要停用通訊協定處理常式，請執行下列步驟：

    **互動式方法**

    1.  依序按一下 \[開始\] 和 \[執行\]，再輸入 "regedit" (不必輸入引號)，然後再按一下 \[確定\]。
    2.  展開 **HKEY\_CLASSES\_ROOT**，按一下 \[OneNote\]，再按一下 \[登錄\] 功能表，然後選取 \[匯出\]。
    3.  在 \[匯出登錄檔案\] 對話方塊中輸入 “OneNote registry backup.reg”，然後按 \[儲存\]。 這個動作預設會在 \[我的文件\] 資料夾中建立此登錄機碼的備份。
    4.  展開 \[OneNote\] 鍵，在右窗格中按一下 URL 通訊協定登錄值，然後再按 \[刪除\] 鍵。 當系統提示您透過 \[確認刪除值\] 對話方塊來刪除登錄值時，請選取 \[是\]。

    **管理的部署指令碼**

    1.  首先，您可以使用下列命令，從管理的部署指令碼製作登錄機碼的備份：

    `Regedit.exe /e OneNote_registry_backup.reg HKEY_CLASSES_ROOT\OneNote`

    1.  接著將下述內容儲存在副檔名為 .REG 的檔案中 (亦即 Delete\_OneNote\_URL\_Protocol.reg)：

    `Windows Registry Editor Version 5.00[HKEY_CLASSES_ROOT\OneNote]"URL Protocol"=-`

    1.  使用下述命令，在目標電腦上執行步驟 2 所建立的登錄指令碼：

    `Regedit.exe /s Delete_OneNote_URL_Protocol.reg`

    **因應措施的影響**

    這個因應措施會停用 OneNote 通訊協定處理常式。

    **如何復原因應措施：**

    還原此登錄機碼，方法是使用 Regedit 來還原儲存在 .REG 檔中的設定。

-   **將 OneNote 通訊協定處理常式重新導向至 About 通訊協定處理常式**

    **注意：** 修復 Office 或安裝 Office 安全性更新，可能會復原這個因應措施。

    **警告：** 如果使用「登錄編輯程式」的方式錯誤，可能造成嚴重問題，以致於您必須重新安裝作業系統。 Microsoft 無法保證您可以解決因為不正確使用 \[登錄編輯程式\] 所造成的問題。 請自行承擔使用「登錄編輯程式」的風險。 如需有關如何編輯登錄的資訊，請檢視「登錄編輯程式」(Regedit.exe) 中的＜變更機碼及數值＞說明主題，或是 Regedt32.exe 中的＜新增及刪除登錄中的資訊＞與＜編輯登錄資料＞說明主題。

    **注意：** 我們建議您在編輯登錄之前先將之備份。

    **互動式方法**

    1.  依序按一下 \[開始\] 和 \[執行\]，再輸入 "regedit" (不必輸入引號)，然後再按一下 \[確定\]。
    2.  展開 **HKEY\_CLASSES\_ROOT\\PROTOCOLS**，然後按一下 \[處理常式\]。
    3.  在 \[編輯\] 功能表按一下 \[新增\]，然後再按一下 \[機碼\]。
    4.  在新的機碼對話方塊中，輸入 "onenote" (不必輸入引號)，然後按下 Enter。
    5.  展開新的 **HKEY\_CLASSES\_ROOT\\PROTOCOLS\\Handler\\onenote** 登錄機碼，然後在 \[編輯\] 功能表按一下 \[新增\]，再按一下 \[StringValue\]。
    6.  在新的字串值對話方塊中，輸入 "CLSID" (不必輸入引號)。
    7.  在登錄編輯器的右窗格中，按兩下新的 "CLSID" 登錄值，然後將下述內容輸入或貼到 \[值資料\] 對話方塊中： {3050F406-98B5-11CF-BB82-00AA00BDCE0B}，接著再按 \[確定\] 按鈕。

    **管理的部署指令碼**

    1.  將下述內容儲存在副檔名為 .REG 的檔案中 (亦即 OneNote\_Handler.reg)：

    `Windows Registry Editor Version 5.00[HKEY_CLASSES_ROOT\PROTOCOLS\Handler\onenote]"CLSID"="{3050F406-98B5-11CF-BB82-00AA00BDCE0B}"`

    1.  使用下述命令，在目標電腦上執行步驟 1 所建立的登錄指令碼：

#### 統一資源定位器驗證錯誤弱點的常見問題集 - CVE-2008-3007

**這個弱點的範圍為何？**      
這是遠端執行程式碼的弱點。 成功利用此弱點的攻擊者可以取得受影響系統的完整控制權。 攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。

**造成這個弱點的原因為何？**      
此弱點是由驗證錯誤造成，而驗證錯誤則是在傳遞蓄意製作的統一資源定位器來開啟蓄意製作的 OneNote 檔時發生。

**攻擊者可能會利用這項弱點採取什麼行動？**      
成功利用此弱點的攻擊者可以取得受影響系統的完整控制權。 如果使用者以系統管理的使用者權限登入，則攻擊者即可取得受影響系統的完整控制權。 攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。 系統上帳戶使用者權限較低的使用者，其受影響的程度比擁有系統管理權限的使用者要小。

**攻擊者如何利用這項弱點？**      
在網頁式攻擊的案例中，攻擊者可以說服使用者按一下蓄意製作的統一資源定位器，而此統一資源定位器是參照一個導致遠端執行程式碼的蓄意製作的 OneNote 檔。

**什麼元件會受到這個弱點的影響？**      
Office OneNote 2007 是一個數位筆記本，它可以提供地方讓使用者收集筆記和資訊；提供強大的搜尋功能，讓使用者快速尋找他們想要尋找的東西；以及提供好用的共用筆記本，讓使用者更有效的管理資訊超載與共同合作。

**因為這個弱點而承受風險的主要系統有哪些？**      
使用受影響軟體的系統 (例如工作站和終端機伺服器)，其風險最高。 若系統管理員允許使用者登入伺服器並執行程式，則伺服器可能遭受更大的風險。 然而，最佳實務強烈建議您制止這種行為。

**更新的作用何在？**      
更新程式是藉由修改 Microsoft Office 驗證統一資源定位器的方式，來移除弱點。

**當本安全性公告發行時，Microsoft 是否已接獲任何消息，指出這項弱點已遭有心人士利用？**      
否。 當本安全性公告初次發行時，Microsoft 並未接到任何有關本弱點已成為公開攻擊媒介的消息，也沒有發現任何以此概念發展的程式碼公開範例。
