Microsoft Security Bulletin MS08-001 - 重大
===========================================

Windows TCP/IP 中的弱點可能會允許遠端執行程式碼 (941644)
--------------------------------------------------------

發行: 2008年1月8日 | 更新: 2008年1月26日

**版本:** 3.0

### 一般資訊

#### 提要

這個重大的安全性更新可解決傳輸控制通訊協定暨網際網路通訊協定 (TCP/IP) 處理中兩項未公開報告的弱點。 成功利用此弱點的攻擊者可以取得受影響系統的完整控制權。 攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。

對於所有受支援版本的 Windows XP 與 Windows Vista，此乃重大的安全性更新；對於所有受支援版本的 Windows Server 2003，此乃重要的安全性更新；對於受支援版本的 Microsoft Windows 2000，此乃中度的安全性更新。如需更多資訊，請參閱本節中的＜受影響及不受影響的軟體＞小節。

此安全性更新會修正 Windows 核心對於內含多點傳送與 ICMP 要求之 TCP/IP 結構的處理方式，藉此解決此項弱點。 如需更多有關弱點的資訊，請參閱下節＜弱點資訊＞下的特定弱點項目的＜常見問題集 (FAQ)＞小節。

**建議。**  Microsoft 建議客戶立即套用此更新

**已知問題。** 無

#### 受影響及不受影響的軟體

下列軟體已經過測試判斷哪些版號或版本會受到影響。 其他版本超過它們的支援週期或不受影響。 若要瞭解您的軟體版本的支援週期，請造訪 [Microsoft 產品技術支援週期網站](http://go.microsoft.com/fwlink/?linkid=21742)。

Windows 核心 TCP/IP/IGMPv3 和 MLDv2 弱點 - CVE-2007-0069  
--------------------------------------------------------
  
Windows 核心對於儲存 IGMPv3 和 MLDv2 查詢狀態之 TCP/IP 結構的處理方式，含有遠端執行碼弱點。 Microsoft Windows XP、Windows Server 2003 與 Windows Vista 的受支援版本均支援 IGMPv3。除了 IGMPv3 之外，Windows Vista 還支援 MDLv2，增加對 IPv6 網路的多點傳送支援。 匿名攻擊者可以透過網路傳送蓄意製作的 IGMPv3 和 MLDv2 封包到電腦，以利用這項弱點。 成功利用此弱點的攻擊者可以取得受影響系統的完整控制權。 攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。
  
若要以一般性弱點清單中的標準項目來檢視此弱點，請參閱 [CVE-2007-0069](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-0069)。
  
#### Windows 核心 TCP/IP/IGMPv3 和 MLDv2 弱點 - CVE-2007-0069 的緩和因素
  
緩和因素是指存在於預設狀態中的設定、共用設定或一般最佳作法，可能會減少弱點影響的嚴重性。 下列緩和因素可能對您的狀況有所助益：
  
-   最佳實作的防火牆和標準預設防火牆設定有助於防止網路受到來自企業外的攻擊。 最佳方式建議連線至網際網路的系統盡可能曝露最少數量的連接埠。 封鎖多點傳送流量 (尤其是 IGMPv3 和 MLDv2) 的周邊防火牆有助於保護內部網路資產不受源自企業邊緣外部的攻擊影響。  
-   Microsoft Windows 2000 並未受此弱點影響。  
-   Windows Server 2003 依預設並沒有啟用任何多點傳送位址，不會受到此弱點的影響。 然而，安裝使用多點傳送的應用程式可能會造成作業系統有弱點。  
-   嘗試利用此弱點進行的攻擊，其最可能的結果是導致拒絕服務狀態。 但是，也可能導致遠端執行程式碼。
  
#### Windows 核心 TCP/IP/IGMPv3 和 MLDv2 弱點 - CVE-2007-0069 的因應措施
  
因應措施指的是無法徹底修正弱點，但有助於在套用更新之前封鎖已知攻擊模式的設定變更。 Microsoft 測試了下列因應措施和狀態，討論因應措施是否會降低功能。
  
-   **停用 IGMP 和 MLD 處理**  
  
    您可以在登錄中修改 IGMPLevel 控制項，協助防範攻擊者嘗試利用 IGMP 弱點。
  
    **警告：**如果使用「登錄編輯程式」的方式錯誤，可能造成嚴重問題，以致於您必須重新安裝作業系統。 Microsoft 無法保證您可以解決因為不正確使用 \[登錄編輯程式\] 所造成的問題。 請自行承擔使用「登錄編輯程式」的風險。 如需有關如何編輯登錄的資訊，請檢視「登錄編輯程式」(Regedit.exe) 中的＜變更機碼及數值＞說明主題，或是 Regedt32.exe 中的＜新增及刪除登錄中的資訊＞與＜編輯登錄資料＞說明主題。
  
    **注意：**我們建議您在編輯登錄之前先將之備份。
  
    若要停用 IGMP 和 MLD 處理，請依照下列步驟進行：
  
    1.  按一下 \[開始\]，按一下 \[執行\]，鍵入 **regedit**，然後按一下 \[確定\]。  
    2.  展開 \[HKEY\_LOCAL\_MACHINE\]。  
    3.  再依序展開 \[SYSTEM\]、\[CurrentControlSet\] 及 \[Services\]。  
    4.  依序展開 \[TCPIP\]、\[Parameters\] 及 \[IGMPLevel\]。  
    5.  將 \[DWORD\] 變更為 0。
  
    **注意：**您必須重新啟動系統，才能讓變更生效。
  
    **因應措施的影響：** IGMPv3 / MLDv2 將會停用。 需用這項服務的應用程式將不再依設計方式作用。
  
    **如何復原因應措施：** 使用上述步驟，將登錄機碼設定值設定為其原始值，或刪除此登錄機碼設定值。 依照預設此登錄機碼設定是不存在的。
  
-   **封鎖周邊防火牆的 IGMP 與 MLD**  
  
    防火牆最佳實作和標準預設防火牆設定有助於防止網路受到來自企業外部的 IGMP 與 MLD 攻擊。
  
    **因應措施的影響：** IGMPv3 和 MLDv2 將會被封鎖在企業周邊防火牆外，而需用企業周邊防火牆外的 IGMPv3 和 MLDv2 的應用程式將無法依設計方式作用。
  
-   **封鎖 Vista 防火牆的 IGMP 與 MLD 輸入**  
  
    按一下 \[控制台\]，再按一下 \[系統管理工具\]，然後連按兩下 \[具有進階安全性的 Windows 防火牆\]。
  
    若要封鎖 IGMP：
  
    1.  選取 \[輸入規則\]。  
    2.  選取 \[核心網路功能 - 網際網路群組管理通訊協定 (IGMP-In)\]。  
    3.  按一下滑鼠右鍵，然後選取 \[內容\]。  
    4.  選取 \[封鎖連線\]。
  
    若要封鎖 MLD：
  
    1.  選取 \[輸入規則\]。  
    2.  選取 \[核心網路功能 - 多點傳送接聽程式查詢 (ICMPv6-In)\]。  
    3.  按一下滑鼠右鍵，然後選取 \[內容\]。  
    4.  選取 \[封鎖連線\]。
  
-   **因應措施的影響：** IGMP/MLD 將會遭到封鎖，依賴多點傳送的應用程式或服務將無法再運作。
  
    **如何復原因應措施：** 使用上述的前三個步驟，並選取 \[允許所有的連線\]。
  
#### Windows 核心 TCP/IP/IGMPv3 和 MLDv2 弱點 - CVE-2007-0069 的常見問題集
  
**這個弱點的範圍為何？**    
這是遠端執行程式碼的弱點。 成功利用此弱點的攻擊者可以利用提高的權限在易受影響的系統上執行任意程式碼。 攻擊者接下來將能安裝程式，或檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。
  
**造成這個弱點的原因為何？**    
Windows 核心在儲存 TCP/IP 所處理的 IGMP 要求時，執行的驗證不足。
  
**什麼是 TCP/IP？**  
TCP/IP 係用於在網路上傳輸資料的通訊協定套件。
  
**Windows Small Business Server 和 Windows Home Server 的此弱點等級為何與 Windows Server 2003 不同？**  
Windows Server 2003 預設不會啟用 IGMP，因此不容易受此弱點的影響。 但是，Windows Small Business Server 2003 和 Windows Home Server 預設會啟用 IGMP，因此提高了受此弱點影響的風險。 由於風險較高，因此 Windows Small Business Server 2003 和 Windows Home Server 的嚴重性等級為「重大」。 我們建議擁有這些產品的客戶立即安裝此更新。
  
**什麼是 IGMP？**  
網際網路群組管理通訊協定 (IGMP) 用於 IPv4 多點傳播。 多點傳送是指在網路上單一傳送者與多位接收者之間的通訊。 IGMP 用於交換 IPv4 路由器之間的成員資格狀態資料，該類路由器支援多點傳送與多點傳送群組的成員。 請瀏覽 MSDN 網站，瞭解更多關於 IGMP 的資訊。
  
**什麼是 MLD？**  
多點傳送接聽程式探索 (MLD) 可讓您管理 IPv6 的子網路多點傳送成員資格。MLD 系列包含三種 IPv6 網際網路控制訊息通訊協定 (ICMPv6) 訊息，可取代 IPv4 適用的網際網路群組管理通訊協定 (IGMP)。請瀏覽 MSDN 網站，瞭解更多有關 [MLD](http://msdn2.microsoft.com/en-us/library/ms882958.aspx) 的資訊。
  
**攻擊者可能會利用這項弱點採取什麼行動？**    
成功利用此弱點的攻擊者可以執行任意程式碼。 攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。
  
**攻擊者如何利用這項弱點？**    
攻擊者可以建立蓄意製作的網路封包，並將封包傳送給受影響的系統，以利用此弱點。
  
**因為這個弱點而承受風險的主要系統有哪些？**    
易遭受此弱點影響的，以用戶端系統為主。 然而，任何使用多點傳送的系統均可能受此弱點影響。
  
**更新的作用何在？**    
此安全性更新會修正 Windows 核心對於內含 IGMP 要求之 TCP/IP 結構的處理方式，進而移除此項弱點。
  
**當安全性公告發行時，這項弱點是否已被揭發出來？**    
否。 Microsoft 是經由可靠的來源接獲有關這項弱點的訊息。 當本安全性公告初次發行時，Microsoft 尚未接獲任何有關此弱點已被發佈出來的消息。 本安全性公告除未公開揭露弱點之外，並說明其他內部調查所發現的議題。
  
**當本安全性公告發行時，Microsoft 是否已接獲任何消息，指出這項弱點已遭有心人士利用？**    
否。 當本安全性公告初次發行時，Microsoft 並未接到任何有關本弱點已成為公開攻擊媒介的消息，也沒有發現任何以此概念發展的程式碼公開範例。
  
Windows 核心 TCP/IP/ICMP 弱點 - CVE-2007-0066  
---------------------------------------------
  
由於 Windows 核心處理分散路由器通告 ICMP 查詢的方式不當，因此 TCP/IP 中存在拒絕服務的弱點。 ICMP 路由器探索通訊協定 (RDP) 依預設並未啟用，若要利用此弱點，則必須啟用該協定。 然而，在 Windows 2003 Server 和 Windows XP 上，RDP 可透過 DHCP 中的設定或登錄中的設定開啟，若在 Windows 2000 上，則可透過登錄的設定開啟。 匿名攻擊者可以透過網路傳送蓄意製作的 ICMP 封包到電腦，以利用這項弱點。 成功利用此弱點的攻擊者可使該電腦停止回應，並自動重新啟動。
  
若要以一般性弱點清單中的標準項目來檢視此弱點，請參閱 [CVE-2007-0066](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-0066)。
  
#### Windows 核心 TCP/IP/ICMP 弱點 - CVE-2007-0066 的緩和因素
  
緩和因素是指存在於預設狀態中的設定、共用設定或一般最佳作法，可能會減少弱點影響的嚴重性。 下列緩和因素可能對您的狀況有所助益：
  
-   路由器探索通訊協定 (RDP) 依預設並未啟用，若要利用此弱點，則必須啟用該協定。 如需更多資訊，請參閱 [Microsoft 知識庫文件編號 269734](http://technet2.microsoft.com/windowsserver/en/library/db56b4d4-a351-40d5-b6b1-998e9f6f41c91033.mspx?mfr=true)＜依預設會停用路由器探索通訊協定＞(本文為機器翻譯)。  
-   Windows Vista 不受此弱點的影響。
  
#### Windows 核心 TCP/IP/ICMP 弱點 - CVE-2007-0066 的因應措施
  
因應措施指的是無法徹底修正弱點，但有助於在套用更新之前封鎖已知攻擊模式的設定變更。 Microsoft 測試了下列因應措施和狀態，討論因應措施是否會降低功能。
  
-   **停用路由器探索通訊協定處理功能**  
  
    依照預設，路由器探索通訊協定在所有受影響的平台上均為停用狀態。 若路由器探索通訊協定已經啟用，下列因應措施可將其停用。 您可以在登錄中設定 PerformRouterDiscovery 的值，協助防範攻擊者嘗試利用 ICMP 弱點。
  
    **警告：**如果使用「登錄編輯程式」的方式錯誤，可能造成嚴重問題，以致於您必須重新安裝作業系統。 Microsoft 無法保證您可以解決因為不正確使用 \[登錄編輯程式\] 所造成的問題。 請自行承擔使用「登錄編輯程式」的風險。 如需有關如何編輯登錄的資訊，請檢視「登錄編輯程式」(Regedit.exe) 中的＜變更機碼及數值＞說明主題，或是 Regedt32.exe 中的＜新增及刪除登錄中的資訊＞與＜編輯登錄資料＞說明主題。
  
    **注意：**我們建議您在編輯登錄之前先將之備份。
  
    若要停用 ICMP 處理，請依照下列步驟進行：
  
    1.  按一下 \[開始\]，按一下 \[執行\]，鍵入 **regedit**，然後按一下 \[確定\]。  
    2.  展開 \[HKEY\_LOCAL\_MACHINE\]。  
    3.  再依序展開 \[SYSTEM\]、\[CurrentControlSet\] 及 \[Services\]。  
    4.  依序展開 \[TCPIP\]、\[Parameters\] 及 \[Interfaces\]。  
    5.  選取 \[interface\_name\] 並將 \[PerformRouterDiscovery\] 值設為 **0**。
  
    **注意：**您必須重新啟動系統，才能讓變更生效。
  
    **因應措施的影響：** 路由器探索將會停用。
  
    **如何復原因應措施：** 使用上述步驟，將 **PerformRouterDiscovery** 登錄機碼設定值設定為其原始值，或刪除此登錄機碼設定值。
  
#### Windows 核心 TCP/IP/ICMP 弱點 - CVE-2007-0066 的常見問題集
  
**這個弱點的範圍為何？**    
這是一項拒絕服務的弱點。 成功利用此弱點的攻擊者可使受影響的系統停止回應並要求系統重新啟動。 在這段期間，系統將無法回應要求。 請注意拒絕服務弱點不會讓攻擊者執行程式碼或提高其使用者權限，但會導致受影響系統停止接受要求。

**什麼是 TCP/IP？**  
TCP/IP 係用於在網路上傳輸資料的通訊協定套件。
  
**什麼是 ICMP？**  
網際網路控制訊息通訊協定 (ICMP) 是一個管理連線能力和路由的維護通訊協定。 如需更多關於 ICMP 的功能和實作資訊，請參閱 [RFC 792： 網際網路控制訊息通訊協定 DARPA 網際網路程式通訊協定規格](http://www.ietf.org/rfc/rfc0792.txt?number=792) (英文)。
  
**攻擊者可能會利用這項弱點採取什麼行動？**    
攻擊者可以傳送蓄意製作的 ICMP 封包到受影響的系統，以利用此弱點。 該訊息接著便可能導致受影響的系統停止回應並重新啟動。
  
**攻擊者如何利用這項弱點？**    
攻擊者可以建立蓄意製作的網路封包，並將此封包傳送給受影響的系統，以利用此弱點。
  
**因為這個弱點而承受風險的主要系統有哪些？**    
所有啟用 ICMP 路由器探索通訊協定的系統都可能受到這個弱點影響。
  
**更新的作用何在？**    
此安全性更新會修正 Windows 核心對於內含路由器通告 IGMP 封包之 TCP/IP 結構的處理方式，進而移除此項弱點。
  
**當安全性公告發行時，這項弱點是否已被揭發出來？**    
否。 Microsoft 是經由可靠的來源接獲有關這項弱點的訊息。 當本安全性公告初次發行時，Microsoft 尚未接獲任何有關此弱點已被發佈出來的消息。 本安全性公告除未公開揭露弱點之外，並說明其他內部調查所發現的議題。

-   **檔案版本驗證**  

    由於 Microsoft Windows 的版本眾多，您電腦上實際執行的步驟可能會與此處描述的不同。 如遇到不同的狀況，請參閱產品的說明文件以完成這些步驟。

    1.  按一下 \[開始\]，然後按一下 \[搜尋\]。
    2.  在 \[搜尋結果\] 窗格中，在 \[搜尋小幫手\] 下按一下 \[所有檔案和資料夾\]。
    3.  在 \[部份或完整的檔案名稱\] 方塊中，輸入適當檔案資訊表中的檔案名稱，再按一下 \[搜尋\]。
    4.  在檔案清單中，用滑鼠右鍵按一下適當檔案資訊表中的某個檔案名稱，再按 \[內容\]。  
 
        **注意**：視所安裝的作業系統或程式的版本而定，檔案資訊表中列出的檔案未必會全部安裝。
    5.  在 \[版本\] 索引標籤上，找出您電腦上安裝的檔案版本，同時與適當檔案資訊表中記錄的版本做一比較。  
        **注意：**在安裝時，檔案版本以外的屬性可能會變更。 若要驗證是否成功套用更新程式，利用檔案資訊表比對其他檔案屬性並不是妥當的做法。 此外，在某些情況下，檔案的名稱在安裝時可能會有所變更。 如果缺少檔案或版本資訊，請採用其他可用的方法來驗證更新程式的安裝情形。

-   **登錄機碼驗證**

**確認已經套用更新**  

-   **Microsoft Baseline Security Analyzer**  

    您可以使用 Microsoft Baseline Security Analyzer (MBSA) 工具確認受影響系統上是否已經套用安全性更新程式。 如需詳細資訊，請參閱本公告前面的＜偵測與部署工具及指南＞一節。

-   **檔案版本驗證**  

    由於 Microsoft Windows 的版本眾多，您電腦上實際執行的步驟可能會與此處描述的不同。 如遇到不同的狀況，請參閱產品的說明文件以完成這些步驟。

    1.  按一下 \[開始\]，然後按一下 \[搜尋\]。
    2.  在 \[搜尋結果\] 窗格中，在 \[搜尋小幫手\] 下按一下 \[所有檔案和資料夾\]。
    3.  在 \[部份或完整的檔案名稱\] 方塊中，輸入適當檔案資訊表中的檔案名稱，再按一下 \[搜尋\]。
    4.  在檔案清單中，用滑鼠右鍵按一下適當檔案資訊表中的某個檔案名稱，再按 \[內容\]。  
 
        **注意**：視所安裝的作業系統或程式的版本而定，檔案資訊表中列出的檔案未必會全部安裝。
    5.  在 \[版本\] 索引標籤上，找出您電腦上安裝的檔案版本，同時與適當檔案資訊表中記錄的版本做一比較。  
        **注意：**在安裝時，檔案版本以外的屬性可能會變更。 若要驗證是否成功套用更新程式，利用檔案資訊表比對其他檔案屬性並不是妥當的做法。 此外，在某些情況下，檔案的名稱在安裝時可能會有所變更。 如果缺少檔案或版本資訊，請採用其他可用的方法來驗證更新程式的安裝情形。

-   **登錄機碼驗證**  

    您也可以查看本節「參考表」中所列的登錄機碼，確認此安全性更新所安裝的檔案。

    這些登錄機碼可能未包含完整的安裝檔案清單。 而且，當系統管理員或 OEM 將安全性更新整合或匯集到 Windows 安裝原始檔時，可能無法正確建立這些登錄機碼。

**注意：**如需有關 wusa.exe 安裝程式的詳細資訊，請參閱 [Microsoft 知識庫文件編號 934307](http://support.microsoft.com/kb/934307)。

-   **檔案版本驗證**
