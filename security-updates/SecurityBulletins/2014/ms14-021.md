Microsoft 資訊安全公告 MS14-021 - 重大
======================================

Internet Explorer 適用的資訊安全更新 (2965111)
----------------------------------------------

發行日期： (2014 年 5 月 1 日)

**版本：** 1.1

一般資訊
--------

### 提要

此資訊安全更新可解決 Internet Explorer 中一項已公開揭露的資訊安全風險。如果使用者使用受影響版本的 Internet Explorer 檢視蓄意製作的網頁，此資訊安全風險可能會允許遠端執行程式碼。成功利用此資訊安全風險的攻擊者可以取得與目前使用者相同的使用者權限。系統上帳戶使用者權限較低的使用者，其受影響的程度比擁有系統管理權限的使用者要小。

對於受影響的 Windows 用戶端上的 Internet Explorer 6 (IE 6)、Internet Explorer 7 (IE 7)、Internet Explorer 8 (IE 8)、Internet Explorer 9 (IE 9)、Internet Explorer 10 (IE 10) 和 Internet Explorer 11 (IE 11)，此資訊安全更新的等級為「重大」；對於受影響的 Windows 伺服器上的 Internet Explorer 6 (IE 6)、Internet Explorer 7 (IE 7)、Internet Explorer 8 (IE 8)、Internet Explorer 9 (IE 9)、Internet Explorer 10 (IE 10) 和 Internet Explorer 11 (IE 11)，等級為「中度」。如需更多資訊，請參閱本節中的＜受影響及不受影響的軟體＞小節。

此資訊安全更新會修改 Internet Explorer 處理記憶體中物件的方式，藉此解決此資訊安全風險。如需更多有關此資訊安全風險的資訊，請參閱本公告稍後特定資訊安全風險項目的＜常見問題集 (FAQ)＞小節。

此資訊安全更新可解決 [Microsoft 資訊安全摘要報告 2963983](https://technet.microsoft.com/library/security/2963983) 中首先提出的資訊安全風險。

**建議。** 大部分客戶都已啟用自動更新，並且不必須採取任何行動，因為資訊安全更新將自動下載和安裝。如需有關自動更新中特定設定選項的資訊，請參閱 [Microsoft 知識庫文件編號 294871](https://support.microsoft.com/kb/294871)。對於沒有啟用自動更新的客戶，可採取[開啟或關閉自動更新](http://windows.microsoft.com/zh-tw/windows/turn-automatic-updating-on-off)中的步驟來啟用自動更新。

若是系統管理員和企業安裝，或是想要手動安裝此資訊安全更新的使用者 (包括未啟用自動更新的客戶)，Microsoft 建議客戶透過更新管理軟體或利用 [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) 服務檢查更新以立即套用更新。另外亦可透過在本公告後文中**受影響的軟體**表格所列的下載連結取得更新。

另請參閱本公告下文的＜偵測與部署工具及指南＞一節。

更新常見問題集
--------------

<span id="sectionToggle1"></span>
**Internet Explorer 11 列有多項更新。我是否需要安裝所有更新？**  
否。依據您的系統所設定的更新接收方式，可能僅需針對 Internet Explorer 11 套用其中一項更新。

對於在 Windows 7 或 Windows Server 2008 R2 上執行 Internet Explorer 11 的系統：

-   2964358 更新適用於已安裝 2929437 更新的系統。
-   2964444 更新適用於未安裝 2929437 更新的系統。

對於在 Windows 8.1、Windows Server 2012 R2 或 Windows RT 8.1 上執行 Internet Explorer 11 的系統：

-   2964358 更新適用於已安裝 2919355 更新的系統。
-   2964444 更新適用於未安裝 2919355 更新的系統。

**Internet Explorer 11 對於安裝 2964358 更新是否有任何必要條件？**  
是。在 Windows 8.1、Windows Server 2012 R2 或 Windows RT 8.1 上執行 Internet Explorer 11 的客戶必須先安裝於 2014 年 4 月發行的 2919355 更新後才可安裝 2964358 更新。如需有關這項必要更新的詳細資訊，請參閱 [Microsoft 知識庫文件編號 2919355](https://support.microsoft.com/kb/2919355)。

在 Windows 7 或 Windows Server 2008 R2 上執行 Internet Explorer 11 的客戶必須先安裝於 2014 年 4 月發行的 2929437 更新後才可安裝 2929437 更新。如需有關這項必要更新的詳細資訊，請參閱 [Microsoft 知識庫文件編號 2929437](https://support.microsoft.com/kb/2929437)。

**此 MS14-021 更新是 Internet Explorer 的積存資訊安全更新嗎？**  
否。此 MS14-021 資訊安全更新只會解決此公告中描述的資訊安全風險。

**我是否必須安裝最新的 Internet Explorer 積存資訊安全更新？**  
是。此資訊安全更新 MS14-021 並非積存資訊安全更新。若客戶尚未安裝最新的 Internet Explorer 積存資訊安全更新，可能會在安裝 MS14-021 更新後發生相容性問題 (例如 Internet Explorer 可能斷斷續續發生停止回應的情形)。

請注意務必依循安裝順序。若要避免相容性問題，您必須在為 Internet Explorer 安裝最新的積存資訊安全更新之後，才能安裝 MS14-021 資訊安全更新。

**如果我已套用在 Microsoft 資訊安全摘要報告 2963983 中所述的因應措施，我是否需要在套用這項更新之前先復原此因應措施？**  
這取決於所套用的因應措施內容。請參閱下方清單：

-   如果您套用的因應措施是**修改 VGX.DLL 的存取控制清單 (ACL)**，則您必須先復原此因應措施後才可套用這項資訊安全更新。若要復原此因應措施，請從提高權限的命令提示字元中，執行下列命令： 

        ```
        echo y| cacls "%CommonProgramFiles%\Microsoft Shared\VGX\vgx.dll" /g original ACLs
        ```

    其中 *original ACLs* 是指套用因應措施之前的原始使用者存取權限。或者，您也可以將 vgx.dll 的存取權限設定開放給從提高權限的命令提示字元中使用下列命令的每個人：

    

        ```
        echo y| cacls "%CommonProgramFiles%\Microsoft Shared\VGX\vgx.dll" /E /R everyone
        ```

    您必須在 64 位元系統上針對 "%CommonProgramFiles(x86)%\\Microsoft Shared\\VGX\\vgx.dll" 重複上述動作。另請注意，如果是使用 icacls (而非 cacls) 來套用因應措施，則請使用 icacls 還原原始的 ACL。

    例如，您可以從提高權限的命令提示字元中執行這些 icacls 的命令：

    

        ```
        icacls "%CommonProgramFiles%\Microsoft Shared\VGX\vgx.dll" /grant everyone:(F)
        icacls "%CommonProgramFiles%\Microsoft Shared\VGX\vgx.dll" /restore %TEMP%\VGX_ACL.TXT
        ```

    其中 *VGX\_ACL.TXT* 是指用於在使用 icacls 套用因應措施時儲存原始使用者存取權限所建立的備份檔案。

    您必須在 64 位元系統上針對 "%CommonProgramFiles(x86)%\\Microsoft Shared\\VGX\\vgx.dll" 重複上述動作。

-   如果您套用的因應措施是**解除登錄 VGX.DLL**，您無需復原因應措施就可直接套用資訊安全更新。然而，此資訊安全更新並不會重新登錄 vgx.dll。請參閱這個資訊安全風險的**＜因應措施＞**小節，取得如何重新登錄 vgx.dll 的步驟。

    **注意** 對於套用原本在版本 1.0 的 Microsoft 資訊安全摘要報告 2963983 中所載示之**解除登錄 VGX.DLL** 因應措施的 64 位元系統使用者，請依照此資訊安全公告所述，為 64 位元的系統套用最新的因應措施。

-   如果您套用的是任何並非列於 [Microsoft 資訊安全摘要報告 2963983](https://technet.microsoft.com/library/security/2963983) 中的因應措施，您無需復原因應措施就可直接套用資訊安全更新。

**此公告中指出的資訊安全風險如何影響 Server Core 安裝？**

Internet Explorer 記憶體損毀資訊安全風險 - CVE-2014-1776
--------------------------------------------------------

<span id="sectionToggle3"></span>
當 Internet Explorer 不正確地存取記憶體中的物件時，即存在遠端執行程式碼資訊安全風險。此資訊安全風險可能會損毀記憶體，使攻擊者有機會以目前使用者的權限層級執行任意程式碼。

### 緩和因素
  
緩和因素是指存在於預設狀態中的設定、共用設定或一般最佳作法，可能會減少資訊安全風險影響的嚴重性。下列緩和因素可能對您的狀況有所助益：
  
-   在網頁型攻擊的案例中，攻擊者可以針對這個經由 Internet Explorer 引起的資訊安全風險來設計並架設蓄意製作的網站，然後引誘使用者檢視該網站。攻擊者也可能利用受侵害的網站，以及接受或裝載使用者提供內容或廣告的網站。這些網站可能含有經過蓄意製作並利用此資訊安全風險的內容。但是，攻擊者無法強迫使用者檢視受攻擊者控制的內容， 而是引誘使用者自行前往。一般的做法是設法讓使用者點選電子郵件訊息或 Instant Messenger 中通往攻擊者網站的連結，或設法讓他們開啟經由電子郵件傳送的附件。  
-   成功利用此資訊安全風險的攻擊者可以取得與目前使用者相同的使用者權限。系統上帳戶使用者權限較低的使用者，其受影響的程度比擁有系統管理權限的使用者要小。  
-   根據預設，目前在 Windows 8 與 Windows 8.1 上所提供的沈浸式瀏覽體驗是搭配[加強的受保護模式 (EPM)](http://go.microsoft.com/fwlink/p/?linkid=257809) 執行。舉例來說，如果使用者在新型的 Windows 平板上使用具備觸控功能的 Internet Explorer 11 瀏覽器，依預設也會使用「加強的受保護模式」。「加強的受保護模式」採用進階的資訊安全保護功能，有助於緩和 64 位元系統遭到此資訊安全風險利用的問題。如需更多資訊，請參閱此資訊安全風險＜常見問題集＞關於「加強的受保護模式」的部份。  
-   依照預設，所有受支援版本的 Microsoft Outlook、Microsoft Outlook Express 及 Windows Mail 都會開啟 \[限制的網站\] 區域的 HTML 電子郵件訊息。\[限制的網站\] 區域會停用指令碼與 ActiveX 控制項，進而協助降低攻擊者能夠利用此資訊安全風險執行惡意程式碼的風險。如果使用者按下電子郵件訊息中的連結，仍有可能因為網頁式攻擊而受此資訊安全風險遭利用的影響。  
-   依照預設，Windows Server 2003、Windows Server 2008、Windows Server 2008 R2、Windows Server 2012 及 Windows Server 2012 R2 上的 Internet Explorer 會以稱為[增強式資訊安全設定](http://technet.microsoft.com/library/dd883248)的受限制模式執行。此模式可緩和此資訊安全風險。如需更多資訊，請參閱此資訊安全風險＜常見問題集＞關於 Internet Explorer 增強式資訊安全設定的部份。
  
### 因應措施
  
因應措施指的是無法徹底修正資訊安全風險，但有助於在套用更新之前封鎖已知攻擊模式的設定變更。Microsoft 測試了下列因應措施和狀態，討論因應措施是否會降低功能：
  
-   **解除登錄 VGX.DLL**
  
    **32 位元 Windows 系統：**
  
    **附註** 必須從提高權限的命令提示字元中輸入下列命令。
  
    1.  從提高權限的命令提示字元中，輸入下列命令： ```

  
            "%SystemRoot%\System32\regsvr32.exe" -u "%CommonProgramFiles%\Microsoft Shared\VGX\vgx.dll"
  
          
```
  
        在執行命令後，應會出現一個對話方塊，以確認解除登錄程序已完成。按一下 \[確定\] 關閉這個話方塊。請注意，**-s** 命令列參數可用於抑制對話方塊，以允許撰寫這個命令的指令碼。
  
    2.  關閉並重新開啟 Internet Explorer，讓變更生效。
  
    **64 位元的 Windows 系統：**
  
    **注意** 對於套用原本在版本 1.0 的 [Microsoft 資訊安全摘要報告 2963983](https://technet.microsoft.com/library/security/2963983) 中所載示之**解除登錄 VGX.DLL** 因應措施的 64 位元系統使用者，請如下所述為 64 位元的系統套用最新的因應措施。

### 常見問題集
  
**這個資訊安全風險的範圍為何？**  
這是目前使用者內容中的一個遠端執行程式碼資訊安全風險。
  
**造成這項資訊安全風險的原因為何？**  
當 Internet Explorer 未正確存取記憶體中的物件時，記憶體可能發生損毀，讓攻擊者能以目前使用者的權限層級執行任意程式碼。
  
**攻擊者可能會利用此資訊安全風險採取什麼行動？**  
成功利用此資訊安全風險的攻擊者可以取得與目前使用者相同的使用者權限。如果使目前用者以系統管理的使用者權限登入，成功利用此資訊安全風險的攻擊者可以取得受影響系統的完整控制權。攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。
  
**攻擊者如何利用這項資訊安全風險？**  
攻擊者可以針對這個經由 Internet Explorer 引起的資訊安全風險來設計並架設蓄意製作的網站，然後引誘使用者檢視該網站。攻擊者也可能利用受侵害的網站，以及接受或裝載使用者提供內容或廣告的網站。這些網站可能含有經過蓄意製作並利用此資訊安全風險的內容。但是，攻擊者無法強迫使用者檢視受攻擊者控制的內容， 而是引誘使用者自行前往。一般的做法是設法讓使用者點選電子郵件訊息或 Instant Messenger 中通往攻擊者網站的連結，或設法讓他們開啟經由電子郵件傳送的附件。
  
**因為這個資訊安全風險而承受風險的主要系統有哪些？**  
常使用 Internet Explorer 的系統 (例如工作站或終端伺服器) 的風險最大。
  
**我使用的是 Windows Server 2003、Windows Server 2008、Windows Server 2008 R2、Windows Server 2012 或 Windows Server 2012 R2 的 Internet Explorer。這樣是否會減輕此資訊安全風險的影響？**  
是。依照預設，Windows Server 2003、Windows Server 2008、Windows Server 2008 R2、Windows Server 2012 及 Windows Server 2012 R2 上的 Internet Explorer 會以稱為[增強式資訊安全設定](http://technet.microsoft.com/library/dd883248)的受限制模式執行。增強式資訊安全設定是一組預先設定好的 Internet Explorer 設定，可以降低使用者或系統管理員在伺服器下載及執行蓄意製作之網頁內容的可能性。對於您尚未新增至 Internet Explorer \[信任的網站\] 區域的網站，這是一種緩和因素。
  
**EMET 4.x 是否有助於緩和企圖利用此資訊安全風險的攻擊？**  
是，EMET 4.0 和 EMET 4.1 均有助於緩和此問題。Enhanced Mitigation Experience Toolkit (EMET) 可讓使用者管理資訊安全緩和技術，此技術讓攻擊者更難利用特定軟體中的資訊安全風險。在有安裝 EMET，並將其設定為能用於 Internet Explorer 的系統上，EMET 有助於緩和 Internet Explorer 中的此資訊安全風險。
  
**附註** EMET 3.0 無法減輕此問題。
  
如需更多關於 EMET 的資訊，請參閱 [The Enhanced Mitigation Experience Toolkit](http://technet.microsoft.com/security/jj653751)。
  
**「加強的受保護模式 (EPM)」與「加強的受保護模式的 64 位元處理」是否有助於緩和企圖利用此資訊安全風險的攻擊？**  
是。對於 Internet Explorer 10 與 Internet Explorer 11，啟用「加強的受保護模式 (EPM)」與「加強的受保護模式的 64 位元處理」有助於緩和企圖在 64 位元系統上利用此資訊安全風險的攻擊。「加強的受保護模式」無法緩和此資訊安全風險在 32 位元系統上的攻擊，因為沒有「加強的受保護模式的 64 位元處理」功能。
  
**附註** 新型的沉浸式瀏覽器依預設會啟用「加強的受保護模式」。對於桌上型電腦上的 Internet Explorer，EPM 必須以手動方式啟用。
  
對於執行適用於 x64 型系統的 Windows 7 與適用於 x64 型系統的 Windows 8 之桌上型電腦上的 Internet Explorer 10，您必須啟用 **\[加強的受保護模式 (EPM)\]**。如果您在這些作業系統上啟用 EPM，另外也會啟用 \[**加強的受保護模式的 64 位元處理\]**。Internet Explorer 10 並無個別用來開啟或關閉這項功能的選項。
  
對於執行適用於 x64 型系統的 Windows 7 與適用於 x64 型系統的 Windows 8.1 之桌上型電腦上的 Internet Explorer 11，您必須分別啟用 **\[加強的受保護模式 (EPM)\]** 與 **\[加強的受保護模式的 64 位元處理\]**。
  
請注意，啟用「加強的受保護模式」之後，不相容的附加元件將自動停用。如果碰到的網站需要使用停用的附加元件，您可以僅針對該網站而停用「加強的受保護模式」。如果您為了網站而停用「加強的受保護模式」，則瀏覽該網站時將不會套用 EPM 所提供的任何保護。如需更多關於 EPM 的資訊，請參閱[傳統型 IE 上加強的受保護模式](http://msdn.microsoft.com/library/ie/dn265025)。
  
**這項更新有何作用？**  
此更新會修改 Internet Explorer 處理記憶體中物件的方式，藉此解決此資訊安全風險。
