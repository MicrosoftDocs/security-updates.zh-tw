弱點詳細資料
------------

<span></span>
#### Microsoft Indexing Service 弱點 - CVE-2006-0032：

由於 Indexing Service 處理查詢驗證的方式，使其含有資訊洩漏的弱點。 這項弱點可能會允許攻擊者代替使用者來執行用戶端指令碼。 指令碼可能會偽造內容、洩漏資訊、在受影響的網站上採取使用者可能會採取的行為。

#### Microsoft Indexing Service 弱點 - CVE-2006-0032 的緩和因素：

-   根據預設，Internet Information Services（IIS） 不會安裝在 Windows XP 或 Windows Server 2003 上。
-   在 Windows Server 2003 中，Indexing Service 預設為停用。
-   在 Windows Server 2003 中，即使安裝了 Indexing Service，依照預設仍然無法從 IIS 存取。 必須利用手動步驟，使 IIS 成為 Indexing Service 的網頁式介面。 依照預設，Indexing Service 僅用來執行本機及遠端檔案系統查詢。

#### Microsoft Indexing Service 弱點 - CVE-2006-0032 的因應措施：

Microsoft 已經測試過以下的因應措施。 雖然這些因應措施不能徹底解決弱點，但有助於封鎖已知的攻擊行為。 如果因應措施會降低功能，以下小節將會描述功能降低的情況。

-   **請勿從擔任伺服器角色的系統瀏覽網際網路**
    請勿從擔任任何其他伺服器角色的網頁伺服器或系統瀏覽網際網路。
-   **在 Internet Explorer 中停用網頁編碼自動偵測**
    在 Internet Explorer 中停用網頁編碼自動偵測，能協助防止受影響系統遭到嘗試利用此弱點的攻擊。

    設定元件和服務：

    1.  啟動 Internet Explorer。
    2.  按一下 \[檢視\]，再按一下 \[編碼\]。如果 \[自動選取\] 旁邊具有核取符號，則以滑鼠左鍵按一下 \[自動選取\] 以取消選取此設定。

    **因應措施的影響：** Internet Explorer 可能無法正確偵測未指定預設字元編碼的網頁上使用的編碼，從而導致編碼字元無法正確顯示。 使用者仍可手動選取正確的編碼類型以正確顯示頁面。

-   **在執行 IIS 5.0 的 Windows 2000 上執行 URLScan**
    安裝 [URLScan](http://www.microsoft.com/technet/security/tools/urlscan.mspx) 有助於保護受影響的系統，避免遭受嘗試利用此弱點的攻擊。

    1.  安裝 URLScan，依據其預設設定，URLScan 2.5 將封鎖對於受影響 Index Server 延伸模組的 .IDA、.IDQ 及 .HTW 檔案之要求。
    2.  重新啟動 IISAdmin 與 WWW Publishing 等服務讓變更生效。

    **因應措施的影響：** URLScan 的預設設定將封鎖對 Index Server 檔案類型 (.IDA、.IDQ、.HTW) 的要求，因此將會封鎖使用這些檔案類型的所有網頁搜尋。

-   **在執行 IIS 5.0 的 Windows 2000 中從 Internet Information Service 移除 Index Server ISAPI 延伸模組指令碼對應**
    從 IIS 中移除 Index Server ISAPI 延伸模組指令碼對應，能協助防止受影響系統遭到嘗試利用此弱點的攻擊。

    1.  按一下 \[開始\]，再按 \[控制台\]。或是，指向 \[設定\]，然後按一下 \[控制台\]。
    2.  按兩下 \[系統管理工具\]。
    3.  按兩下 \[Internet Information Services\]。
    4.  在 WebServer 上按一下滑鼠右鍵，然後選取 \[內容\]。
    5.  . 在主要內容中進行選取，選取 \[WWW Service\]，然後按一下 \[編輯\]。
    6.  在 \[WWW ServiceMaster 內容\] 中，選取 \[主目錄\]，然後按一下 \[設定\]。
    7.  在 \[應用程式設定\] 中，將 .HTW、.IDA 及 .IDQ 等副檔名反白，然後按一下 \[移除\]。

    **因應措施的影響：** URLScan 的預設設定將封鎖對 Index Server 檔案類型 (.IDA、.IDQ、.HTW) 的要求，因此將會封鎖使用這些檔案類型的所有網頁搜尋。

-   **移除 Indexing Service**
    **如果不再需要 Indexing Service，您可以按照此程序以將其移除。**

    設定元件和服務：

    1.  按一下 \[開始\]，再按 \[控制台\]。或是，指向 \[設定\]，然後按一下 \[控制台\]。
    2.  按兩下 \[新增或移除程式\]。
    3.  按一下 \[新增/移除 Windows 元件\]。
    4.  按一下以清除 \[Indexing Service\] 核取方塊，移除 Indexing Service。
    5.  依照螢幕上的指示執行，完成 \[Windows 元件精靈\]。

    **因應措施的影響：** 如果移除了此服務，透過周遊資料夾階層與掃描所要求字串的每個檔案仍可提供所有搜尋功能，但搜尋回應速度將會較慢。 如果使用 MMC Indexing Service 嵌入式管理單元建立新目錄，該目錄將保持離線狀態，直至啟動該服務。

-   **在執行 IIS 6.0 的 Windows 2003 中從 IIS 停用 Indexing Service 延伸模組**
    **如果不再需要 Indexing Service 延伸模組，您可以按照此程序以將其停用。**

    設定元件和服務：

    1.  按兩下 \[系統管理工具\]。
    2.  按兩下 \[Internet Information Services\]。
    3.  按一下 \[網頁服務延伸\]。
    4.  按一下 \[索引服務\]。
    5.  按一下 \[禁止\]。

    **因應措施的影響：** 如果移除了此服務延伸模組，透過周遊資料夾階層與掃描所要求字串的每個檔案仍可提供所有搜尋功能，但搜尋回應速度將會較慢。 如果使用 MMC Indexing Service 嵌入式管理單元建立新目錄，該目錄將保持離線狀態，直至啟動該服務。

#### Microsoft Indexing Service 弱點 - CVE-2006-0032 的常見問題集：

**這個弱點的範圍為何？**  
由於 Indexing Service 處理查詢驗證的方式，使其含有資訊洩漏的弱點。 這項弱點可能會允許攻擊者代替使用者來執行用戶端指令碼。 指令碼可能會偽造內容、洩漏資訊、在受影響的網站上採取使用者可能會採取的行為。

**造成這個弱點的原因為何？**  
Indexing Service 未適當地驗證查詢參數，導致有執行跨網站指令碼的可能性。

**什麼是 Indexing Service？**  
Indexing Service 是受影響作業系統的基礎服務。 它的原名是 Index Server，原始功能是為 Internet Information Services (IIS) 網頁伺服器的內容製作索引。 Indexing Service 目前已可為檔案系統與虛擬網頁伺服器兩者的內容與屬性建立索引目錄。Indexing Service 可供應用程式與指令碼使用，以提供在檔案系統或網頁伺服器中高效率管理、查詢資訊及製作索引的方式。 Indexing Service 同時也提供查詢機制，可以有效率地存取目錄中的資訊。 索引資訊是利用 Microsoft 所提供篩選器 (也可選擇使用自訂篩選器)，篩選檔案系統及網頁伺服器所產生。

**什麼是跨網站指令碼 ?**
跨網站指令碼 (XSS) 是一種安全性弱點，讓攻擊者可以利用網站將程式碼「插入」使用者的工作階段。 此攻擊涉及的是動態產生 HTML 頁面的網頁伺服器。 若這些伺服器將瀏覽器輸入的資料內嵌於所產生的動態頁面傳回給瀏覽器，這些伺服器可能會被操控並將內容包含在動態頁面中。 這會允許惡意指令碼被執行。 網頁瀏覽器對於「受信任」網站的基本假設，加上使用 Cookie 來記憶常瀏覽網站的狀態，都可能會擴大此問題。 此攻擊不會修改網站內容。 它會插入可在瀏覽器與受信任伺服器相關聯之安全性內容中執行的新惡意指令碼。

**跨網站指令碼如何運作？**  
網頁含有文字和 HTML 標記。 這些文字和 HTML 標記是由伺服器產生並且由用戶端解譯。 產生靜態網頁的伺服器對於用戶端解譯伺服器所傳送的網頁的方式，具有完整的控制權。 然而，產生動態網頁的伺服器對於用戶端解譯伺服器輸出的方式，便不具有控制權。 如果在動態網頁內加入了不受信任的內容，伺服器和用戶端都沒有足夠的資訊可以辨識這種情況，也無法採取保護措施。

**攻擊者可能會利用這項弱點採取什麼行動？**  
成功利用此弱點的攻擊者可以執行惡意程式碼。 如果該惡意指令碼被執行，它將會在該用戶端使用者的安全性內容中執行。 此指令碼可以在使用者的電腦上進行任何該網站經授權可進行的動作。 這些動作可能包含：監控使用者的網路工作階段、轉送資訊予第三方、在使用者的系統上執行其他程式碼，以及讀取或寫入 Cookie。

**攻擊者如何利用這項弱點？**  
在網頁式攻擊的案例中，攻擊者可架設一個網站，並在其中包含利用此弱點的網頁。 此外，受侵害的網站以及接受或存放使用者提供之內容或廣告的網站裡，也可能包含蓄意製作以利用本弱點的內容。 但是，攻擊者無法強迫使用者造訪網站， 而是引誘使用者自行前往。一般的做法是設法讓使用者按一下電子郵件或 Instant Messenger 訊息中通往攻擊者網站的連結。

**因為這個弱點而承受風險的主要系統有哪些？**  
可透過 IIS 存取 Indexing Service 的系統，會有因為這項弱點而遭受來自網際網路攻擊的風險。 如果未啟用 Indexing Service，則系統不受此問題影響。

用戶端系統可能會允許攻擊者代替使用者來執行用戶端指令碼。 指令碼可能會偽造內容、洩漏資訊、在受影響的網站上採取使用者可能會採取的行為。

**更新的作用何在？**  
本更新程式會修改 Indexing Service 驗證查詢參數的方法，進而消除此項弱點。

**當安全性公告發行時，這項弱點是否已揭發出來？**  
否。 Microsoft 是經由可靠的來源接獲有關這項弱點的訊息。

**當本安全性公告發行時，Microsoft 是否已接獲任何消息，指出這項弱點已遭有心人士惡用？**  
否。 當本安全性公告初次發行時，Microsoft 並未接到任何有關本弱點已成為公開攻擊媒介的消息，也沒有發現任何以此概念發展的程式碼公開範例。
