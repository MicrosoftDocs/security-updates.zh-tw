弱點詳細資料
------------

<span></span>
#### Client Service for NetWare 記憶體損毀弱點 - CVE-2006-4688：

在 Client Services for NetWare (CSNW) 中有一個遠端執行程式碼弱點，可讓成功利用該弱點之攻擊者取得受影響系統的完整控制權。

#### Client Service for NetWare 記憶體損毀弱點 - CVE-2006-4688 的緩和因素：

-   Windows XP Home Edition 不受此問題的影響。 Windows XP Home Edition 並未含有受影響的元件。
-   在 Windows Server 2003 與 Windows Server 2003 Service Pack 1 中，攻擊者必須是擁有有效登入認證的授權使用者，才能順利攻擊受影響的系統。
-   對於需要受影響元件的客戶，最佳實作的防火牆和標準預設防火牆設定有助於防止網路受到來自企業外的攻擊。 最佳方式建議連線至網際網路的系統盡可能曝露最少數量的連接埠。
-   依照預設，Client Service for NetWare 並未安裝於任何受影響的作業系統版本上。 唯有安裝此項服務的客戶，才有可能受此問題的影響。

#### Client Service for NetWare 記憶體損毀弱點 - CVE-2006-4688 的因應措施：

Microsoft 已經測試過以下的因應措施。 雖然這些因應措施不能徹底解決弱點，但有助於封鎖已知的攻擊行為。 如果因應措施會降低功能，以下小節將會描述功能降低的情況。

**注意：**CSNW 常與網際網路封包交換 (IPX) 和循序封包交換 (SPX) 等通訊協定有關聯。 但是，CSNW 可以透過任何已安裝的通訊協定，而被利用。 因為 TCP/IP 是最廣為使用的通訊協定，因此下列的因應措施是以 TCP/IP 為根據。 若 IPX 和 SPX 等通訊協定也在使用中，您也應該針對這些通訊協定封鎖適當的連接埠。 如需更多關於 IPX 和 SPX 的資訊，請造訪此網站。

-   **如果不需要使用 Client Service for NetWare，請將其移除。**

    如果不再需要使用 Client Service for NetWare，請將其移除。 請依照下列步驟執行。

    **Windows XP**

    1.  開啟 \[網路連線\]。
    2.  在區域連線上按一下滑鼠右鍵，然後按一下 \[內容\]。
    3.  在 \[這個連線使用下列項目\] 清單中，按一下 \[Client Service for NetWare\]。
    4.  按一下 \[一般\] 索引標籤，然後按一下 \[解除安裝\]。
    5.  按照畫面上的指示，完成移除程序。

    **Windows 2000**

    1.  以系統管理員身分登入。
    2.  在 \[網路上的芳鄰\] 上按一下滑鼠右鍵。
    3.  按一下 \[內容\]。
    4.  按一下 \[區域連線\]。
    5.  反白 **CSNW** 並按一下 \[解除安裝\]。
    6.  在顯示的快顯對話方塊上，按一下 \[是\]。

    **因應措施的影響：** 有些組織仰賴此受影響的元件來執行重要的功能。 除非系統管理員充分瞭解移除受影響元件所造成的後果，否則不應移除此該元件。

#### Client Service for NetWare 記憶體損毀弱點 - CVE-2006-4688 的常見問題集：

**這個弱點的範圍為何？**
這是 Client Service for NetWare (CSNW) 中的遠端執行程式碼弱點。 成功利用此弱點的攻擊者可以從遠端取得受影響系統的完整控制權。 攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。 在 Windows 2000 Server 上，此項服務亦稱為 Gateway Service for NetWare。

**造成這個弱點的原因為何？**
Client Service for NetWare 中存在未檢查的緩衝區。

**什麼是 Client Service for NetWare？**
Client Service for NetWare (CSNW) 能讓用戶端存取 NetWare 檔案、列印與目錄服務。 Microsoft 和 Novell 均針對此功能，提供用戶端服務： 分別為 Microsoft Client Service for NetWare 與 Novell Client for Microsoft Windows XP。 此弱點會影響 Microsoft Client Service for NetWare。 如需有關 NetWare 存取的詳細資訊，請造訪下列網站。 在 Windows 2000 Server 上，此項服務亦稱為 Gateway Service for NetWare。

**攻擊者可能會利用這項弱點採取什麼行動？**
成功利用此弱點的攻擊者可以取得受影響系統的完整控制權。

**什麼人可以利用此弱點？**
能將蓄意製作的訊息傳遞給受影響系統的任何匿名使用者都可以嘗試利用此弱點。

**攻擊者如何利用這項弱點？**
攻擊者可以建立一系列蓄意製作的網路訊息，並將訊息傳送給受影響的系統，直接在網路上利用這項弱點。 訊息即可使受影響的系統執行程式碼。

**因為這個弱點而承受風險的主要系統有哪些？**
所有已安裝 Client Services for NetWare (亦稱為 Gateway Service for NetWare) 的系統，均易遭受此弱點的影響。 依照預設，此元件並未安裝於任何受影響的作業系統版本上。 唯有手動安裝此元件的客戶，才有可能受此問題的影響。

**是否可以透過網際網路利用這個弱點？**

**Software Update Services：**

Microsoft Software Update Services (SUS) 能讓系統管理員以迅速可靠的方式，針對 Windows 2000 和 Windows Server 2003 伺服器以及執行 Windows 2000 Professional 或 Windows XP Professional 的桌面系統，部署最新的重要更新程式及安全性更新程式。

若想瞭解如何透過 Software Update Services 部署安全性更新，請造訪 [Software Update Services 網站](http://go.microsoft.com/fwlink/?linkid=21133)。

**Windows Server Update Services：**

透過 Windows Server Update Services (WSUS)，系統管理員可迅速可靠地將 Windows 2000 作業系統及後續系統、Office XP 及後續版本、Exchange Server 2003 及 SQL Server 2000 等最新的重大更新與安全性更新部署到 Windows 2000 及後續作業系統中。

如需更多關於利用 Windows Server Update Services 部署安全性更新的資訊，請造訪 [Windows Server Update Services 網站](http://www.microsoft.com/taiwan/windowsserversystem/updateservices/evaluation/overview.mspx)。

**Systems Management Server：**

Microsoft Systems Management Server (SMS) 提供了深具彈性的企業解決方案，能夠對更新程式進行方便的管理。 利用 SMS，系統管理員能判斷有哪些 Windows 系統需要安全性更新，並控制更新程式在企業中的部署，同時將對使用者造成的干擾降到最低。 如需更多關於系統管理員如何使用 SMS 2003 部署安全性更新的資訊，請造訪 [SMS 2003 的安全性補充程式管理網站](http://www.microsoft.com/taiwan/smserver/evaluation/capabilities/patch.htm)。 SMS 2.0 使用者也可以利用 [SMS 軟體更新服務功能套件](http://www.microsoft.com/taiwan/smserver/downloads/20/featurepacks/suspack/default.htm)來協助部署安全性更新。 如需關於 SMS 的詳細資訊，請造訪 [SMS 網站](http://www.microsoft.com/taiwan/smserver/)。

**注意：**SMS 使用 Microsoft Baseline Security Analyzer、Microsoft Office Detection Tool 以及企業更新掃描工具，為安全性公告的更新偵測及部署作業提供相當廣泛的支援。 不過這些工具可能無法偵測部分的軟體更新。 在這些情況中，系統管理員可以利用 SMS 的清查功能，判斷特定系統所需要的更新程式。 如需這個程序的詳細資訊，請造訪這個[網站](http://go.microsoft.com/fwlink/?linkid=33341)。 某些安全性更新程式在電腦重新啟動之後，會需要系統管理員的權限。 系統管理員可以用 Elevated Rights Deployment Tool (隨 [SMS 2003 Administration Feature Pack](http://www.microsoft.com/taiwan/smserver/downloads/2003/adminpack.htm) 和 [SMS 管理功能套件](http://www.microsoft.com/taiwan/smserver/downloads/20/featurepacks/adminpack/default.htm) 提供) 來安裝這些更新。
