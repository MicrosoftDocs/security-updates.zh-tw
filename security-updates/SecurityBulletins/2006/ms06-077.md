Microsoft Security Bulletin MS06-077 - 重要
===========================================

遠端安裝服務中的弱點可能允許遠端執行程式碼 (926121)
---------------------------------------------------

發行: 2006年12月13日 | 更新: 2007年3月23日

**版本:** 1.1

**應該閱讀此文件的對象：** 使用 Microsoft Windows 的客戶

**最高的嚴重性等級：** 重要

**建議：** 客戶應該儘快套用此更新程式

**安全性更新取代資訊：** 無

**警告：**[Microsoft 知識庫文件編號 926121](http://support.microsoft.com/kb/926121) 詳細記錄了目前已知安裝此安全性更新可能會發生的問題。 該文件並說明了解決這些問題的建議解決方案。 如需更多資訊，請參閱 [Microsoft 知識庫文件編號 926121](http://support.microsoft.com/kb/926121)。

弱點詳細資料
------------

<span></span>
#### RIS 可寫入路徑弱點 - CVE-2006-5584：

當伺服器預設為允許匿名使用者覆寫現有作業系統檔案或上載蓄意製作的檔案時，遠端安裝服務可在此伺服器上啟用 TFTP 服務。 這樣可能讓攻擊者得以侵害 RIS 伺服器所提供的作業系統安裝。

#### RIS 可寫入路徑弱點 - CVE-2006-5584 的緩和因素：

-   攻擊者必須能存取遠端安裝伺服器的 TFTP，才能放置蓄意製作的檔案或程式。
-   依照預設，遠端安裝服務不會安裝到 Windows 2000 Service Pack 4 上。
-   對於需要受影響元件的客戶，最佳實作的防火牆和標準預設防火牆設定有助於防止網路受到來自企業外圍以外的攻擊。 最佳方式建議連線至網際網路的系統盡可能曝露最少數量的連接埠。

#### RIS 可寫入路徑弱點 - CVE-2006-5584 的因應措施：

Microsoft 已經測試過以下的因應措施。 雖然這些因應措施不能徹底解決弱點，但有助於封鎖已知的攻擊行為。 如果因應措施會降低功能，以下小節將會描述功能降低的情況。

-   **將 TFTP 服務設為唯讀。**

    您可以透過設定登錄中的值，以將 TFTP 服務設為唯讀。

    **注意：**不當使用「登錄編輯程式」可能會造成嚴重的問題，甚至可能需要重新安裝您的作業系統。 Microsoft 無法保證能夠順利解決因不當使用「登錄編輯程式」所造成的問題。 請自行承擔使用「登錄編輯程式」的風險。 如需有關如何編輯登錄的資訊，請檢視「登錄編輯程式」(Regedit.exe) 中的＜變更機碼及數值＞說明主題，或是 Regedit.exe 中的＜新增及刪除登錄中的資訊＞與＜編輯登錄資料＞說明主題。

    **注意：**我們建議您在編輯登錄之前先將之備份。

    若要避免 RIS 伺服器接受未授權的存取，請使用 Regedt.exe 增加下列機碼到登錄中。

    1.  按一下 \[開始\]，按一下 \[執行\]，鍵入 regedt32，再按一下 \[確定\]。
    2.  在「登錄編輯程式」中，建立下列登錄機碼：
    3.  HKEY\_LOCAL\_MACHINE\\SYSTEM\\CurrentControlSet\\Services\\TFTPD\\Parameters
    4.  新增 REG\_SZ 值： **Masters**。 將值設定為 **0**。這個值將停用可寫入的 TFTPD 存取權限。
    5.  您必須重新啟動 TFTP 服務讓這個變更生效
    6.  TP 服務：
    7.  按一下 \[開始\]，按一下 \[執行\]，鍵入 cmd，再按一下 \[確定\]。
    8.  在 cmd 視窗中，鍵入 Net stop TFTPD，然後按 ENTER 鍵。

-   **在防火牆封鎖通訊埠 69 (UDP)。**

    本通訊埠用來起始與受影響元件的連線。 在防火牆封鎖 UDP 連接埠 69 有助於保護防火牆後方的系統免遭此弱點的攻擊。 建議您封鎖所有網際網路中來路不明的輸入通訊，以防他人利用其他連接埠攻擊系統。 如需更多關於連接埠的資訊，請造訪這個[網站](http://go.microsoft.com/fwlink/?linkid=21312)。

-   **不需使用 TFTP 服務時，將之關閉**

    1.  按一下 \[開始\]，按一下 \[執行\]，鍵入 cmd，再按一下 \[確定\]。
    2.  在 cmd 視窗中，鍵入 Net stop TFTPD，然後按 ENTER 鍵。

    **因應措施的影響：** 如果您停用 TFTP 服務，使用者將無法上載、檢視或寫入 TFTP 目錄的內容。

-   為了防範利用此弱點的網路攻擊，請啟用進階 TCP/IP 篩選功能 (如果系統支援此功能)。

    您可以啟用進階的 TCP/IP 篩選功能，封鎖所有來路不明的對內流量。 如需更多關於如何設定 TCP/IP 篩選功能的資訊，請參閱 [Microsoft 知識庫文件編號 309798](http://support.microsoft.com/kb/309798)。

-   為了防範利用此弱點的網路攻擊，請在受影響的系統上使用 IPSec 封鎖受影響的連接埠。

    使用網際網路通訊協定安全性 (IPSec) 保護網路通訊。 如需更多關於 IPSec 及如何套用篩選的資訊，請參閱 [Microsoft 知識庫文件編號 313190](http://support.microsoft.com/kb/313190) 與 [Microsoft 知識庫文件編號 813878](http://support.microsoft.com/kb/813878)。

#### RIS 可寫入路徑弱點 - CVE-2006-5584 的常見問題集：

**這個弱點的範圍為何？**  
這是遠端執行程式碼的弱點。 成功利用此弱點的攻擊者可以從遠端取得受影響系統的完整控制權。 攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。

**造成這個弱點的原因為何？**  
這個弱點是由於系統允許透過 RIS TFTP 服務，匿名存取主控作業系統建置的檔案結構所造成。

**什麼是 RIS？**  
遠端安裝服務 (RIS) 是一個以開機前執行環境 (Pre-boot Execution Environment；PXE) 為基礎的部署技術，可讓 Windows 安裝程式在網路上啟動。

**什麼人可以利用此弱點？**  
可傳遞蓄意製作的可執行檔或檔案給受影響的 RIS 系統的任何匿名使用者，日後均可嘗試在使用遭竄改的作業系統所建立的系統上利用這個弱點。

**因為這個弱點而承受風險的主要系統有哪些？**  
已安裝 RIS 的所有 Windows 2000 Server Service Pack 4 最可能受這個弱點影響。

**是否可以透過網際網路利用這個弱點？**  
是。 攻擊者可以嘗試透過網際網路利用這個弱點，但是他們必須先取得網路權限，才能存取 RIS 服務。 最佳實作的防火牆和標準預設防火牆設定有助於防止來自網際網路外界的攻擊。

**更新的作用何在？**  
此更新會禁止匿名 TFTP 使用者寫入 RIS 主控作業系統建置的檔案結構，藉此移除這個弱點。 此更新在本公告的＜因應措施＞部分中新增登錄機碼資訊。

**當安全性公告發行時，這項弱點是否已揭發出來？**  
否。 Microsoft 是經由可靠的來源接獲有關這項弱點的訊息。 當本安全性公告初次發行時，Microsoft 尚未接獲任何有關此弱點已被發佈出來的消息。

**當本安全性公告發行時，Microsoft 是否已接獲任何消息，指出這項弱點已遭有心人士利用？**  
否。 當本安全性公告初次發行時，Microsoft 並未接到任何有關本弱點已成為公開攻擊媒介的消息，也沒有發現任何以此概念發展的程式碼公開範例。
