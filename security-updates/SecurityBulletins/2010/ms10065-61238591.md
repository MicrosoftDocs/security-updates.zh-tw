Microsoft Security Bulletin MS10-065 - 重要
===========================================

Microsoft Internet Information Services (IIS) 中的資訊安全風險可能會允許遠端執行程式碼 (2267960)
------------------------------------------------------------------------------------------------

發行: 2010年9月15日

**版本:** 1.0

### 一般資訊

#### 提要

這個資訊安全更新可解決 Internet Information Services (IIS) 中兩項未公開報告的資訊安全風險，以及一項公開揭露的資訊安全風險。 若用戶端將蓄意製作的 HTTP 要求傳送至伺服器，則其中最嚴重的資訊安全風險可能會允許遠端執行程式碼。 成功利用此資訊安全風險的攻擊者可以取得受影響系統的完整控制權。

對於 IIS 5.1、IIS 6.0、IIS 7.0 與 IIS 7.5，此資訊安全更新的等級為「重要」。如需更多資訊，請參閱本節中的＜受影響及不受影響的軟體＞小節。

此資訊安全更新可修改 IIS 處理蓄意製作之 HTTP 要求的方式，以解決資訊安全風險。 如需更多有關資訊安全風險的資訊，請參閱下節＜資訊安全風險資訊＞下的特定資訊安全風險項目的＜常見問題集 (FAQ)＞小節。

**建議。** 大部分客戶都已啟用自動更新，並且不必須採取任何行動，因為資訊安全更新將自動下載和安裝。 沒有啟用自動更新的客戶則必須檢查更新，並手動安裝更新。 如需有關自動更新中特定設定選項的資訊，請參閱 [Microsoft 知識庫文件編號 294871](http://support.microsoft.com/kb/294871/zh-tw)。

若是系統管理員和企業安裝，或是想要手動安裝此資訊安全更新的使用者，Microsoft 建議客戶使用更新管理軟體，立即套用更新，或使用 [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) 服務檢查更新。

另請參閱本公告下文的＜偵測與部署工具及指南＞章節。

**已知問題。** [Microsoft 知識庫文件編號 2267960](http://support.microsoft.com/kb/2267960/zh-tw) 中記載目前已知安裝此資訊安全更新後，可能會發生的問題。 該文件並說明了解決這些問題的建議解決方案。 目前已知問題和建議解決方案僅適用於此軟體的特定版本，本文會提供詳細文章的連結。

#### 受影響及不受影響的軟體

下列軟體已經過測試判斷哪些版號或版本會受到影響。 其他版本超過它們的支援週期或不受影響。 若要瞭解您的軟體版本的支援週期，請造訪 [Microsoft 產品技術支援週期網站](http://go.microsoft.com/fwlink/?linkid=21742)。

與本資訊安全更新相關的常見問題集 (FAQ)
--------------------------------------

**為什麼此公告有三個各自獨立的更新？**  
此公告中所述各資訊安全風險的受影響產品有差異，因此有三個各自獨立的更新。 請參閱「受影響的軟體」表，以判斷哪個更新適用於您的 Web 伺服器。

**何處可找到檔案資訊詳細資料？**  
關於檔案資訊詳細資料的位置，請見＜資訊安全更新部署＞一節中的參考表。

**為何本次更新中仍然包含數個已發佈的安全性資訊安全風險？**  
因為解決這些資訊安全風險所需的修正作業位於相關檔案中，所以在本次更新中仍然提供對這些資訊安全風險的支援。 客戶不需要安裝數個幾乎相同的更新，只需要安裝本更新即可。

**我所使用的軟體是這個資訊安全公告中討論的軟體之舊版。 該怎麼辦？**  
本公告所列出的受影響軟體版本已經過測試判斷哪些版本會受到影響。 其他版本超出它們的支援週期。 如需瞭解產品生命週期的相關資訊，請造訪 [Microsoft 產品技術支援週期](http://go.microsoft.com/fwlink/?linkid=21742)網站。

使用此軟體舊版的客戶應優先考慮移轉至支援的版本，以避免因潛在的資訊安全風險而遭受攻擊。 若要瞭解您的軟體版本的支援週期，請參閱[選擇一個產品檢視其支援週期資訊](http://go.microsoft.com/fwlink/?linkid=169555)。 如需更多軟體版本 Service Pack 的相關資訊，請參閱[產品技術支援週期所支援的 Service Packs](http://go.microsoft.com/fwlink/?linkid=89213)。

IIS 重複參數要求拒絕服務資訊安全風險 - CVE-2010-1899
----------------------------------------------------

Internet Information Services (IIS) 中存在一項拒絕服務資訊安全風險，可能會允許成功利用此資訊安全風險的攻擊者中斷服務，造成伺服器沒有回應。 攻擊者會傳送蓄意製作的 URL 要求至 IIS 所主控之網站上的作用中伺服器頁面，以利用此該資訊安全風險。

若要在「一般性資訊安全風險」清單標準條目中檢視此資訊安全風險，請參閱 [CVE-2010-1899](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-1899) (英文)。

#### IIS 重複參數要求拒絕服務資訊安全風險 - CVE-2010-1899 的緩和因素

緩和因素是指存在於預設狀態中的設定、共用設定或一般最佳作法，可能會減少資訊安全風險影響的嚴重性。 下列緩和因素可能對您的狀況有所助益：

-   ASP 網頁在 IIS 6.0 上預設為禁止。
-   資訊安全風險僅在 ASP 指令碼從回應的要求寫入參數時遭到利用。

#### IIS 重複參數要求拒絕服務資訊安全風險 - CVE-2010-1899 的因應措施

因應措施指的是無法徹底修正資訊安全風險，但有助於在套用更新之前封鎖已知攻擊模式的設定變更。 Microsoft 測試了下列因應措施和狀態，討論因應措施是否會降低功能：

-   **暫時在 IIS 伺服器上停用 ASP**

    在 \[IIS 管理員\] 中，將 ASP 的狀態從 \[允許\] 變更為 \[禁止\]。

    適用於 Windows XP 及 Windows Server 2003

    1.  執行 inetmgr.exe 以啟動 IIS 管理員。
    2.  在左窗格的伺服器名稱下，選取 \[網頁服務延伸\] 節點。
    3.  在右窗格的 \[動態伺服器網頁\] 上按一下滑鼠右鍵，然後選取 \[禁止\]。

    適用於 Windows Vista、Windows Server 2008、Windows 7 和 Windows Server 2008 R2

    1.  執行 inetmgr.exe 以啟動 IIS 管理員。
    2.  在左窗格中選取伺服器節點。
    3.  在右窗格中選取 \[處理常式對應\]，然後按兩下 \[AspClassic\]。
    4.  按一下 \[要求限制\]，選取 \[存取\] 索引標籤，然後按一下 \[無\]。
    5.  按一下 \[確定\]，然後按一下 \[否\]，不允許 ISAPI 延伸。

    **因應措施的影響。** ASP 網頁將無法再運作。

    **如何復原因應措施：**

    在 \[IIS 管理員\] 中，將 ASP 的狀態從 \[禁止\] 變更為 \[允許\]。

    適用於 Windows XP 及 Windows Server 2003

    1.  執行 inetmgr.exe 以啟動 IIS 管理員。
    2.  在左窗格的伺服器名稱下，選取 \[網頁服務延伸\] 節點。
    3.  在右窗格的 \[動態伺服器網頁\] 上按一下滑鼠右鍵，然後選取 \[允許\]。

    適用於 Windows Vista、Windows Server 2008、Windows 7 和 Windows Server 2008 R2

    1.  執行 inetmgr.exe 以啟動 IIS 管理員。
    2.  在左窗格中選取伺服器節點。
    3.  在右窗格中選取 \[處理常式對應\]，然後按兩下 \[AspClassic\]。
    4.  按一下 \[要求限制\]，選取 \[存取\] 索引標籤，然後按一下 \[指令碼\]。
    5.  按一下 \[確定\]，然後按一下 \[否\]，不允許 ISAPI 延伸。

#### IIS 重複參數要求拒絕服務資訊安全風險 - CVE-2010-1899 的常見問題集

**這個資訊安全風險的範圍為何？**  
這是一項拒絕服務的資訊安全風險。 若攻擊者傳送蓄意製作的要求到 Web 伺服器，該伺服器可能會變得沒有回應。 資訊安全風險不會造成伺服器失敗或導致任何資料遺失，而伺服器最終會隨著服務重設而恢復正常運作。

**造成這個資訊安全風險的原因為何？**  
此拒絕服務資訊安全風險是由 ASP 指令碼處理程式碼中的堆疊溢位所造成。

**攻擊者可能會利用這項資訊安全風險採取什麼行動？**  
這是一項拒絕服務的資訊安全風險。 成功利用此資訊安全風險的攻擊者將使受影響系統停止回應並自動重新啟動。 在這段期間，伺服器將無法回應要求。 若攻擊者使服務重新啟動五次，則必須有系統管理員互動，服務才能重新啟動。 請注意，拒絕服務資訊安全風險不會允許攻擊者執行程式碼或提高其使用者權限，但會導致受影響系統停止接受要求。

**攻擊者如何利用這項資訊安全風險？**  
攻擊者可能會傳送蓄意製作的 URL 要求到主控 .asp 頁面的網站，以利用此拒絕服務資訊安全風險。

**因為這個資訊安全風險而承受風險的主要系統有哪些？**  
執行 Microsoft Internet Information Services 的所有系統都可能受此資訊安全風險影響。

**更新的作用何在？**  
此更新會修改 IIS 和 asp.dll 處理蓄意製作之 HTTP 要求的方式。

**當資訊安全公告發行時，這項資訊安全風險是否已揭發出來？**  
否。 Microsoft 是經由協同合作的來源接獲有關這項資訊安全風險的訊息。

**當本資訊安全公告發行時，Microsoft 是否已接獲任何消息，指出這項資訊安全風險已遭有心人士利用？**  
否。 當本資訊安全公告初次發行時，Microsoft 尚未接到任何有關本資訊安全風險已公開用來攻擊客戶的消息。

要求標頭緩衝區溢位資訊安全風險 - CVE-2010-2730
----------------------------------------------

Internet Information Services (IIS) 中存在遠端執行程式碼資訊安全風險，攻擊者可能會傳送蓄意製作的 HTTP 要求至已啟用 FastCGI 的 IIS 伺服器，以利用該資訊安全風險。

若要在「一般性資訊安全風險」清單標準條目中檢視此資訊安全風險，請參閱 [CVE-2010-2730](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-2730) (英文)。

#### 要求標頭緩衝區溢位資訊安全風險 - CVE-2010-2730 的緩和因素

緩和因素是指存在於預設狀態中的設定、共用設定或一般最佳作法，可能會減少資訊安全風險影響的嚴重性。 下列緩和因素可能對您的狀況有所助益：

-   在 IIS 中，依預設未啟用 FastCGI。

#### 要求標頭緩衝區溢位資訊安全風險 - CVE-2010-2730 的因應措施

因應措施指的是無法徹底修正資訊安全風險，但有助於在套用更新之前封鎖已知攻擊模式的設定變更。 Microsoft 測試了下列因應措施和狀態，討論因應措施是否會降低功能：

-   停用 FastCGI

    1.  啟動伺服器管理員並編輯 \[角色\]。
    2.  取消選取 CGI 核取方塊，並按一下 \[下一步\]。

    **因應措施的影響。** 依附於 FastCGI 的模組將無法再運作。

    **如何復原因應措施：**

    啟動伺服器管理員，並在 \[Web 伺服器\] 下新增 \[CGI\] 功能。

-   安裝 URL Rewrite 模組

    安裝此模組後，無論您是否有使用模組，都會受到保護。 如需 URL Rewrite 的相關資訊，請參閱 [URL Rewrite](http://www.iis.net/download/urlrewrite%20(英文))。

#### 要求標頭緩衝區溢位資訊安全風險 - CVE-2010-2730 的常見問題集

**這個資訊安全風險的範圍為何？**  
這是遠端執行程式碼的資訊安全風險。 成功利用此資訊安全風險的遠端攻擊者，便可能會以與工作者處理序識別 (WPI) 相同的權限，在 IIS 伺服器上執行動作。 此識別在 Windows 7 與 Windows Server 2008 R2 上將設定為虛擬應用程式集區識別。

**造成這個資訊安全風險的原因為何？**  
此資訊安全風險是由已啟用 FastCGI 之 IIS 伺服器處理要求標頭的方式所造成。

**什麼是 FastCGI？**  
FastCGI for IIS 可讓使用熱門的應用程式架構，以高效能和可靠的方式支援 IIS Web 伺服器主控的 FastCGI 通訊協定。 FastCGI 是通用閘道介面 (CGI) 的高效能替代方法，通用閘道介面是外部應用程式與 Web 伺服器間的標準介面，從第一版開始即為 IIS 功能集支援的一部分。

**攻擊者可能會利用這項資訊安全風險採取什麼行動？**  
成功利用此資訊安全風險的攻擊者可以取得受影響系統的完整控制權。 攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。

**攻擊者如何利用這項資訊安全風險？**  
IIS 對於送至已啟用 FastCGI 之 IIS 伺服器的要求標頭的處理方式中，存在遠端執行程式碼資訊安全風險。 攻擊者可建立蓄意製作的 HTTP 要求來利用此資訊安全風險。

**因為這個資訊安全風險而承受風險的主要系統有哪些？**  
主控內容並已啟用 FastCGI 的 Web 伺服器。

**更新的作用何在？**  
此更新會修改已啟用 FastCGI 之 IIS 伺服器處理要求標頭的方式。

**當資訊安全公告發行時，這項資訊安全風險是否已揭發出來？**  
否。 Microsoft 是經由協同合作的來源接獲有關這項資訊安全風險的訊息。

**當本資訊安全公告發行時，Microsoft 是否已接獲任何消息，指出這項資訊安全風險已遭有心人士利用？**  
否。 當本資訊安全公告初次發行時，Microsoft 尚未接到任何有關本資訊安全風險已公開用來攻擊客戶的消息。

目錄驗證略過資訊安全風險 - CVE-2010-2731
----------------------------------------

Internet Information Services (IIS) 中存在權限提高資訊安全風險。 成功利用此資訊安全風險的攻擊者不必經過驗證，就能存取限制的資源。

若要在「一般性資訊安全風險」清單標準條目中檢視此資訊安全風險，請參閱 [CVE-2010-2731](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-2731) (英文)。

#### 目錄驗證略過資訊安全風險 - CVE-2010-2731 的緩和因素

緩和因素是指存在於預設狀態中的設定、共用設定或一般最佳作法，可能會減少資訊安全風險影響的嚴重性。 下列緩和因素可能對您的狀況有所助益：

-   不使用驗證的網站不易受此資訊安全風險影響。 僅使用匿名驗證的網站不受權限提高攻擊模式的影響。

#### 目錄驗證略過資訊安全風險 - CVE-2010-2731 的因應措施

因應措施指的是無法徹底修正資訊安全風險，但有助於在套用更新之前封鎖已知攻擊模式的設定變更。 Microsoft 測試了下列因應措施和狀態，討論因應措施是否會降低功能：

-   **安裝並使用 URLScan**

    URLScan 有助於防範因 URL 要求形式不當而引發大量問題，包括本公告涵蓋的公開描述問題在內。 URLScan 無法如緩和程式碼模組或以下的 global.asax 指令碼那般全面保護您的系統。 如需 URLScan 的相關資訊，請參閱 TechNet 文章 [UrlScan 安全性工具](http://www.microsoft.com/technet/security/tools/urlscan.mspx) (英文)。

#### 目錄驗證略過資訊安全風險 - CVE-2010-2731 的常見問題集

**這個資訊安全風險的範圍為何？**  
這是權限提高的資訊安全風險。 攻擊者可能會傳送蓄意製作的要求到需要驗證的 URL，以略過驗證並針對攻擊者不應存取的內容執行 ASP 指令碼。

**造成這個資訊安全風險的原因為何？**  
此資訊安全風險是由 IIS 剖析蓄意製作之 URL 的方式所造成。

**SSL 有助於緩和此資訊安全風險？**  
這是一個目錄略過資訊安全風險。 安全通訊端層 (SSL) 無法協助緩和此資訊安全風險。

**攻擊者可能會利用這項資訊安全風險採取什麼行動？**  
攻擊者可能會利用此資訊安全風險略過目錄型基本驗證，以及存取並執行應要求驗證的資源。

**攻擊者如何利用這項資訊安全風險？**  
攻擊者可能會傳送蓄意製作的 URL，以略過目錄型基本驗證。

**因為這個資訊安全風險而承受風險的主要系統有哪些？**  
使用基本驗證的 Web 服務可能因為這個資訊安全風險而承受風險。

**更新的作用何在？**  
此更新會修改 IIS 處理蓄意製作之要求的方式，以防止未驗證的使用者存取 IIS 伺服器所主控應要求驗證的資源。

**當資訊安全公告發行時，這項資訊安全風險是否已揭發出來？**  
是。 這項資訊安全風險已經遭到公開揭發。 此資訊安全風險已被指派「一般性資訊安全風險」編號 [CVE-2010-2731](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-2731)。

**當本資訊安全公告發行時，Microsoft 是否已接獲任何消息，指出這項資訊安全風險已遭有心人士利用？**  
否。 當本資訊安全公告初次發行時，Microsoft 尚未接到任何有關本資訊安全風險已公開用來攻擊客戶的消息。

    由於 Microsoft Windows 的版本眾多，您電腦上實際執行的步驟可能會與此處描述的不同。 如遇到不同的狀況，請參閱產品的說明文件以完成這些步驟。

    1.  按一下 \[開始\]，然後按一下 \[搜尋\]。
    2.  在 \[搜尋結果\] 窗格中，在 \[搜尋小幫手\] 下按一下 \[所有檔案和資料夾\]。
    3.  在 \[部份或完整的檔案名稱\] 方塊中，輸入適當檔案資訊表中的檔案名稱，再按一下 \[搜尋\]。

    5.  在 \[版本\] 索引標籤上，比較檔案版本與適當檔案資訊表中記錄的版本，判斷您系統上安裝的檔案版本。

**確認已套用資訊安全更新**

-   **Microsoft Baseline Security Analyzer**

    您可以使用 Microsoft Baseline Security Analyzer (MBSA) 工具確認受影響系統上是否已經套用資訊安全更新程式。 如需詳細資訊，請參閱本公告前段的＜偵測與部署工具及指南＞一節。

-   **檔案版本驗證**

    由於 Microsoft Windows 的版本眾多，您電腦上實際執行的步驟可能會與此處描述的不同。 如遇到不同的狀況，請參閱產品的說明文件以完成這些步驟。

    1.  按一下 \[開始\]，然後按一下 \[搜尋\]。
    2.  在 \[搜尋結果\] 窗格中，在 \[搜尋小幫手\] 下按一下 \[所有檔案和資料夾\]。
    3.  在 \[部份或完整的檔案名稱\] 方塊中，輸入適當檔案資訊表中的檔案名稱，再按一下 \[搜尋\]。
    4.  在檔案清單中，用滑鼠右鍵按一下適當檔案資訊表中的某個檔案名稱，再按 \[內容\]。
        **注意：**視系統所安裝的作業系統或程式的版本而定，檔案資訊表中列出的檔案未必會全部安裝。
    5.  在 \[版本\] 索引標籤上，比較檔案版本與適當檔案資訊表中記錄的版本，判斷您系統上安裝的檔案版本。
        **注意：**在安裝時，檔案版本以外的屬性可能會變更。 若要驗證是否成功套用更新程式，利用檔案資訊表比對其他檔案屬性並不是妥當的做法。 此外，在某些情況下，檔案的名稱在安裝時可能會有所變更。 如果缺少檔案或版本資訊，請採用其他可用的方法來驗證更新程式的安裝情形。

-   **登錄機碼驗證**

    您也可以查看本節「參考表」中所列的登錄機碼，確認此資訊安全更新所安裝的檔案。

    這些登錄機碼可能未包含完整的安裝檔案清單。 而且，當系統管理員或 OEM 將資訊安全更新整合或匯集到 Windows 安裝原始檔時，可能無法正確建立這些登錄機碼。
