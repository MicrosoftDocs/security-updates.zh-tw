Security Bulletin

Microsoft Security Bulletin MS10-024 - 重要
===========================================

Microsoft Exchange 和 Windows SMTP 服務中的弱點可能會允許拒絕服務 (981832)
--------------------------------------------------------------------------

發行: 2010年4月14日 | 更新: 2010年7月14日

**版本:** 2.0

### 一般資訊

#### 提要

此資訊安全更新可解決 Microsoft Exchange 和 Windows SMTP 服務中一項公開揭露的弱點和一項未公開報告的弱點。 如果攻擊者將蓄意製作的 DNS 回應傳送至執行 SMTP 服務的電腦，這些弱點中較嚴重者可能會允許拒絕服務。 Windows Server 2003、Windows Server 2003 x64 Edition 或 Windows XP Professional x64 Edition 預設不會安裝 SMTP 元件。

對於所有受支援版本的 Microsoft Windows 2000、Windows XP 和 Windows Server 2003；32 位元和 x64 版本的 Windows Server 2008；適用於 x64 型系統的 Windows Server 2008 R2；以及 Microsoft Exchange Server 2003，此資訊安全更新的等級為「重要」。對於 Microsoft Exchange Server 2000，此資訊安全更新的等級為「中度」。如需詳細資訊，請參閱本節中的＜受影響及不受影響的軟體＞小節。

此資訊安全更新會修正 SMTP 剖析 MX 記錄的方式，以及 SMTP 解譯 SMTP 命令回應時的記憶體配置方式，進而解決這些弱點。 如需更多有關弱點的資訊，請參閱下節＜弱點資訊＞下的特定弱點項目的＜常見問題集 (FAQ)＞小節。

**建議。** 大部分客戶都已啟用自動更新，並且不必須採取任何行動，因為資訊安全更新將自動下載和安裝。 沒有啟用自動更新的客戶則必須檢查更新，並手動安裝更新。 如需有關自動更新中特定設定選項的資訊，請參閱 [Microsoft 知識庫文件編號 294871](https://support.microsoft.com/kb/294871/zh-tw)。

若是系統管理員和企業安裝，或是想要手動安裝此資訊安全更新的使用者，Microsoft 建議客戶使用更新管理軟體，立即套用更新，或使用 [Microsoft Update](https://go.microsoft.com/fwlink/?linkid=40747) 服務檢查更新。

另請參閱本公告下文的＜偵測與部署工具及指南＞章節。

**已知問題。** [Microsoft 知識庫文件編號 981832](https://support.microsoft.com/kb/981832/zh-tw) 中記載了客戶安裝此資訊安全更新時，可能會發生的目前已知問題。 該文件並說明了解決這些問題的建議解決方案。

#### 受影響及不受影響的軟體

下列軟體已經過測試判斷哪些版號或版本會受到影響。 其他版本超過它們的支援週期或不受影響。 若要瞭解您的軟體版本的支援週期，請造訪 [Microsoft 產品技術支援週期網站](https://go.microsoft.com/fwlink/?linkid=21742)。

與本資訊安全更新相關的常見問題集 (FAQ)
--------------------------------------

<span></span>
**為何於 2010 年 7 月 13 日重新發行此公告？**  
Microsoft 重新發行此公告是為了重新提供 Windows Server 2008 和 Windows Server 2008 R2 的更新，以解決 KB976323 列出的一項已知問題；若在已安裝 Internet Information Services (IIS) 的 Windows Server 2008 系統上安裝此資訊安全更新，則 SMTP 設定選項會重新設定。 您安裝此更新前所設定的 SMTP 設定選項均會遺失。 在您安裝此資訊安全更新後，若需任何 SMTP 設定選項，必須手動重新套用。

本次重新提供只會影響到那些與 KB976323 相關的套件。已成功更新系統的客戶不需要重新安裝此更新。。

**為何於 2010 年 4 月 15 日修訂此資訊安全公告？**  
Microsoft 修訂此資訊安全公告的目的，是為了宣布一項針對 Microsoft Exchange Server 2000 Service Pack 3 所做的偵測變更。此變更僅屬偵測變更。 經過更正的偵測可為先前未收到此資訊安全更新的受影響系統提供資訊安全更新。 本公告中資訊安全更新檔案沒有變更。 在這些系統上已成功安裝 KB976703 更新的客戶不需重新安裝更新程式。

**何處可找到檔案資訊詳細資料？**  
關於檔案資訊詳細資料的位置，請見＜資訊安全更新部署＞一節中的參考表。

**Microsoft Exchange Server 2007 和 Microsoft Exchange Server 2010 的更新，與 Microsoft Exchange Server 2000 和 Microsoft Exchange Server 2003 的更新差異何在？**  
Microsoft Exchange 2007 發行之後，Microsoft Exchange 也逐漸朝向新的服務模型邁進。 為更深入瞭解 Microsoft Exchange 服務模型，請參閱 TechNet 文章 [Exchange 2007 服務](https://go.microsoft.com/fwlink/?linkid=89577)。 關於新的 Microsoft Exchange 服務模型如有疑問，請連絡 [Microsoft 說明與支援](https://support.microsoft.com/)。

**為何本次更新中仍然包含數個已回報的安全性弱點？**  
因為解決這些弱點所需的修正作業位於相關檔案中，所以在本次更新中仍然提供對這些弱點的支援。 客戶不需要安裝數個幾乎相同的更新，只需要安裝本更新即可。

**此更新程式是否包含功能上的任何安全性相關變更？**  
是。 除了此公告的＜弱點資訊＞一節中列出的變更外，此更新程式還包括對 Microsoft Exchange Server 2000 Service Pack 3 所做的一個深度防禦變更，可用來解決使用 Outlook Web Access (OWA) 時中文字元集問題。 此問題只會影響 Exchange 2000 系統。 請注意，Microsoft Exchange 2000 Server Service Pack 3 的更新需要 Exchange 2000 Server Post-Service Pack 3 (SP3) 更新彙總套件，此為必要條件。 安裝本資訊安全公告所提供的深度防禦更新之前，必須先安裝 Exchange 2000 更新彙總套件 (KB870540)。 此更新程式可偵測出更新彙總套件是否已經安裝。 如果尚未安裝更新彙總套件，您將會被導向下載網站。 如需詳細資訊，請參閱 [Microsoft 知識庫文件編號 870540](https://support.microsoft.com/kb/870540)。若要下載必要的更新程式，請參閱 [Exchange 2000 更新彙總套件 (KB870540)](https://www.microsoft.com/download/details.aspx?familyid=363a57a4-8bed-4bbb-bbe4-abc11ab04611)。

這個更新也包括 Microsoft Exchange 2007 和Microsoft Exchange 2010 的一項深度防禦變更，可將額外的來源連接埠 Entropy 加到 SMTP 服務所啟動的 DNS 交易。

**我是否需要套用 Windows 和 Exchange 二者的更新？**  
如果系統有安裝 Microsoft Exchange，Exchange 和 Windows 二者的更新都必須套用。 如果已啟用 SMTP 服務但並未執行 Exchange 服務，則只需要套用 Windows 更新。

**我用的是 Exchange 2007 或 Exchange 2010。如果它們不受本公告所描述的弱點影響，為什麼要提供更新給我？**  
Microsoft Exchange 2007 和 Microsoft Exchange 2010 的更新只包含深度防禦變更，可將額外的來源連接埠 Entropy 加到 SMTP 服務所啟動的 DNS 交易。

**我是否必須按特定順序安裝 Exchange Server 2007 伺服器的更新彙總套件？**  
請參考 TechNet 文章：[如何安裝 Exchange 2007 的最新 Service Pack 或更新彙總套件](https://technet.microsoft.com/zh-tw/library/ee221147(exchg.80).aspx)。

**我所使用的軟體是這個資訊安全公告中討論的軟體之舊版。 該怎麼辦？**  
本公告所列出的受影響軟體版本已經過測試判斷哪些版本會受到影響。 其他版本超出它們的支援週期。 如需瞭解產品生命週期的相關資訊，請造訪 [Microsoft 產品技術支援週期](https://go.microsoft.com/fwlink/?linkid=21742)網站。

使用此軟體舊版的客戶應優先考慮移轉至支援的版本，以避免因潛在的弱點而遭受攻擊。 若要瞭解您的軟體版本的支援週期，請參閱[選擇一個產品檢視其支援週期資訊](https://go.microsoft.com/fwlink/?linkid=169555)。 如需更多軟體版本 Service Pack 的相關資訊，請參閱[產品技術支援週期所支援的 Service Packs](https://go.microsoft.com/fwlink/?linkid=89213)。

SMTP Server MX 紀錄弱點 - CVE-2010-0024
---------------------------------------

<span></span>
Microsoft Windows 簡易郵件傳送通訊協定 (SMTP) 元件處理蓄意製作之 DNS 郵件交換程式 (MX) 資源記錄的方式中，存在一項拒絕服務弱點。 攻擊者無需經過驗證即可利用這項弱點，因此可藉由傳送蓄意製作的網路訊息至執行 SMTP 服務的電腦來利用此弱點。 成功利用此弱點的攻擊者可使 SMTP 服務停止回應，直到重新啟動為止。

若要在「一般性弱點」清單標準條目中檢視此弱點，請參閱 [CVE-2010-0024](https://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0024) (英文)。

#### SMTP Server MX 記錄弱點 - CVE-2010-0024 的緩和因素

緩和因素是指存在於預設狀態中的設定、共用設定或一般最佳作法，可能會減少弱點影響的嚴重性。 下列緩和因素可能對您的狀況有所助益：

-   Windows Server 2003、Windows Server 2003 x64 Edition 或 Windows XP Professional x64 Edition 預設不會安裝 SMTP 元件。

#### SMTP Server MX 記錄弱點 - CVE-2010-0024 的因應措施

Microsoft 尚未找到此項弱點的任何因應措施。

#### SMTP Server MX 記錄弱點 - CVE-2010-0024 的常見問題集

**這個弱點的範圍為何？**  
這是一項拒絕服務的弱點。 成功利用此弱點的攻擊者可使受影響的 SMTP 服務停止回應，直到從佇列中移除蓄意製作的訊息，並手動重新啟動服務為止。 請注意拒絕服務弱點不會讓攻擊者執行程式碼或提高其使用者權限，但會導致受影響系統停止接受要求。

**造成這個弱點的原因為何？**  
形成這個弱點的原因，是由於 Microsoft Windows 簡易郵件傳送通訊協定 (SMTP) 元件不正確地剖析蓄意製作的 DNS 郵件交換程式 (MX) 記錄所致。

**什麼是 SMTP 服務？**  
簡易郵件傳送通訊協定 (SMTP) 是隨電子郵件服務或 Internet Information Services (IIS) 安裝的一項服務，用途是傳送電子郵件。 SMTP 控制電子郵件經由網際網路傳遞到目的地伺服器的方式。 SMTP 服務會在伺服器之間傳送和接收電子郵件。 如需詳細資訊，請參閱 TechNet 文章：[Simple Mail Transfer Protocol (SMTP) 服務](https://technet.microsoft.com/zh-tw/library/cc759281(ws.10).aspx)。

**什麼是 MX 資源記錄？**  
郵件交換 (MX) 資源記錄會指定 DNS 網域名稱的郵件交換伺服器。 郵件交換伺服器是用來處理和轉寄 DNS 網域名稱的郵件之主機。 「處理郵件」意指將郵件傳遞給收件者，或轉給不同的郵件傳輸類型。 「轉寄郵件」意指將郵件傳送到其最終的目的地伺服器、使用簡易郵件傳送通訊協定 (SMTP) 將郵件傳送到較接近最終目的地的另一部郵件交換伺服器，或排入指定時間長度的佇列。 如需更多關於 MX 資源記錄的資訊，請參閱 TechNet 文章：[資源記錄類型](https://technet.microsoft.com/zh-tw/library/cc958958.aspx) (英文)。

**攻擊者可能會利用這項弱點採取什麼行動？**  
成功利用此弱點的攻擊者可使 SMTP 服務停止回應，直到從佇列中移除蓄意製作的訊息，並手動重新啟動服務為止。

**攻擊者如何利用這項弱點？**  
攻擊者可建立惡意的 DNS 伺服器，將蓄意製作的回應傳回 MX 資源記錄查詢，來利用這項弱點。

**因為這個弱點而承受風險的主要系統有哪些？**  
所有採用系統隨附的原生 SMTP 元件的作業系統，都有遭受此弱點攻擊的風險。

**更新的作用何在？**  
此資訊安全更新可修正 SMTP 服務剖析 MX 記錄的方式，進而解決此弱點。

**當資訊安全公告發行時，這項弱點是否已被揭發出來？**  
是。 這項弱點已經遭到公開揭發。 這項弱點已被指派「一般性弱點」編號 [CVE-2010-0024](https://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0024)。

**當本資訊安全公告發行時，Microsoft 是否已接獲任何消息，指出這項弱點已遭有心人士利用？**  
否。 當本資訊安全公告初次發行時，Microsoft 尚未接到任何有關本弱點已公開用來攻擊客戶的消息。

SMTP 記憶體配置弱點 - CVE-2010-0025
-----------------------------------

<span></span>
由於 Microsoft Windows 簡易郵件傳送通訊協定 (SMTP) 元件 處理記憶體配置的方式，導致 SMTP 元件產生資訊洩漏弱點。 攻擊者可將後面接著 STARTTLS 命令的無效命令傳送到受影響的伺服器，來利用此弱點。 成功利用此弱點的攻擊者可讀取受影響伺服器上儲存的隨機電子郵件訊息片段。 請注意，這個弱點不會直接允許攻擊者執行程式碼或提升權限，但可能會用來產生可以進一步破壞受影響系統的資訊。

若要在「一般性弱點」清單標準條目中檢視此弱點，請參閱 [CVE-2010-0025](https://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0025) (英文)。

#### SMTP 記憶體配置弱點 - CVE-2010-0025 的緩和因素

Microsoft 尚未找到此項弱點的任何緩和因素。

#### SMTP 記憶體配置弱點 - CVE-2010-0025 的因應措施

Microsoft 尚未找到此項弱點的任何因應措施。

#### SMTP 記憶體配置弱點 - CVE-2010-0025 的常見問題集

**這個弱點的範圍為何？**  
這是一個資訊洩漏的弱點。 成功利用此弱點的攻擊者可讀取受影響伺服器上儲存的隨機電子郵件訊息片段。 請注意，這個弱點不會直接允許攻擊者執行程式碼或提升權限，但可能會用來產生可以進一步破壞受影響系統的資訊。

**造成這個弱點的原因為何？**  
形成這個弱點的原因，是由於 Microsoft Windows 簡易郵件傳送通訊協定 (SMTP) 元件在解譯 SMTP 命令回應時不當配置記憶體所致。

**什麼是 SMTP 服務？**  
簡易郵件傳送通訊協定 (SMTP) 是隨電子郵件服務或 Internet Information Services (IIS) 安裝的一項服務，用途是傳送電子郵件。 SMTP 控制電子郵件經由網際網路傳遞到目的地伺服器的方式。 SMTP 服務會在伺服器之間傳送和接收電子郵件。 如需詳細資訊，請參閱 TechNet 文章：[Simple Mail Transfer Protocol (SMTP) 服務](https://technet.microsoft.com/zh-tw/library/cc759281(ws.10).aspx)。

**什麼是 TLS 通訊協定？**  
傳輸層安全性 (TLS) 交握通訊協定負責建立或恢復安全工作階段的必備驗證及金鑰交換。 建立安全工作階段時，交握通訊協定會管理下列項目：

-   加密套件交涉
-   伺服器及用戶端 (選擇性) 的驗證
-   工作階段金鑰資訊交換

如需詳細資訊，請參閱 TechNet 文章：[TLS/SSL 如何運作](https://technet.microsoft.com/en-us/library/cc783349(ws.10).aspx) (英文)。

**什麼是 SSL？**  
安全通訊端層 (SSL) 是「傳輸層安全性」通訊協定的前身。 透過公開金鑰與秘密金鑰技術的結合，SSL 可以提供相同的功能，並支援安全的網路通訊。 如需詳細資訊，請參閱 TechNet 文章：[TLS/SSL 如何運作](https://technet.microsoft.com/en-us/library/cc783349(ws.10).aspx) (英文)。

**什麼是安全通道 (SChannel) 安全性套件？**  
安全通道 (Schannel) 安全性套件是實作網際網路標準驗證通訊協定、安全通訊端層 (SSL) 和傳輸層安全性 (TLS) 的安全性支援提供者 (SSP)。 這些元件用於實作安全通訊以支援幾種常見的網際網路及網路應用程式，例如網頁瀏覽。 Schannel 是協助提供驗證服務，以在用戶端和伺服器之間提供安全通訊之安全性套件的一部分。 如需詳細資訊，請參閱 MSDN 文章：[Secure Channel](https://msdn.microsoft.com/en-us/library/aa380123(vs.85).aspx) (英文)。

**攻擊者可能會利用這項弱點採取什麼行動？**  
成功利用此弱點的攻擊者可存取受影響伺服器上的隨機電子郵件訊息片段。

**攻擊者如何利用這項弱點？**  
攻擊者可將一組蓄意製作的 SMTP 命令傳送給受影響的 SMTP 伺服器，以利用這項弱點。 攻擊者隨後即可在受影響系統的記憶體中讀取資訊。

**因為這個弱點而承受風險的主要系統有哪些？**  
所有採用系統隨附的原生 SMTP 元件的作業系統，都有遭受此弱點攻擊的風險。

**更新的作用何在？**  
此資訊安全更新修正了 SMTP 元件在解譯 SMTP 命令回應時採取的記憶體配置方式，進而解決此弱點。

**當資訊安全公告發行時，這項弱點是否已被揭發出來？**  
否。 Microsoft 是經由可靠的來源接獲有關這項弱點的訊息。

**當本資訊安全公告發行時，Microsoft 是否已接獲任何消息，指出這項弱點已遭有心人士利用？**  
否。 當本資訊安全公告初次發行時，Microsoft 尚未接到任何有關本弱點已公開用來攻擊客戶的消息。

Application Compatibility Toolkit (ACT) 包含必要的工具和文件，可讓您在環境中部署 Microsoft Windows Vista、Windows Update、Microsoft 資訊安全更新或新版 Windows Internet Explorer 之前，評估及減輕應用程式相容性問題。

**確認已經套用更新**

-   **Microsoft Baseline Security Analyzer**

    您可以使用 Microsoft Baseline Security Analyzer (MBSA) 工具確認受影響系統上是否已經套用資訊安全更新程式。 如需詳細資訊，請參閱本公告前段的＜偵測與部署工具及指南＞一節。

-   **檔案版本驗證**

    由於 Microsoft Windows 的版本眾多，您電腦上實際執行的步驟可能會與此處描述的不同。 如遇到不同的狀況，請參閱產品的說明文件以完成這些步驟。

    1.  按一下 \[開始\]，然後按一下 \[搜尋\]。
    2.  在 \[搜尋結果\] 窗格中，在 \[搜尋小幫手\] 下按一下 \[所有檔案和資料夾\]。
    3.  在 \[部份或完整的檔案名稱\] 方塊中，輸入適當檔案資訊表中的檔案名稱，再按一下 \[搜尋\]。
    4.  在檔案清單中，用滑鼠右鍵按一下適當檔案資訊表中的某個檔案名稱，再按 \[內容\]。

    由於 Microsoft Windows 的版本眾多，您電腦上實際執行的步驟可能會與此處描述的不同。 如遇到不同的狀況，請參閱產品的說明文件以完成這些步驟。

    2.  在 \[搜尋結果\] 窗格中，在 \[搜尋小幫手\] 下按一下 \[所有檔案和資料夾\]。
    3.  在 \[部份或完整的檔案名稱\] 方塊中，輸入適當檔案資訊表中的檔案名稱，再按一下 \[搜尋\]。

    5.  在 \[版本\] 索引標籤上，比較檔案版本與適當檔案資訊表中記錄的版本，判斷您系統上安裝的檔案版本。  

        **注意：**在安裝時，檔案版本以外的屬性可能會變更。 若要驗證是否成功套用更新程式，利用檔案資訊表比對其他檔案屬性並不是妥當的做法。 此外，在某些情況下，檔案的名稱在安裝時可能會有所變更。 如果缺少檔案或版本資訊，請採用其他可用的方法來驗證更新程式的安裝情形。

    由於 Microsoft Windows 的版本眾多，您電腦上實際執行的步驟可能會與此處描述的不同。 如遇到不同的狀況，請參閱產品的說明文件以完成這些步驟。

    3.  在 \[一般\] 索引標籤下，將檔案大小與公告 KB 文件提供的檔案資訊表進行比對。
