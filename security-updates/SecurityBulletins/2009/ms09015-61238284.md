Security Bulletin

Microsoft Security Bulletin MS09-015 - 中等
===========================================

SearchPath 中的混合威脅弱點可能會允許權限提高 (959426)
------------------------------------------------------

發行: 2009年4月15日

**版本:** 1.0

### 一般資訊

#### 提要

這個安全性更新可解決 Windows SearchPath 函式中一項公開報告的弱點：如果使用者下載蓄意製作的檔案至特定位置，之後開啟了可能在特定情況下載入該檔案的應用程式，則可能會允許權限提高。

對於所有受支援版本的 Windows XP、Windows Server 2003、Windows Vista、Windows Server 2008，這個安全性更新的等級是中度。在 Microsoft Windows 2000 上，這個更新程式僅會歸類為深度防禦變更。 詳細資訊載於＜與本安全性更新相關的常見問題集 (FAQ)＞一節。 如需更多資訊，請參閱本節中的＜受影響及不受影響的軟體＞小節。

此安全性更新是藉由修改 Windows 從桌面載入檔案的方式來解決弱點。 如需更多有關弱點的資訊，請參閱下節＜弱點資訊＞下的特定弱點項目的＜常見問題集 (FAQ)＞小節。

這個安全性更新也能解決 [Microsoft 安全性摘要報告 953818](http://technet.microsoft.com/security/advisory/953818) 中首先提出的弱點。

**建議。** 大部分客戶都已啟用自動更新，並且不必須採取任何行動，因為安全性更新將自動下載和安裝。 沒有啟用自動更新的客戶則必須檢查更新，並手動安裝更新。 如需有關自動更新中特定設定選項的資訊，請參閱 [Microsoft 知識庫文件編號 294781](http://support.microsoft.com/kb/294871/zh-tw)。

若是系統管理員和企業安裝，或是想要手動安裝此安全性更新的使用者，Microsoft 建議客戶考慮使用更新管理軟體套用安全性更新，或使用 [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) 服務檢查更新。

**已知問題。** 無

#### 受影響及不受影響的軟體

下列軟體已經過測試判斷哪些版號或版本會受到影響。 其他版本超過它們的支援週期或不受影響。 若要瞭解您的軟體版本的支援週期，請造訪 [Microsoft 產品技術支援週期網站](http://go.microsoft.com/fwlink/?linkid=21742)。

與本安全性更新相關的常見問題集 (FAQ)  
------------------------------------
  
 
**何處可找到檔案資訊詳細資料？**    
如需檔案資訊詳細資料，請參閱 [Microsoft 知識庫文件編號 959426](http://support.microsoft.com/kb/959426/zh-tw)。
  
**此更新程式是否包含功能上的任何安全性相關變更？**    
是。 除了本公告＜弱點資訊＞一節中列出的變更外，此更新還包含一項深度防禦變更，加入了新的 API，稱為 SetSearchPathMode。
  
**SetSearchPathMode 這個深度防禦變更有什麼效果？**    
這個新的 API 可允許您在使用 SearchPath 函式尋找檔案時，採用獨立程序的模式。 如此讓應用程式能夠強制要求在優先搜尋完應用程式與系統位置後，才搜尋目前目錄。
  
**如果我只套用這個安全性更新，是否就能有完全的保護，免受 CVE-2008-2540 中所述的混合威脅之害？**    
否，您也必須另外啟用 Internet Explorer 積存安全性更新 [MS09-014](http://technet.microsoft.com/security/bulletin/ms09-014) 所提供的深度防禦保護。不過，您必須先確認自己的環境適用於啟用此深度防禦保護，以防範該混合威脅。
  
若要啟用 Internet Explorer 中的深度防禦保護，請依 [MS09-014](http://technet.microsoft.com/security/bulletin/ms09-014) 中＜與本安全性更新相關的常見問題集 (FAQ)＞該節所述，啟用 SearchPath 功能控制機碼。如需更多有關該混合威脅的資訊，請參閱一般性弱點編號 [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540)。
  
**為什麼 Microsoft Windows 2000 雖列於「受影響的軟體」表，卻沒有註明安全性影響與嚴重性等級？**  
Microsoft Windows 2000 列入表中卻未註明安全性影響和嚴重性等級，是因為此更新對 Microsoft Windows 2000 的修正，與其他支援的 Windows 作業系統不同。 此更新僅對 Microsoft Windows 2000 提供一個專用的深度防禦變更。
  
**為什麼這個混合威脅在 Microsoft Windows 2000 上用的修正，會與其他平台不同？ **  
Microsoft Windows 2000 系統沒有實作 SetSearchPathMode 函式所需的架構，因此不可能在 Micorosoft Windows 2000 排除此弱點。若要排除，便必須重新架構 Microsoft Windows 2000 作業系統許多部分，無法僅修改受影響的元件。 重新修改架構的成果可能與 Microsoft Windows 2000 有相當程度地不相容，無法保證原本為 Microsoft Windows 2000 設計的應用程式能繼續在重新架構後的系統上執行。 不過，Microsoft 未發現任何有效的攻擊模式可在 Microsoft Windows 2000 上攻擊此弱點。
  
因為這些架構變更，所以在 Microsoft Windows 2000 上，您必須使用登錄機碼 HKLM\\System\\CCS\\control\\Session Manager\\SafeDLLSearchMode，啟用 Microsoft Windows 2000 專用的深度防禦變更，。 如需相關指示，請參閱 [Microsoft 知識庫文件編號 959426](http://support.microsoft.com/kb/959426/zh-tw)。
  
**什麼是 SearchPath 函式？**    
SearchPath 函式是用於在指定的路徑搜尋指定的檔案。 如需詳細資訊，請參閱 MSDN 文章「[SearchPath 函式](http://msdn.microsoft.com/en-us/library/aa365527.aspx)」(英文)。
  
**什麼是 LoadLibrary 函式？**    
LoadLibrary 函式將指定的模組載入至呼叫程序的位址空間。 該指定模組也可能導致載入其他模組。 如需詳細資訊，請參閱 MSDN 文章「[LoadLibrary 函式](http://msdn.microsoft.com/en-us/library/ms684175(vs.85).aspx)」(英文)。
  
**我所使用的軟體是這個安全性公告中討論的軟體之舊版。 該怎麼辦？**    
本公告所列出的受影響軟體版本已經過測試判斷哪些版本會受到影響。 其他版本超出它們的支援週期。 若要瞭解您的軟體版本的支援週期，請造訪 [Microsoft 產品技術支援週期網站](http://go.microsoft.com/fwlink/?linkid=21742)。
  
使用此軟體舊版的客戶應優先考慮移轉至支援的版本，以避免因潛在的弱點而遭受攻擊。 如需更多有關 Windows 產品週期的資訊，請造訪 [Microsoft 產品技術支援週期](http://go.microsoft.com/fwlink/?linkid=21742)。 如需瞭解這些軟體版本延伸安全性更新支援服務週期的相關資訊，請造訪 [Microsoft 產品支援服務](http://go.microsoft.com/fwlink/?linkid=33328)。
  
需要舊版額外支援服務的客戶，請連絡 Microsoft 客戶小組人員、技術支援經理或適當的 Microsoft 協力廠商，以取得所需的額外支援。 尚未簽訂聯盟、優先或授權合約的客戶，可以連絡當地的 Microsoft 銷售辦公室。 如需連絡資訊，請造訪 [Microsoft 全球資訊網站](http://go.microsoft.com/fwlink/?linkid=33329)，選擇國家，然後按一下 \[Go\] 查看各地的連絡電話號碼。 連絡時，請指明要連絡當地優先支援服務行銷經理。 如需更多資訊，請參閱 [Windows 作業系統產品技術支援週期常見問題集](http://go.microsoft.com/fwlink/?linkid=33330) (英文)。

混合威脅權限提高弱點 - CVE-2008-2540  
------------------------------------
  
 
Windows 的 SearchPath 函式在系統中尋找並開啟檔案的方法有權限提高弱點的混合威脅。 攻擊者可能利用這個弱點，引誘使用者下載蓄意製作的檔案至特定位置，然後開啟可能在特定情況下載入該檔案的應用程式。
  
若要以一般性弱點清單中的標準項目來檢視此弱點，請參閱 [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540)。
  
#### 混合威脅權限提高弱點 - CVE-2008-2540 的緩和因素
  
Microsoft 尚未找到此項弱點的任何緩和因素。
  
#### 混合威脅權限提高弱點 - CVE-2008-2540 的因應措施
  
Microsoft 尚未找到此項弱點的任何因應措施。
  
#### 混合威脅權限提高弱點 - CVE-2008-2540 的常見問題集
  
**這個弱點的範圍為何？**    
這是混合威脅權限提高弱點。 成功利用此弱點的攻擊者可以從遠端取得受影響系統的完整控制權。 攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。
  
**造成這個弱點的原因為何？**    
應用程式程式庫位於 Windows 系統的方式，有可能會讓 Internet Explorer 可以從桌面開啟蓄意製作的檔案。 不先提示就讓檔案下載到使用者系統的混合威脅，會造成 Internet Explorer 不小心從桌面 (而非 Windows 系統) 載入該下載的檔案。
  
**攻擊者可能會利用這項弱點採取什麼行動？**    
成功利用此弱點的攻擊者可以執行任意程式碼。 攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。
  
**攻擊者如何利用這項弱點？**    
攻擊者可建立蓄意製作的檔案，然後引誘使用者將檔案下載到桌面。 使用者之後必須啟動一個應用程式，該應用程式須會開啟此蓄意製作的檔案。 不過，攻擊者並沒有任何方法能夠強迫使用者下載此類檔案並置於桌面。
  
**因為這個弱點而承受風險的主要系統有哪些？**    
所有使用者會下載檔案並啟動應用程式的系統，例如工作站與終端機伺服器，均因此弱點而承受風險。
  
**更新的作用何在？**    
這個更新會修改應用程式庫的搜尋順序，優先搜尋系統目錄，之後才搜尋目前使用中的目錄。 這個更新也提供深度防禦變更，一個稱為 SetSearchPathMode的新 API，以協助進階使用者設定 SearchPath 函式應使用的搜尋順序。 如需此深度防禦變更的更多資訊，請參閱＜與本安全性更新相關的常見問題集 (FAQ)＞一節。
  
**當安全性公告發行時，這項弱點是否已被揭發出來？**    
是。 這項弱點已經遭到公開揭發。 這項弱點已被指派一般性弱點編號 [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540)。[這個安全性更新也能解決 Microsoft 安全性摘要報告 953818](http://technet.microsoft.com/security/advisory/953818) 中首次提出的弱點。
  
**當本安全性公告發行時，Microsoft 是否已接獲任何消息，指出這項弱點已遭有心人士利用？**    
否。 當本安全性公告初次發行時，Microsoft 並未接到任何有關本弱點已成為公開攻擊媒介的消息，也沒有發現任何以此概念發展的程式碼公開範例。

    5.  在 \[版本\] 索引標籤上，比較檔案版本與適當檔案資訊表中記錄的版本，判斷您系統上安裝的檔案版本。
        **注意：**在安裝時，檔案版本以外的屬性可能會變更。 若要驗證是否成功套用更新程式，利用檔案資訊表比對其他檔案屬性並不是妥當的做法。 此外，在某些情況下，檔案的名稱在安裝時可能會有所變更。 如果缺少檔案或版本資訊，請採用其他可用的方法來驗證更新程式的安裝情形。
