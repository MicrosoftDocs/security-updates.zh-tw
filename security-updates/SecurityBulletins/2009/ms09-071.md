Security Bulletin

Microsoft Security Bulletin MS09-071 - 重大
===========================================

網際網路驗證服務中的弱點可能會允許遠端執行程式碼 (974318)
---------------------------------------------------------

發行: 2009年12月9日 | 更新: 2009年12月30日

**版本:** 1.1

### 一般資訊

#### 提要

這個資訊安全更新可解決 Microsoft Windows 中兩項未公開報告的弱點。 若處理 PEAP 驗證嘗試時，將網際網路驗證服務伺服器接收到的訊息錯誤地複製到記憶體，這類弱點可能會允許遠端執行程式碼。 成功利用上述任一弱點的攻擊者可以取得受影響系統的完整控制權。 使用網際網路驗證服務的伺服器，只有在搭配 MS-CHAP v2 驗證使用 PEAP 時，才會受影響。

對適用於 32 位元系統的 Windows Server 2008 Service Pack 2 和適用於 x64 型系統的 Windows Server 2008 Service Pack 2，此資訊安全更新的等級為重大。對於其他所有受支援或受影響版本的 Windows，請參閱本節中的＜受影響及不受影響的軟體＞小節

本資訊安全更新會修正網際網路驗證服務對 PEAP 用戶端驗證要求的驗證方式，藉此解決這類弱點。 如需更多有關弱點的資訊，請參閱下節＜弱點資訊＞下的特定弱點項目的＜常見問題集 (FAQ)＞小節。

**建議。**  大部分客戶都已啟用自動更新，並且不必須採取任何行動，因為資訊安全更新將自動下載和安裝。 沒有啟用自動更新的客戶則必須檢查更新，並手動安裝更新。 如需有關自動更新中特定設定選項的資訊，請參閱 [Microsoft 知識庫文件編號 294871](https://support.microsoft.com/kb/294871/?ln=zh-tw)。

若是系統管理員和企業安裝，或是想要手動安裝此資訊安全更新的使用者，Microsoft 建議客戶使用更新管理軟體，立即套用更新，或使用 [Microsoft Update](https://go.microsoft.com/fwlink/?linkid=40747) 服務檢查更新。

另請參閱本公告下文的＜偵測與部署工具及指南＞章節。

**已知問題。** 無

#### 受影響及不受影響的軟體

下列軟體已經過測試判斷哪些版號或版本會受到影響。 其他版本超過它們的支援週期或不受影響。 若要瞭解您的軟體版本的支援週期，請造訪 [Microsoft 產品技術支援週期網站](https://go.microsoft.com/fwlink/?linkid=21742)。

與本資訊安全更新相關的常見問題集 (FAQ)  
--------------------------------------
  
<span></span>
**為何於 2009 年 12 月 9 日修訂此公告？**    
本公告已修訂以進行若干內容新增與修正：
  
-   於＜提要＞、＜網際網路驗證服務記憶體損毀弱點 – CVE-2009-2505 的常見問題集＞以及＜MS-CHAP 驗證略過弱點 – CVE-2009-3677 的常見問題集＞等章節新增 Windows Server 2008 的網路原則伺服器相關資訊。  
-   於＜網際網路驗證服務記憶體損毀弱點 – CVE-2009-2505 的常見問題集＞以及＜MS-CHAP 驗證略過弱點 – CVE-2009-3677 的常見問題集＞小節新增 Microsoft Windows 2000 Server 的網路驗證伺服器相關資訊。  
-   修正＜網際網路驗證服務記憶體損毀弱點 – CVE-2009-2505 的常見問題集＞一節中的公開揭露項目。  
-   修正＜安全性更新部署＞一節中適用於 Windows XP Professional x64 Edition Service Pack 2 的登錄機碼驗證資訊。
  
**何處可找到檔案資訊詳細資料？**    
關於檔案資訊詳細資料的位置，請見＜資訊安全更新部署＞一節中的參考表。
  
**為何本次更新中仍然包含數個已回報的安全性弱點？**    
因為解決這些弱點所需的修正作業位於相關檔案中，所以在本次更新中仍然提供對這些弱點的支援。 客戶不需要安裝數個幾乎相同的更新，只需要安裝本更新即可。
  
**為什麼自動更新和 Microsoft 下載中心把「受影響的軟體」表中的資訊安全更新列為不同的嚴重性等級？**    
部分受影響的 Windows 版本對於不同版本的作業系統會評為不同的嚴重性等級。 在各個案例中，自動更新與 Microsoft 下載中心提供的可下載資訊安全更新套件會指出作業系統最高的彙總嚴重性等級。
  
**我所使用的軟體是這個資訊安全公告中討論的軟體之舊版。 該怎麼辦？**    
本公告所列出的受影響軟體版本已經過測試判斷哪些版本會受到影響。 其他版本超出它們的支援週期。 如需瞭解產品生命週期的相關資訊，請造訪 [Microsoft 產品技術支援週期](https://go.microsoft.com/fwlink/?linkid=21742)網站。
  
使用此軟體舊版的客戶應優先考慮移轉至支援的版本，以避免因潛在的弱點而遭受攻擊。 若要瞭解您的軟體版本的支援週期，請參閱[選擇一個產品檢視其支援週期資訊](https://go.microsoft.com/fwlink/?linkid=169555)。 如需更多軟體版本 Service Pack 的相關資訊，請參閱[產品技術支援週期所支援的 Service Packs](https://go.microsoft.com/fwlink/?linkid=89213)。
  
需要舊版軟體額外支援服務的客戶，請連絡 Microsoft 客戶小組人員、技術支援經理或適當的 Microsoft 協力廠商，以取得所需的額外支援。 尚未簽訂聯盟、優先或授權合約的客戶，可以連絡當地的 Microsoft 銷售辦公室。 如需連絡資訊，請造訪 [Microsoft 全球資訊網站](https://go.microsoft.com/fwlink/?linkid=33329)，在 \[Contact Information\] (連絡資訊) 清單中選擇國家，然後按一下 \[Go\] 查看各地的連絡電話號碼。 連絡時，請指明要連絡當地優先支援服務行銷經理。 如需更多資訊，請參閱 [Microsoft 技術支援週期準則常見問答集](https://go.microsoft.com/fwlink/?linkid=169557)。

網際網路驗證服務記憶體損毀弱點 - CVE-2009-2505  
----------------------------------------------
  
<span></span>
在網際網路驗證服務上實作受保護的可延伸驗證通訊協定 (PEAP) 時，存在遠端執行程式碼弱點。 處理 PEAP 驗證嘗試時，將伺服器接收到的訊息錯誤地複製到記憶體，因此出現這個弱點。 成功利用此弱點的攻擊者可以取得受影響系統的完整控制權。
  
若要以一般性弱點清單中的標準項目來檢視此弱點，請參閱 [CVE-2009-2505](https://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2505)。
  
#### 網際網路驗證服務記憶體損毀弱點 CVE-2009-2505 的緩和因素
  
緩和因素是指存在於預設狀態中的設定、共用設定或一般最佳作法，可能會減少弱點影響的嚴重性。 下列緩和因素可能對您的狀況有所助益：
  
-   使用網際網路驗證服務的伺服器，只有在設定為使用 PEAP-MS-CHAP v2 驗證時才會受到影響。
  
#### 網際網路驗證服務記憶體損毀弱點 CVE-2009-2505 的因應措施
  
因應措施指的是無法徹底修正弱點，但有助於在套用更新之前封鎖已知攻擊模式的設定變更。 Microsoft 測試了下列因應措施和狀態，討論因應措施是否會降低功能：
  
-   **在您的網際網路驗證服務伺服器上使用 PEAP-MS-CHAP v2 以外的驗證通訊協定**
  
    **因應措施的影響** 驗證通訊協定的變更需要由用戶端重新設定。
  
#### 網際網路驗證服務記憶體損毀弱點 CVE-2009-2505 的常見問題集
  
**這個弱點的範圍為何？**    
這是遠端執行程式碼的弱點。 成功利用此弱點的攻擊者可以從遠端取得受影響系統的完整控制權。 攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。
  
**造成這個弱點的原因為何？**    
這個弱點是因為 PEAP 驗證要求的驗證不足所造成。 在 IAS 伺服器處理 PEAP 驗證嘗試時複製特定記憶體結構，可能會引發錯誤。
  
**什麼是受保護的可延伸驗證通訊協定 (PEAP)？**  
受保護的可延伸驗證通訊協定 (PEAP) 是可延伸驗證通訊協定 (EAP) 通訊協定系列的一員。 PEAP 會使用傳輸層安全性 (TLS)，在驗證的 PEAP 用戶端 (例如透過無線網路通訊的行動電腦) 與 PEAP 驗證者 (例如網際網路驗證服務 (IAS) 或遠端驗證撥入使用者服務 (RADIUS) 伺服器) 之間建立加密通道。 PEAP 並不會指定驗證方法，但會提供額外的安全性給其他 EAP 驗證通訊協定，例如透過 PEAP 所提供之 TLS 加密通道而運作的 EAP-MS-CHAP v2。 PEAP 是用於 802.1X 無線用戶端電腦的驗證方法，但不受虛擬私人網路 (VPN) 或其他遠端存取用戶端的支援。
  
**什麼是 Microsoft Challenge Handshake 驗證通訊協定第 2 版 (MS-CHAP v2)？**  
MS-CHAP 使用挑戰/回應機制驗證連線，而不傳送任何密碼。 MS-CHAP v2 可提供相互驗證、產生較複雜的 Microsoft 點對點加密 (MPPE) 初始資料加密金鑰，並針對收發資料提供不同的加密金鑰。 為了在密碼變更期間降低密碼外洩的風險，並不支援舊式的 MS-CHAP 密碼變更方法。
  
MS-CHAP v2 是項相互驗證的通訊協定，也就是說，用戶端和伺服器雙方都需證明各自知曉使用者密碼。 首先，遠端存取伺服器會傳送挑戰給用戶端，要求遠端存取用戶端提出證明。 接著，遠端存取用戶端也會傳送挑戰給伺服器，要求遠端存取伺服器提出證明。 若伺服器無法正確回應用戶端的挑戰，證明其知曉使用者密碼，用戶端會中斷連線。 沒有相互驗證，遠端存取用戶端就無法與未經授權的遠端存取伺服器建立連線。
  
**什麼是網際網路驗證服務和網路原則伺服器？**    
在 Microsoft Windows 2000 Server 中，網際網路驗證服務 (IAS) 是 Microsoft 實作的遠端驗證撥入使用者服務 (RADIUS) 伺服器。 做為 RADIUS 伺服器，IAS 會針對多種類型的網路存取 (包括無線和虛擬私人網路 (VPN) 連線) 執行集中連線驗證、授權與帳戶處理。 在 Microsoft Windows Server 2000 中，IAS 不支援 RADIUS Proxy，因為其使用的是 Option Pack 版。
  
在 Windows Server 2003 中，網際網路驗證服務 (IAS) 是 Microsoft 實作的遠端驗證撥入使用者服務 (RADIUS) 伺服器和 Proxy。 做為 RADIUS 伺服器，IAS 會針對多種類型的網路存取 (包括無線和虛擬私人網路 (VPN) 連線) 執行集中連線驗證、授權與帳戶處理。 做為 RADIUS Proxy，IAS 會轉送驗證和帳戶處理訊息到其他 RADIUS 伺服器。
  
網路原則伺服器 (NPS) 是 Microsoft 在 Windows Server 2008 中實作的遠端驗證撥入使用者服務 (RADIUS) 伺服器和 Proxy。NPS 取代了 Windows Server 2003 中的網際網路驗證服務 (IAS)。
  
網際網路驗證服務和網路原則伺服器均有受此弱點影響的 PEAP MS-CHAP v2 程式碼。
  
**為何此弱點在 Windows 作業系統用戶端上的嚴重性等級為「重要」？**    
雖然 Windows 作業系統用戶端包含易受影響的 MS-CHAP v2 程式碼，但作業系統的內含元件對於該程式碼的使用方式並不會暴露此弱點。 理論上，協力廠商應用程式若是支援使用 MS-CHAP v2 的驗證方式，其系統可能會受影響。 因為用戶端並未刻意提供此類功能，這種情況不太可能會發生。 然而，Microsoft 還是在此資訊安全公告中提供更新，在用戶端平台上根本解決這項問題。
  
**為何此弱點在適用於 Itanium 系統的 Windows Server 2008 上的嚴重性等級為「重要」？**    
適用於 Itanium 系統的 Windows Server 2008 不允許網際網路驗證服務的安裝。 但是，電腦有易受影響的二進位檔案。 理論上，協力廠商應用程式若是支援使用 MS-CHAP v2 的驗證方式，其系統可能會受影響。 因為這些伺服器並未刻意提供此類功能，這種情況不太可能會發生。 然而，Microsoft 還是在此資訊安全公告中提供更新，在適用於 Itanium 系統的 Windows Server 2008 上根本解決這項問題。
  
**攻擊者可能會利用這項弱點採取什麼行動？**    
成功利用此弱點的攻擊者能以系統管理員權限執行任意程式碼。 攻擊者接下來能在系統上安裝程式；檢視、變更或刪除資料；或建立新帳戶。
  
**攻擊者如何利用這項弱點？**    
攻擊者可藉由傳送錯誤格式的 PEAP 驗證要求給 IAS 伺服器，來利用這項弱點。
  
**因為這個弱點而承受風險的主要系統有哪些？**    
允許遠端、未受信任的使用者進行驗證的網際網路驗證服務伺服器，特別容易受此弱點影響。
  
**更新的作用何在？**    
此更新會將 IAS 變更為正確驗證 MS-CHAP v2 驗證要求。
  
**當資訊安全公告發行時，這項弱點是否已被揭發出來？**    
否。 Microsoft 是經由可靠的來源接獲有關這項弱點的訊息。
  
**當本資訊安全公告發行時，Microsoft 是否已接獲任何消息，指出這項弱點已遭有心人士利用？**    
否。 當本資訊安全公告初次發行時，Microsoft 尚未接到任何有關本弱點已公開用來攻擊客戶的消息。

適用於 Itanium 系統的 Windows Server 2008 不允許網際網路驗證服務的安裝。 但是，電腦有易受影響的二進位檔案。 理論上，協力廠商應用程式若是支援使用 MS-CHAP v2 的驗證方式，其系統可能會受影響。 因為這些伺服器並未刻意提供此類功能，這種情況不太可能會發生。 然而，Microsoft 還是在此資訊安全公告中提供更新，在適用於 Itanium 系統的 Windows Server 2008 上根本解決這項問題。
  
**攻擊者可能會利用這項弱點採取什麼行動？**    
成功利用這項弱點的攻擊者可能使用特定授權使用者的權限，存取網路資源。 請注意，視網路主控應用程式的驗證要求而定，使用者可能沒有直接存取特定資料或資訊的權限。
  
**攻擊者如何利用這項弱點？**    
攻擊者可以傳送錯誤格式的 MS-CHAP v2 驗證要求給網際網路驗證服務，藉此利用此弱點。
  
**因為這個弱點而承受風險的主要系統有哪些？**    
系統若是使用網際網路驗證服務提供外部使用者網路資源的存取權限，特別容易受此弱點影響。
  
**更新的作用何在？**    
此更新會將 IAS 變更為正確驗證 MS-CHAP v2 驗證要求，藉以解決弱點。
  
**當資訊安全公告發行時，這項弱點是否已被揭發出來？**    
否。 Microsoft 是經由可靠的來源接獲有關這項弱點的訊息。
  
**當本資訊安全公告發行時，Microsoft 是否已接獲任何消息，指出這項弱點已遭有心人士利用？**    
否。 當本資訊安全公告初次發行時，Microsoft 尚未接到任何有關本弱點已公開用來攻擊客戶的消息。

#### 部署資訊
  
**安裝更新**
  
當您安裝本資訊安全更新時，安裝程式會檢查系統上要更新的檔案先前是否曾用 Microsoft Hotfix 進行更新。
  
如需更多關於出現於本公告中術語的相關資訊 (如 *Hotfix*)，請參閱 [Microsoft 知識庫文件編號 824684](https://support.microsoft.com/kb/824684/?ln=zh-tw)。
  
這個資訊安全更新支援以下的安裝參數。
  
| 參數              | 描述                                                                                       |  
|-------------------|--------------------------------------------------------------------------------------------|  
| **/?、/h、/help** | 顯示受支援參數的說明。                                                                     |  
| **/quiet**        | 不顯示狀態或錯誤訊息。                                                                     |  
| **/norestart**    | 與 **/quiet** 結合使用時，系統在安裝後將不會重新啟動，即使需要重新啟動以完成安裝時亦如此。 |
  
**注意：**如需有關 wusa.exe 安裝程式的詳細資訊，請參閱 [Microsoft 知識庫文件編號 934307](https://support.microsoft.com/kb/934307/?ln=zh-tw)。
  
**確認已經套用更新**
  
-   **Microsoft Baseline Security Analyzer**
  
    您可以使用 Microsoft Baseline Security Analyzer (MBSA) 工具確認受影響系統上是否已經套用資訊安全更新程式。 如需詳細資訊，請參閱本公告前段的＜偵測與部署工具及指南＞一節。
  
-   **檔案版本驗證**
  
    由於 Microsoft Windows 的版本眾多，您電腦上實際執行的步驟可能會與此處描述的不同。 如遇到不同的狀況，請參閱產品的說明文件以完成這些步驟。
  
    1.  按一下 \[開始\]，然後在 \[開始搜尋\] 中輸入更新檔案名稱。  
    2.  當該檔案出現在 \[程式\] 下方時，在檔案名稱上按一下滑鼠右鍵，然後按一下 \[內容\]。  
    3.  在 \[一般\] 索引標籤下，將檔案大小與公告 KB 文件提供的檔案資訊表進行比對。  
    4.  您也可以按一下 \[詳細資料\] 索引標籤，然後將檔案版本與修改日期等資訊與公告 KB 文件提供的檔案資訊表進行比對。  
    5.  最後，您也可以按一下 \[舊版\]，將舊版檔案的檔案資訊，與檔案的新或更新版本之檔案資訊進行比較。
