Microsoft Security Bulletin MS07-039 - 重大
===========================================

Windows Active Directory 中的弱點可能會允許遠端執行程式碼 (926122)
------------------------------------------------------------------

發行: 2007年7月11日 | 更新: 2007年7月19日

**版本:** 1.1

### 一般資訊

#### 提要

這個重大安全性更新可解決在 Windows 2000 Server 和 Windows Server 2003 上實作 Active Directory 的一項未公開報告的弱點，此弱點可能會允許遠端執行程式碼，或發生拒絕服務的情況。 嘗試利用此弱點進行的攻擊，其最可能的結果是導致拒絕服務狀態。 但是，也可能導致遠端執行程式碼。 在 Windows Server 2003 上，攻擊者必須擁有有效的登入認證，才能利用這項弱點。 成功利用此弱點的攻擊者可以取得受影響系統的完整控制權。 攻擊者接下來能在系統上安裝程式；檢視、變更或刪除資料；或建立新帳戶。

這是適用於受支援版本的 Windows 2000 的重大安全性更新，也是適用於受支援版本 Windows Server 2003 的重要安全性更新。如需更多資訊，請參閱本節中＜受影響及不受影響的軟體＞小節。

本安全性更新可透過驗證用戶端 LDAP 要求中可轉換屬性的數目，來解決此項弱點。 如需更多有關此弱點的資訊，請參閱下節＜弱點資訊＞。

**建議。** Microsoft 建議客戶立即套用此更新。

**已知問題。**無。

下列軟體已經過測試判斷哪些版本會受到影響。 其他版本超過它們的支援週期或不受影響。 若要瞭解您的軟體版本的支援週期，請造訪 [Microsoft 產品技術支援週期網站](http://go.microsoft.com/fwlink/?linkid=21742)。

與本安全性更新相關的常見問題集 (FAQ)  
------------------------------------
  
**我沒有安裝任何受影響的軟體， 為什麼還是提供我這項安全性更新？**    
雖然本安全性更新所述弱點僅會影響以網域控制站角色運作的 Windows 2000 Server 與 Windows Server 2003 系統，但此弱點程式碼在所有支援版本的 Windows 2000 與 Windows 2003 中均存在，因此為包含此弱點程式碼的所有系統提供本更新。
  
**我安裝了 ADAM，該怎麼辦？**    
此弱點程式碼在 ADAM Service Pack 1 中不存在。但 ADAM Service Pack 1 之前的版本的確含有弱點程式碼。建議您安裝 ADAM Service Pack 1。 如需更多有關 ADAM Service Pack 1 的資訊，請造訪 [ADAM SP1 下載網站](http://www.microsoft.com/downloads/details.aspx?familyid=9688f8b9-1034-4ef6-a3e5-2a2a57b5c8e4&displaylang=en)。
  
**什麼是 ADAM？**    
ADAM 可為具備目錄功能的應用程式提供資料儲存與擷取服務，而沒有 Active Directory 服務所需的相依性。 ADAM 可提供 Active Directory 的大部分功能，但 ADAM 不需要部署網域或網域控制站，並且其目錄結構描述完全獨立於企業結構描述之外，而在 Active Directory 網域中則可能會用到企業結構描述。 您可在一台電腦中同時執行多個 ADAM 執行個體，並使每個 ADAM 執行個體具有獨立的管理結構描述。  
如需更多有關 ADAM 的資訊，請參閱 [Active Directory Application Mode 簡介](http://www.microsoft.com/windowsserver2003/adam/default.mspx) (英文)。
  
**我所使用的軟體是這個安全性公告中討論的軟體之舊版。 該怎麼辦？**    
本公告所列出的受影響軟體版本已經過測試判斷哪些版本會受到影響。 其他版本超出它們的支援週期。 若要瞭解您的軟體版本的支援週期，請造訪 [Microsoft 產品技術支援週期網站](http://go.microsoft.com/fwlink/?linkid=21742)。
  
使用此軟體舊版的客戶應優先考慮移轉至支援的版本，以避免因潛在的弱點而遭受攻擊。 如需更多有關 Windows 產品週期的資訊，請造訪下列 [Microsoft 產品技術支援週期網站](http://go.microsoft.com/fwlink/?linkid=21742)。 如需瞭解這些軟體版本延伸安全性更新支援服務週期的相關資訊，請造訪 [Microsoft 產品支援服務網站](http://go.microsoft.com/fwlink/?linkid=33328)。

Windows Active Directory 遠端執行程式碼弱點 - CVE-2007-0040  
-----------------------------------------------------------
  
Active Directory 驗證 LDAP 要求的方式中存在遠端執行程式碼的弱點。 成功利用此弱點的攻擊者可以取得受影響系統的完整控制權。
  
若要以一般性弱點清單中的標準項目來檢視此弱點，請參閱 [CVE-2007-0040](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-0040)。
  
#### Windows Active Directory 遠端執行程式碼弱點 - CVE-2007-0040 的緩和因素
  
緩和因素是指存在於預設狀態中的設定、共用設定或一般最佳作法，可能會減少弱點影響的嚴重性。 下列緩和因素可能對您的狀況有所助益：
  
-   對於需要受影響元件的客戶，最佳實作的防火牆和標準預設防火牆設定有助於防止網路受到來自企業外圍以外的攻擊。 最佳方式建議連線至網際網路的系統盡可能曝露最少數量的連接埠。  
-   嘗試利用此弱點進行的攻擊，其最可能的結果是導致拒絕服務狀態。 但是，也可能導致遠端執行程式碼。  
-   在 Windows Server 2003 上，攻擊者必須持有效登入認證才能利用這個弱點；匿名使用者無法利用此弱點。
  
#### Windows Active Directory 遠端執行程式碼弱點 - CVE-2007-0040 的因應措施
  
因應措施指的是無法徹底修正弱點，但有助於在套用更新之前封鎖已知攻擊模式的設定變更。 Microsoft 測試了下列因應措施和狀態，討論因應措施是否會降低功能：
  
-   **在防火牆封鎖下列項目：**  
  
    TCP 連接埠 389
  
    此連接埠可用來起始與受影響元件的連線。 在企業防火牆封鎖此連接埠 (含輸入及輸出) 有助於保護防火牆後的系統免遭此弱點的影響。 建議您封鎖所有網際網路中來路不明的輸入通訊，以防他人利用其他連接埠攻擊系統。 如需更多關於連接埠的資訊，請造訪 [TCP 和 UDP 連接埠的指派](http://go.microsoft.com/fwlink/?linkid=21312) (英文)。
  
-   **為了防範利用此弱點的網路攻擊，請在受影響的系統上使用 IPSec 封鎖受影響的連接埠。**  
  
    使用網際網路通訊協定安全性 (IPSec) 保護網路通訊。 如需更多關於 IPSec 及如何套用篩選的資訊，請參閱 [Microsoft 知識庫文件編號 313190](http://support.microsoft.com/kb/313190) 與 [Microsoft 知識庫文件編號 813878](http://support.microsoft.com/kb/813878)。
  
#### Windows Active Directory 遠端執行程式碼弱點 - CVE-2007-0040 的常見問題集
  
**這個弱點的範圍為何？**  
這是遠端執行程式碼的弱點。 成功利用此弱點的攻擊者可以從遠端取得受影響系統的完整控制權。 攻擊者接下來將能安裝程式，或檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。
  
**注意：** 嘗試利用此弱點進行的攻擊，其最可能的結果是導致拒絕服務狀態。 但是，也可能導致遠端執行程式碼，而且範圍比僅拒絕服務更大。
  
**造成這個弱點的原因為何？**  
對於 LDAP 要求中的可轉換屬性之數目，LDAP 服務未執行充分的檢查。
  
**攻擊者可能會利用這項弱點採取什麼行動？**  
成功利用此弱點的攻擊者可以取得受影響系統的完整控制權。
  
**攻擊者如何利用這項弱點？**  
對於 Windows 2000 Server，凡是對目標網路具有存取權的匿名使用者，都能遞送蓄意製作的網路封包到受影響的系統，以利用這項弱點。 在 Windows Server 2003 上，攻擊者必須持有效的驗證認證，才可能利用這項弱點。
  
**因為這個弱點而承受風險的主要系統有哪些？**  
Active Directory 必須使用 LDAP。 因此只要是作為 Active Directory Server 執行的 Windows 2000 Server 或 Windows Server 2003，都會受此弱點影響。
  
**是否可以透過網際網路利用這個弱點？**    
是。 攻擊者可以在網際網路上利用這項弱點。 然而，常見的防火牆實務作法都在企業基礎架構內包含 Active Directory 要求。 最佳實作的防火牆有助於防止來自網際網路所發動的攻擊。
  
**什麼是 LDAP？**    
[LDAP](http://msdn2.microsoft.com/en-us/library/aa503238.aspx) (輕量型目錄存取通訊協定) 是一種業界標準通訊協定，可讓獲授權的使用者查詢或修改中繼目錄的資料。 在 Windows Server 2000 和 Windows Server 2003 中便是利用 LDAP 通訊協定來存取 Active Directory 內的資料。
  
**更新的作用何在？**  
此更新可透過對 LDAP 要求中的可轉換屬性數目執行檢查，從而移除此項弱點。
  
**當安全性公告發行時，這項弱點是否已揭發出來？**  
否。 Microsoft 是經由可靠的來源接獲有關這項弱點的訊息。
  
**當本安全性公告發行時，Microsoft 是否已接獲任何消息，指出這項弱點已遭有心人士利用？**  
否。 當本安全性公告初次發行時，Microsoft 並未接到任何有關本弱點已成為公開攻擊媒介的消息，也沒有發現任何以此概念發展的程式碼公開範例。
  
Windows Active Directory 拒絕服務弱點 - CVE-2007-3028  
-----------------------------------------------------
  
Microsoft Active Directory 驗證用戶端所傳送 LDAP 要求的方式中，存在拒絕服務的弱點。 攻擊者可以傳送蓄意製作的 LDAP 要求到執行 Active Directory 的伺服器，以利用這項弱點。 成功利用此弱點的攻擊者可使伺服器暫時停止回應。
  
若要以一般性弱點清單中的標準項目來檢視此弱點，請參閱 [CVE-2007-3028](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-3028)。

#### Windows Active Directory 拒絕服務弱點 - CVE-2007-3028 的常見問題集
  
**這個弱點的範圍為何？**  
這是一項拒絕服務的弱點。 利用此弱點的攻擊者可使受影響的系統暫時停止回應。 在這段期間，伺服器將無法回應要求。 請注意拒絕服務弱點不會讓攻擊者執行程式碼或提高其使用者權限，但會導致受影響系統停止接受要求。
  
**造成這個弱點的原因為何？**  
LDAP 服務對用戶端 LDAP 要求中所傳送的資料執行不正確的解碼。
  
**攻擊者如何利用這項弱點？**  
攻擊者可以傳送蓄意製作的要求到受影響的系統，以利用此弱點。 該訊息接著便可能導致受影響的系統停止回應。
  
**因為這個弱點而承受風險的主要系統有哪些？**  
Active Directory 必須使用 LDAP。 因此只要是作為 Active Directory Server 執行的 Windows 2000 Server，都會受此弱點影響。
  
**是否可以透過網際網路利用這個弱點？**    
是。 攻擊者可以在網際網路上利用這項弱點。 然而，常見的防火牆實務作法都在企業基礎架構內包含 Active Directory 要求。 最佳實作的防火牆有助於防止來自網際網路所發動的攻擊。
  
**什麼是 LDAP？**    
[LDAP](http://msdn2.microsoft.com/en-us/library/aa503238.aspx) (輕量型目錄存取通訊協定) 是一種業界標準通訊協定，可讓獲授權的使用者查詢或修改中繼目錄的資料。 在 Windows Server 2000 和 Windows Server 2003 中便是利用 LDAP 通訊協定來存取 Active Directory 內的資料。
  
**更新的作用何在？**  
此更新可驗證用戶端所傳送 LDAP 要求的邏輯，藉以移除此項弱點。
  
**當安全性公告發行時，這項弱點是否已揭發出來？**  
否。 Microsoft 是經由可靠的來源接獲有關這項弱點的訊息。
  
**當本安全性公告發行時，Microsoft 是否已接獲任何消息，指出這項弱點已遭有心人士利用？**  
否。 當本安全性公告初次發行時，Microsoft 並未接到任何有關本弱點已成為公開攻擊媒介的消息，也沒有發現任何以此概念發展的程式碼公開範例。

    由於 Microsoft Windows 的版本眾多，您電腦上實際執行的步驟可能會與此處描述的不同。 如遇到不同的狀況，請參閱產品的說明文件以完成這些步驟。

    1.  按一下 \[開始\]，然後按一下 \[搜尋\]。
    2.  在 \[搜尋結果\] 窗格中，在 \[搜尋小幫手\] 下按一下 \[所有檔案和資料夾\]。
    3.  在 \[部份或完整的檔案名稱\] 方塊中，輸入適當檔案資訊表中的檔案名稱，再按一下 \[搜尋\]。
    4.  在檔案清單中，用滑鼠右鍵按一下適當檔案資訊表中的某個檔案名稱，再按 \[內容\]。
 
    5.  在 \[版本\] 索引標籤上，比較檔案版本與適當檔案資訊表中記錄的版本，判斷您系統上安裝的檔案版本。  
    
        **注意：**在安裝時，檔案版本以外的屬性可能會變更。 若要驗證是否成功套用更新程式，利用檔案資訊表比對其他檔案屬性並不是妥當的做法。 此外，在某些情況下，檔案的名稱在安裝時可能會有所變更。 如果缺少檔案或版本資訊，請採用其他可用的方法來驗證更新程式的安裝情形。

-   **登錄機碼驗證**  

    您也可以查看本節「參考表」中所列的登錄機碼，確認此安全性更新所安裝的檔案。

    這些登錄機碼可能未包含完整的安裝檔案清單。 而且，當系統管理員或 OEM 將安全性更新整合或匯集到 Windows 安裝原始檔時，可能無法正確建立這些登錄機碼。
