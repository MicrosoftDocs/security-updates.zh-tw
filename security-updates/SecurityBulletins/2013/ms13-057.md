Security Bulletin

Microsoft Security Bulletin MS13-057 - 重大
===========================================

Windows Media Format Runtime 中的資訊安全風險可能會允許遠端執行程式碼 (2847883)
-------------------------------------------------------------------------------

發行: 2013年7月9日 | 更新: 2013年8月30日

**版本:** 3.0

### 一般資訊

#### 提要

此資訊安全更新可解決 Microsoft Windows 中一項未公開報告的資訊安全風險。如果使用者開啟蓄意製作的媒體檔案，則此資訊安全風險可能會允許遠端執行程式碼。成功利用此資訊安全風險的攻擊者可以取得與本機使用者相同的使用者權限。系統上帳戶使用者權限較低的使用者，其受影響的程度比擁有系統管理權限的使用者要小。

對於所有受支援版本的 Windows (但 Itanium 型版本的 Windows 伺服器除外)，此資訊安全更新的等級為「重大」。如需更多資訊，請參閱本節中的＜受影響及不受影響的軟體＞小節。

此資訊安全更新可修改 Windows Media Player 開啟某些媒體檔案的方式，藉此解決此資訊安全風險。如需更多有關此資訊安全風險的資訊，請參閱下節**＜資訊安全風險資訊＞**下的特定資訊安全風險項目的＜常見問題集 (FAQ)＞小節。

**建議。** 大部分客戶都已啟用自動更新，並且不必須採取任何行動，因為資訊安全更新將自動下載和安裝。沒有啟用自動更新的客戶則必須檢查更新，並手動安裝更新。如需有關自動更新中特定設定選項的資訊，請參閱 [Microsoft 知識庫文件編號 294871](http://support.microsoft.com/kb/294871?ln=zh-tw)。

若是系統管理員和企業安裝，或是想要手動安裝此資訊安全更新的使用者，Microsoft 建議客戶使用更新管理軟體，立即套用更新，或使用 [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) 服務檢查更新。

另請參閱本公告下文的＜偵測與部署工具及指南＞一節。

更新常見問題集
--------------


**為何於 2013 年 8 月 27 日修訂此公告？**  
Microsoft 修訂此公告的目的在於針對指定版本的 Windows 重新發行下列更新：

-   適用於 Windows XP： 2803821、2834902、2834903、2834904 和 2834905 更新
-   適用於 Windows Server 2003： 2803821、2834902 和 2834904 更新
-   適用於 Windows Vista： 2803821 更新
-   適用於 Windows Server 2008： 2803821 更新

重新發行的更新可解決應用程式相容性問題 (原本 WMV 編碼視訊可能無法於播放期間正確呈現)。客戶應安裝重新發行的更新。

**為何於 2013 年 8 月 13 日修訂此公告？**  
Microsoft 修訂此公告的目的在於針對 Windows 7 與 Windows Server 2008 R2 重新發行 2803821 更新。重新發行的更新可解決應用程式相容性問題 (原本 WMV 編碼視訊可能無法於播放期間正確呈現)。Windows 7 和 Windows Server 2008 R2 客戶應安裝重新發行的更新。

**我已安裝原始更新。我是否仍需要套用重新發行的更新，以避免受到此公告中說明的資訊安全風險影響？**  
  已安裝原始 2803821、2834902、2834903、2834904 和 2834905 更新的客戶不會受到 CVE-2013-3127 問題的影響。不過，客戶需要安裝重新發行的更新，以避免部分客戶遇到的播放問題，這些問題已由重新發行的更新解決。

**此公告中指出的資訊安全風險如何影響 Server Core 安裝？**   
支援版本的 Windows Server 2008、Windows Server 2008 R2 或 Windows Server 2012 若是使用 Server Core 安裝選項所安裝，便不受本更新所解決的資訊安全風險影響，如「不受影響的軟體」表中所述。如需此安裝選項的詳細資訊，請參閱下列 TechNet 文章：[管理 Server Core 安裝： 概觀](http://technet.microsoft.com/library/ee441255)、[維護 Server Core 安裝](http://technet.microsoft.com/library/ff698994)，以及 [Server Core 與 Full Server 整合概觀](http://technet.microsoft.com/library/hh831758) (以上皆為英文)。

**「受影響的軟體」表中列有兩項適用於 Windows XP Media Center Edition 的資訊安全更新。是否兩項更新都必須安裝，才能避免受本*公告*所述之資訊安全風險的影響？**   
Windows XP Media Center Edition 有兩項適用的資訊安全更新 (2834904 和 2834905)。客戶該安裝哪一項更新取決於其 Media Player 版本是否升級至版本 11。在 Windows XP Media Center Edition 上執行 Windows Media Format Runtime 9.5 的客戶應套用 2834905 更新；然而，已升級至 Windows Media Format Runtime 11 或 Windows Media Player 11 的客戶則應套用 2834904 更新。客戶的系統若已升級，便只會收到 2834904。

**「受影響的軟體」表中針對 Windows XP Service Pack 3 上的 Windows Media Format Runtime 9.5 列出兩項資訊安全更新。是否兩項更新都必須安裝，才能避免受本公告所述之資訊安全風險的影響？**   
Windows XP Service Pack 3 上的 Windows Media Format Runtime 9.5 有兩項適用的資訊安全更新 (2834902 和 2834903)。客戶該安裝哪一項更新取決於安裝於其系統上的 Windows Media Format Runtime 二進位檔案版本。執行 Windows Media Format Runtime 9.5 NL 的客戶應安裝 2834902 更新，而執行 Windows Media Format Runtime 9.5 L 的客戶則應安裝 2834903 更新。我們會根據客戶系統上存在的二進位檔案，為客戶提供正確的更新。

**我執行的是 Windows Server 2008、Windows Server 2008 R2 或 Windows Server 2012，這些項目均被列為受影響的軟體，但為何我沒有收到更新？**  
Windows Server 2008、Windows Server 2008 R2 和 Windows Server 2012 上的 Windows Media Player 是隨著選用「桌面體驗」功能安裝。因此，如果這些系統上並未安裝或啟用「桌面體驗」功能，Windows Media Player 中易受影響的元件便不會存在，我們也不會提供更新。如需更多關於此選用功能的資訊，請參閱[桌面體驗概觀](http://technet.microsoft.com/zh-tw/library/cc772567(v=ws.10).aspx)。

**「受影響的軟體」表中針對 Windows XP、 Windows Server 2003、Windows Vista 和 Windows Server 2008 列出數項資訊安全更新。是否必須將適用於我的作業系統的更新全數安裝，才能避免受本公告所述之資訊安全風險的影響？**   
是的，為避免受本公告所述之資訊安全風險的影響，客戶應套用適用於其 Windows Media Format Runtime 或 Windows Media Player 版本的更新，以及針對其作業系統所列出任何不限版本的轉碼器更新。

**如有多項資訊安全更新，我是否必須按照特定順序來進行安裝？**   
否。特定的作業系統如有多項更新，套用時無需按照特定順序。

**如何判斷系統上安裝的 Windows Media Player 版本？**   
如需瞭解如何判斷系統上安裝的 Windows Media Player 版本，請參閱 [Microsoft 知識庫文件編號 190990](http://support.microsoft.com/kb/190990?ln=zh-tw)。

**如何判斷系統上安裝的 Windows Media Format Runtime 版本？**  
  若要判斷系統上所安裝的 Windows Media Format Runtime 版本，＜受影響的軟體＞中有針對您的作業系統列出相關的 .dll 檔案 (例如 Windows XP Service Pack 3 上的是 *wmvdmod.dll*)，請瀏覽至包含該檔案的資料夾。若是 32 位元系統，該檔案位於 %SystemRoot%\\System32 資料夾。若是 x64 型系統，該檔案位於 %SystemRoot%\\ syswow64 資料夾。

**我在 Windows Vista Service Pack 2 或 Windows Server 2008 Service Pack 2 上安裝 2803821 更新，然後發現 wmvdecod.dll 的版本從 11.0.6001.xxxx 降級至 6.0.6002.yyyy。這是正常的情況嗎？**  
  是的，這是在 Windows Vista Service Pack 2 或 Windows Server 2008 Service Pack 2 上套用 wmvdecod.dll 更新後，預期會出現的結果。

**我所使用的軟體是此資訊安全公告中討論的軟體之舊版。該怎麼辦？**   
本公告所列出的受影響軟體版本已經過測試判斷哪些版本會受到影響。其他版本超出它們的支援週期。如需瞭解產品生命週期的更多資訊，請參閱 [Microsoft 支援週期](http://support.microsoft.com/default.aspx?scid=fh;%5bln%5d;lifecycle?ln=zh-tw)網站。

使用此軟體舊版的客戶應優先考慮移轉至支援的版本，以避免因潛在的資訊安全風險而遭受攻擊。若要瞭解您的軟體版本的支援週期，請參閱[選擇一個產品檢視其支援週期資訊](http://support.microsoft.com/gp/lifeselect?ln=zh-tw)。如需更多關於上述軟體版本的 Service Pack 的資訊，請參閱 [Service Pack 週期支援政策](http://support.microsoft.com/?ln=zh-tw&scid=gp%3b%5bln%5d%3blifecycle&x=13&y=15)。

需要舊版軟體額外支援服務的客戶，請連絡 Microsoft 客戶小組人員、技術支援經理或適當的 Microsoft 協力廠商，以取得所需的額外支援。尚未簽訂聯盟、優先或授權合約的客戶，可以連絡當地的 Microsoft 銷售辦公室。如需連絡資訊，請參閱 [Microsoft 全球資訊](http://www.microsoft.com/worldwide/)網站，在 \[Contact Information\] (連絡資訊) 清單中選擇國家，然後按一下 **\[Go\]** (前往) 查看各地的連絡電話號碼。連絡時，請指明要連絡當地優先支援服務行銷經理。如需更多資訊，請參閱 [Microsoft 技術支援週期準則常見問答集](http://support.microsoft.com/gp/lifepolicy?ln=zh-tw)。

WMV 視訊解碼器遠端執行程式碼資訊安全風險 - CVE-2013-3127
--------------------------------------------------------


Windows Media Format Runtime 處理某些媒體檔案的方式中，存在遠端執行程式碼的資訊安全風險。攻擊者若引誘使用者開啟蓄意製作的媒體檔案，此資訊安全風險可能會允許攻擊者執行任意程式碼。攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。

若要以一般性資訊安全風險清單中的標準項目來檢視此資訊安全風險，請參閱 [CVE-2013-3127](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2013-3127)。

#### 緩和因素

緩和因素是指存在於預設狀態中的設定、共用設定或一般最佳作法，可能會減少資訊安全風險影響的嚴重性。下列緩和因素可能對您的狀況有所助益：

-   成功利用此資訊安全風險的攻擊者可以取得與本機使用者相同的使用者權限。系統上帳戶使用者權限較低的使用者，其受影響的程度比擁有系統管理權限的使用者要小。
-   此資訊安全風險無法透過電子郵件自動受到攻擊。使用者必須開啟以電子郵件附件方式傳送的蓄意製作的媒體檔案，攻擊才能成功。
-   在網頁式攻擊的案例中，攻擊者可架設一個網站，並在其中包含用來利用此資訊安全風險的網頁。此外，受侵害的網站以及接受或存放使用者提供之內容或廣告的網站裡，也可能包含蓄意製作以利用本資訊安全風險的內容。但是，在所有情況下，攻擊者無法強迫使用者造訪這類網站， 而是引誘使用者自行前往。一般的做法是設法讓使用者按一下電子郵件或 Instant Messenger 訊息中連往攻擊者網站的連結。
-   在 Windows Server 2008、Windows Server 2008 R2 和 Windows Server 2012 系統上，「桌面體驗」功能預設不會啟用，系統上必須具備這項功能，Windows Media Player 及其易受影響的元件才會出現。

#### 因應措施

因應措施指的是無法徹底修正資訊安全風險，但有助於在套用更新之前封鎖已知攻擊模式的設定變更。Microsoft 測試了下列因應措施和狀態，討論因應措施是否會降低功能：

-   **使用互動式管理指令碼來限制 WMV 轉碼器的存取**

    **在 Windows XP 和 Windows Server 2003 (32 位元系統) 上：**

    透過已提高權限的命令提示字元執行下列命令：

    `cacls %windir%\System32\Wmvdecod.dll /E /P everyone:N`

    **在 Windows XP 和 Windows Server 2003 (64 位元系統) 上：**

    從已提高權限的命令提示字元執行下列命令：

    `cacls %windir%\System32\Wmvdecod.dll /E /P everyone:Ncacls %windir%\sysWOW64\Wmvdecod.dll /E /P everyone:N`

    **在 Windows Vista 和 Windows Server 2008 (32 位元系統) 上：**

    從已提高權限的命令提示字元執行下列命令：

    `takeown /f %windir%\System32\Wmvdecod.dllicacls %windir%\System32\Wmvdecod.dll /save %TEMP%\FILENAME_ACL.TXTicacls %windir%\System32\Wmvdecod.dll /deny everyone:(F)`

    **在 Windows Vista 和 Windows Server 2008 (64 位元系統) 上：**

    從已提高權限的命令提示字元執行下列命令：

    `takeown /f %windir%\System32\Wmvdecod.dllicacls %windir%\System32\Wmvdecod.dll /save %TEMP%\FILENAME_ACL.TXTicacls %windir%\System32\Wmvdecod.dll /deny everyone:(F)takeown /f %windir%\sysWOW64\Wmvdecod.dllicacls %windir%\SysWOW64\Wmvdecod.dll /save %TEMP%\FILENAME_ACL.TXTicacls %windir%\SysWOW64\Wmvdecod.dll /deny everyone:(F)`

    **因應措施的影響。**執行這些步驟後，使用 WMV 轉碼器的應用程式便無法播放內容。

-   **使用互動式管理指令碼來停用 WMV 轉碼器**

    停用 WMV 轉碼器有助於保護受影響的系統不致遭受此資訊安全風險的攻擊。
    **注意：**Windows XP 不支援回復。

    **警告：**如果使用「登錄編輯程式」的方式錯誤，可能造成嚴重問題，以致於您必須重新安裝作業系統。Microsoft 無法保證您可以解決因為不正確使用 \[登錄編輯程式\] 所造成的問題。請自行承擔使用 \[登錄編輯程式\] 的風險。

#### 常見問題集

**此資訊安全風險的範圍為何？**   
這是遠端執行程式碼的資訊安全風險。成功利用此資訊安全風險的攻擊者可以取得受影響系統的完整控制權。攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。

**造成此資訊安全風險的原因為何？**   
此資訊安全風險是因 Windows Media Player 無法正確剖析蓄意製作的媒體檔案所造成。

**什麼是 Windows Media Player？**   
Windows Media Player 是供個人電腦使用的 Windows 作業系統的一項功能， 可是播放音訊與視訊。

**如何判斷系統上安裝的 Windows Media Player 版本？**   
如需瞭解如何判斷系統上安裝的 Windows Media Player 版本，請參閱 [Microsoft 知識庫文件編號 190990](http://support.microsoft.com/kb/190990?ln=zh-tw)。

**攻擊者可能會利用此資訊安全風險採取什麼行動？**   
一旦成功利用此資訊安全風險，攻擊者便能以登入的使用者身分執行任意程式碼。攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。如果使用者以系統管理的使用者權限登入，則攻擊者即可取得受影響系統的完整控制權。系統上帳戶使用者權限較低的使用者，其受影響的程度比擁有系統管理權限的使用者要小。

**攻擊者如何利用此資訊安全風險？**   
攻擊者可將蓄意製作的媒體檔案裝載於某個網路位置，再引誘使用者開啟該檔案，藉此利用此資訊安全風險。攻擊者不能強迫使用者開啟該檔案， 而是必須引誘使用者去執行這個動作，通常是設法讓使用者按下電子郵件或 Instant Messenger 訊息中的連結。

**因為此資訊安全風險而承受風險的主要系統有哪些？**   
使用 Windows Media Player 的系統 (通常為工作站) 最容易受此資訊安全風險影響。

**更新的作用何在？**   
此更新可修改 Windows Media Player 開啟某些媒體檔案的方式，藉此解決此資訊安全風險。

**本資訊安全公告發行時，此資訊安全風險是否已揭發出來？**   
否。Microsoft 是經由協同合作的來源接獲有關此資訊安全風險的訊息。

**當本資訊安全公告發行時，Microsoft 是否已接獲任何消息，指出此資訊安全風險已遭有心人士利用？**   
否。本資訊安全公告初次發行時，Microsoft 尚未接到任何有關本資訊安全風險已公開用來攻擊客戶的消息。
