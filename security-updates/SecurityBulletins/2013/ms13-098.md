Microsoft 資訊安全公告 MS13-098 - 重大
======================================

Windows 中的資訊安全風險可能會允許遠端執行程式碼 (2893294)
----------------------------------------------------------

發行日期： 2013 年 12 月 10 日 | 已更新： 2014 年 7 月 29 日

**版本：** 1.6

一般資訊
--------

### 提要

此資訊安全更新可解決 Microsoft Windows 中一項未公開報告的資訊安全風險。如果使用者或應用程式在受影響的系統上執行或安裝蓄意製作、已簽署的可攜式執行檔 (PE) 檔案，則此資訊安全風險可能會允許遠端執行程式碼。

對於所有受支援版本的 Windows，此資訊安全更新的等級為「重大」。如需更多資訊，請參閱＜受影響及不受影響的軟體＞一節。

此資訊安全更新會修改 WinVerifyTrust 函數針對可攜式執行檔處理 Windows Authenticode 簽章驗證的方式，進而解決資訊安全風險。如需更多有關此資訊安全風險的資訊，請參閱本公告稍後資訊安全風險的＜常見問題集 (FAQ)＞小節。

**建議。** 大部分客戶都已啟用自動更新，並且不必須採取任何行動，因為資訊安全更新將自動下載和安裝。沒有啟用自動更新的客戶則必須檢查更新，並手動安裝更新。如需有關自動更新中特定設定選項的資訊，請參閱 [Microsoft 知識庫文件編號 294871](https://support.microsoft.com/kb/294871)。

若是系統管理員和企業安裝，或是想要手動安裝此資訊安全更新的使用者，Microsoft 建議客戶使用更新管理軟體，立即套用更新，或使用 [Microsoft Update](https://go.microsoft.com/fwlink/?linkid=40747) 服務檢查更新。

另請參閱本公告下文的＜偵測與部署工具及指南＞一節。

更新常見問題集
--------------

<span id="sectionToggle1"></span>
**為何於 2014 年 7 月 29 日修訂此公告？ 
**之前，此公告指明 Microsoft 會發行更嚴格的 Authenticode 簽章驗證行為 (如 [Microsoft 資訊安全摘要報告 2915720](https://technet.microsoft.com/en-us/security/advisory/2915720) 中所述) 作為預設功能。但是，我們與客戶合作採用這項變更時，判定對現有軟體造成的影響可能會很高。因此，Microsoft 不再計畫強制將更嚴格的驗證行為設為預設功能。不過，更嚴格的驗證基礎功能仍提供使用，由客戶自行決定是否啟用。如需更多資訊，請參閱 [Microsoft 資訊安全摘要報告 2915720](https://technet.microsoft.com/en-us/security/advisory/2915720)。

**此更新是否包含任何與資訊安全有關的功能變更？** 
是。除了本公告＜資訊安全風險資訊＞一節中列出的變更外，此更新還包含 Windows Authenticode 簽章驗證的預設行為變更，而此變更僅由使用者選擇啟用。請注意，在安裝更新時預設未啟用此變更。若需如何啟用變更的詳細資訊與指示，請參閱 [Microsoft 資訊安全摘要報告 2915720](https://technet.microsoft.com/en-us/security/advisory/2915720)。

**Windows 8.1 Preview 和 Windows Server 2012 R2 Preview 是否會受此公告中指出的任何資訊安全風險影響？** 
是。2893294 更新可供 Windows 8.1 Preview 和 Windows Server 2012 R2 Preview 使用。建議執行這些作業系統的客戶為其系統套用更新。該更新是在 [Windows Update](https://go.microsoft.com/fwlink/?linkid=21130) 提供。

**我已套用此更新，但現在軟體的數位簽章無效。該怎麼辦？ **
此更新會導致 WinVerifyTrust 函數針對 PE 檔案執行嚴格的 Windows Authenticode 簽章驗證。套用更新之後，如果 Windows 將 PE 檔案中的內容識別為不符合 Authenticode 規格，該檔案將視為「未簽署」。這可能會影響一些安裝程式。如果您正在使用受影響的安裝程式，Microsoft 建議您在使用安裝程式時，僅從已簽署檔案之通過驗證的部分解壓縮內容。

**我所使用的是這個資訊安全公告中討論的舊版軟體。該怎麼辦？** 
本公告所列出的受影響軟體版本已經過測試判斷哪些版本會受到影響。其他版本超出它們的支援週期。如需瞭解產品生命週期的更多資訊，請參閱 [Microsoft 支援週期](https://go.microsoft.com/fwlink/?linkid=21742)網站。

使用此軟體舊版的客戶應優先考慮移轉至支援的版本，以避免因潛在的資訊安全風險而遭受攻擊。若要瞭解您的軟體版本的支援週期，請參閱[選擇一個產品檢視其支援週期資訊](https://go.microsoft.com/fwlink/?linkid=169555)。如需更多關於上述軟體版本的 Service Pack 的資訊，請參閱 [Service Pack 週期支援政策](https://go.microsoft.com/fwlink/?linkid=89213)。

需要舊版軟體額外支援服務的客戶，請連絡 Microsoft 客戶小組人員、技術支援經理或適當的 Microsoft 協力廠商，以取得所需的額外支援。尚未簽訂聯盟、優先或授權合約的客戶，可以連絡當地的 Microsoft 銷售辦公室。如需連絡資訊，請參閱 [Microsoft 全球資訊](https://go.microsoft.com/fwlink/?linkid=33329)網站，在 \[Contact Information\] (連絡資訊) 清單中選擇國家，然後按一下 \[**Go\]** (前往) 查看各地的連絡電話號碼。連絡時，請指明要連絡當地優先支援服務行銷經理。如需更多資訊，請參閱 [Microsoft 技術支援週期準則常見問答集](https://go.microsoft.com/fwlink/?linkid=169557)。

WinVerifyTrust 簽章驗證資訊安全風險 - CVE-2013-3900
---------------------------------------------------

<span id="sectionToggle3"></span>
WinVerifyTrust 函數針對可攜式執行檔 (PE) 檔案處理 Windows Authenticode 簽章驗證的方式中，存在遠端執行程式碼資訊安全風險。匿名攻擊者可透過這個方法來利用此資訊安全風險：修改現有的已簽署執行檔，將惡意程式碼加入檔案且不讓簽章失效，藉此利用檔案未經驗證的部分。成功利用此資訊安全風險的攻擊者可以取得受影響系統的完整控制權。攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。

若要以一般性資訊安全風險清單中的標準項目來檢視此資訊安全風險，請參閱 [CVE-2013-3900](https://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2013-3900)。

### 緩和因素

Microsoft 尚未找到此項資訊安全風險的任何緩和因素。

### 因應措施

Microsoft 尚未找到此項資訊安全風險的任何因應措施。

### 常見問題集

**這個資訊安全風險的範圍為何？** 
這是遠端執行程式碼的資訊安全風險。

**造成這項資訊安全風險的原因為何？** 
此資訊安全風險導因於在驗證 Windows Authenticode 簽章時，WinVerifyTrust 函數不當驗證蓄意製作之 PE 檔案的檔案摘要。

**什麼是 Windows Authenticode 簽章驗證？ 
**Windows Authenticode 簽章驗證分為兩種主要活動： 針對指定物件的簽章檢查，以及信任驗證。這些活動是由 WinVerifyTrust 函數執行，該函數會執行簽章檢查，然後再將查詢傳給支援動作識別碼的信任提供者 (若有的話)。如需 WinVerifyTrust 函數的詳細技術資訊，請參閱 [WinVerifyTrust 功能](https://msdn.microsoft.com/en-us/library/aa388208(vs.85).aspx) (英文)。

如需 Authenticode 簡介，請參閱 [程式碼簽署簡介](https://msdn.microsoft.com/en-us/library/ms537361(v=vs.85).aspx) (英文)。

**什麼是 Windows Authenticode 可攜式執行檔簽章格式？** 
Authenticode 是一種數位簽章格式，主要用於判斷軟體二進位檔案的來源與完整性。Authenticode 使用公開金鑰加密標準 (PKCS) \#7 簽署資料和 X.509 憑證，將經過 Authenticode 簽署的二進位檔案與軟體發行者的身分繫結在一起。「Authenticode 簽章」一詞指的是利用 Authenticode 簽章驗證功能產生並驗證的數位簽章格式。如需此格式的詳細資訊，請參閱[Windows Authenticode 可攜式執行檔簽章格式](https://www.microsoft.com/whdc/winlogo/drvsign/authenticode_pe.mspx) (英文)。

**攻擊者可能會利用此資訊安全風險採取什麼行動？** 
如果使用者以系統管理的使用者權限登入，成功利用此資訊安全風險的攻擊者可以取得受影響系統的完整控制權。攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。系統上帳戶使用者權限較低的使用者，其受影響的程度比擁有系統管理權限的使用者要小。

**攻擊者如何利用這項資訊安全風險？** 
若要利用此資訊安全風險，必須讓使用者或應用程式執行或安裝蓄意製作且已簽署的 PE 檔案。攻擊者可修改現有的已簽署檔案，加入惡意程式碼，而且不會使簽章失效。此程式碼會依啟動已簽署的 PE 檔案之際所使用的權限執行。

若是電子郵件攻擊，攻擊者可藉由向使用者傳送包含蓄意製作之 PE 檔案的電子郵件訊息，然後引誘使用者開啟該檔案，藉以利用此資訊安全風險。

在網頁式攻擊案例中，攻擊者必須架設含有蓄意製作之 PE 檔案的網站。此外，受侵害的網站以及接受或存放使用者提供之內容的網站裡，也可能包含蓄意製作以利用本資訊安全風險的內容。攻擊者無法強迫使用者造訪主控蓄意製作之 PE 檔案的網站。而是引誘使用者自行前往。一般的做法是設法讓使用者按一下電子郵件中或 Instant Messenger 訊息中連往攻擊者網站的連結。

**因為這個資訊安全風險而承受風險的主要系統有哪些？** 
工作站和終端伺服器的風險最高。

**這項更新有何作用？** 
此更新會修改 WinVerifyTrust 函數針對可攜式執行檔處理 Windows Authenticode 簽章驗證的方式，進而解決資訊安全風險。

**注意** 此更新會導致 WinVerifyTrust 函數針對 PE 檔案執行嚴格的 Windows Authenticode 簽章驗證。套用更新之後，如果 Windows 將 PE 檔案中的內容識別為不符合 Authenticode 規格，該檔案將視為「未簽署」。這可能會影響一些安裝程式。如果您正在使用受影響的安裝程式，Microsoft 建議您在使用安裝程式時，僅從已簽署檔案之通過驗證的部分解壓縮內容。

**本資訊安全公告發行時，這項資訊安全風險是否已公開揭露？** 
否。Microsoft 是經由協同合作的來源接獲有關此資訊安全風險的訊息。

**當本資訊安全公告發行時，Microsoft 是否已接獲任何消息，指出此資訊安全風險已遭有心人士利用？** 
是。Microsoft 已發現有嘗試利用此資訊安全風險、目標明確的攻擊。
