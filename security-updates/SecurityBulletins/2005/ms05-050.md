Microsoft Security Bulletin MS05-050 - 重大
===========================================

DirectShow 中的弱點可能會允許遠端執行程式碼 (904706)
----------------------------------------------------

發行: 2005年10月11日 | 更新: 2005年12月14日

**版本:** 1.3

#### 摘要

**應該閱讀此文件的對象：** 使用 Microsoft Windows 的客戶

**弱點的影響：** 遠端執行程式碼

**最高的嚴重性等級：** 重大

**建議：** 客戶應立即套用此更新程式。

**安全性更新取代資訊：** 本公告取代了一個先前發行的安全性更新。 請參閱本公告的＜常見問題集＞(FAQ) 以取得完整清單。

**警告：** [Microsoft 知識庫文件編號 904706](http://support.microsoft.com/kb/904706) 詳細記錄了目前已知安裝此安全性更新後可能會發生的問題。 該文件並說明了解決這些問題的建議解決方案。 如需更多資訊，請參閱 [Microsoft 知識庫文件編號 904706](http://support.microsoft.com/kb/904706)。

### 一般資訊

提要
----

**提要：**

這個更新程式能解決一項新發現且未公開報告的弱點。 本公告的＜弱點詳細資訊＞部分會提供這項弱點的相關資訊。

如果使用者以系統管理的使用者權限登入，成功利用此弱點的攻擊者可以取得受影響系統的完整控制權。 攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。 系統上帳戶使用者權限較低的使用者，其受影響的程度比擁有系統管理權限的使用者要小。

Microsoft 建議客戶立即套用此更新程式。

與本安全性更新相關的常見問題集 (FAQ)  
------------------------------------
  
**為何 Microsoft 要在 2005 年 12 月 13 日更新公告？**  
Microsoft 於今日更新公告，提醒客戶＜受影響的軟體＞部份中列出的 Windows 2000 Service Pack 4、Windows XP Service Pack 1 和 Windows 2003 之新版安全性更新程式已發行。 如需更多資訊，請參閱＜為何 Microsoft 要在 2005 年 11 月 9 日更新公告＞。
  
我們已修訂更新，以便在系統上安裝錯誤的 DirectX 套件版本時報告安裝失敗。 已在適當的 Windows 版本上套用合適的 DirectX 版本之客戶，則無須採取任何行動。 使用 WU、SUS、WSUS、MU、SMS 的客戶，不必進行任何動作，因為這些偵測方法會偵測並套用適當的版本。 已手動安裝不正確的 DirectX 套件之客戶，建議您評估自己的系統，然後重新部署正確的更新，以便確保更新正確的 DirectX 版本。
  
**為何 Microsoft 要在 2005 年 11 月 9 日更新公告？**  
Microsoft 已發現，在具有 DirectX 8 或 DirectX 9 的電腦上安裝適用於 Windows 2000 with Service Pack 4 的安全性更新套件，此安裝可以順利完成，並且不會顯示出電腦尚未更新的指示。 Microsoft 也已發現，在具有 DirectX 9 的電腦上安裝適用於 Windows XP Service Pack 1 或 Windows 2003 之 DirectX 8.1 的安全性更新套件時，安裝可以順利完成，並且不會顯示出電腦尚未更新的指示。 執行 DirectX 8 或 DirectX 9 的使用者還是會受到 MS05-050 中所述問題的影響，除非為其 DirectX 版本套用正確的套件。 如需更多資訊，請參閱常見問題集之＜安裝此安全性更新後，已知可能會發生的問題有哪些？＞
  
**安裝此安全性更新後，已知可能會發生的問題有哪些？**  
[Microsoft 知識庫文件編號 904706](http://support.microsoft.com/kb/904706) 詳細記錄了目前已知安裝此安全性更新後可能會發生的問題。 該文件並說明了解決這些問題的建議解決方案。 如需更多資訊，請參閱 [Microsoft 知識庫文件編號 904706](http://support.microsoft.com/kb/904706)。
  
-   [Microsoft 知識庫文件編號 909596](http://support.microsoft.com/kb/909596)： 當在執行 Windows Server 2003、Windows XP 或 Windows 2000 的電腦上安裝與安全性公告 MS05-050 有關聯的任一 DirectX 安全性更新 (909596) (MS05-050) 時，電腦可能不會更新  
-   [Microsoft 知識庫文件編號 910415](http://support.microsoft.com/kb/904706)： 當在執行 Windows 2000 多語系使用者介面 (MUI) 的 Windows 2000 電腦上安裝與安全性公告 MS05-050 (910415) 有關聯的任一 DirectX 安全性更新時，電腦可能不會更新。
  
**我的電腦執行 Windows 2000 Service Pack 4，並已安裝 DirectX 8.0、8.0a、8.1、8.1a、8.1b、8.2 或 9.0、9.0a、9.0b、9.0c 等版本其中之一，我也安裝了 DirectX 7.0 更新程式，我還會受到影響嗎？**  
是。 即使您已成功安裝 DirectX 7.0 安全性更新，但是它並不會更新 DirectX 7.0 之後的 DirectX 版本檔案。
  
**我的電腦執行 Windows XP Service Pack 1，並已安裝 DirectX 9.0、9.0a、9.0b、9.0c 等版本其中之一，我也安裝了 DirectX 8.1 更新程式，我還會受到影響嗎？**  
是。 即使您已成功安裝 DirectX 8.1 安全性更新，但是它並不會更新 DirectX 8.1 之後的 DirectX 版本檔案。
  
**我的電腦執行 Windows 2003，並已安裝 DirectX 9.0、9.0a、9.0b、9.0c 等版本其中之一，我也安裝了 DirectX 8.1 更新程式，我還會受到影響嗎？**  
是。 即使您已成功安裝 DirectX 8.1 安全性更新，但是它並不會更新 DirectX 8.1 之後的 DirectX 版本檔案。
  
**我的電腦執行 Windows 2000 Service Pack 4、Windows XP Service Pack 1 或 Windows 2003。我應該如何判斷我是否已安裝正確的安全性更新呢？**  
若要判斷是否已安裝正確的安全性更新，請執行 MBSA 2.0、企業更新掃描工具、SUS、WSUS 或 SMS。 您也可以造訪 Windows Update 網站，判斷您是否已安裝正確的安全性更新。 如需更多關於檔案驗證的相關資訊，請參閱＜DirectX Standalone＞部分的＜在 Windows 2000 上的所有 DirectX 8 和 DirectX 9 版本＞、＜在 Windows XP 上的所有 DirectX 9 版本＞或＜在 Windows 2003 上的所有 DirectX 9 版本＞等主題。

**如何判斷哪些更新適用於我的系統？**  
  
此更新一開始列出的**下載項目**會分成兩類：
  
1.  受影響的軟體  
2.  受影響的元件
  
在「受影響的軟體」下，於作業系統旁列出的下載項目適用於尚未升級為 DirectX 最新版本的系統。 如果系統已升級 DirectX，則使用者必須從「受影響的元件」部分安裝符合 DirectX 目前使用版本的下載項目。
  
**如何判斷目前我的系統上是否正在執行新版 DirectX？**  
DirectX 很常進行更新，並且會放在 Windows Update 上的建議的更新程式類別。 若要判斷您是否已升級至新版 DirectX，請依序按一下 \[開始\]、\[執行\]，鍵入
  
**DXDiag.exe**，然後按一下 \[確定\]。
  
DirectX 診斷工具就會啟動。 檢查 \[DirectX 版本:\] 旁邊的資訊 ，它會顯示出此系統上目前正在執行的 DirectX 版本。

#### DirectShow 弱點 - CAN-2005-2128

DirectShow 中存在的遠端執行程式碼弱點，可以讓成功利用此弱點的攻擊者取得受影響系統的完整控制權。

#### DirectShow 弱點 - CAN-2005-2128 的常見問題集：

**這個弱點的範圍為何？**  
這是遠端執行程式碼的弱點。 成功利用此弱點的攻擊者可以從遠端取得受影響系統的完整控制權。 攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。

**造成這個弱點的原因為何？**  
DirectShow 中的未檢查緩衝區。

**什麼是 DirectShow？**  
Microsoft DirectShow 是用來在 Microsoft Windows 作業系統上串流媒體。 DirectShow 能夠使用高畫質方式擷取和播放多媒體串流。 它會自動偵測並使用影音加速硬體 (如果有的話)，不過也支援不具備加速硬體的系統。 它還和其他 DirectX 技術整合。
部份可以使用 DirectShow 建立的應用程式類型如下：DVD 播放程式、視訊編輯應用程式、AVI 至 ASF 轉換程式、MP3 播放程式和數位視訊擷取應用程式。

**攻擊者可能會利用這項弱點採取什麼行動？**  
成功利用此弱點的攻擊者可以取得受影響系統的完整控制權。

**什麼人可以利用此弱點？**  
在 Windows 作業系統上，可傳送蓄意製作的 AVI 檔案至受影響系統的任何匿名使用者，均可利用這項弱點。

在電子郵件攻擊的情況中，攻擊者可以傳送蓄意製作的 AVI 給使用者，並引誘使用者開啟該檔案，藉此利用此弱點。
在網頁式攻擊的情況中，攻擊者可以架設一個網站，並在其中包含的網頁嘗試利用此弱點。 攻擊者並不能強制使用者造訪惡意的網站， 而是引誘他們自行前往。一般的做法是設法讓使用者按一下通往攻擊者網站的連結。 攻擊者也可以使用橫幅廣告或其他方式顯示惡意的網頁內容，將內容傳遞給受影響的系統。

**因為這個弱點而承受風險的主要系統有哪些？**  
工作站和終端機伺服器的風險最高。 如果不應擁有系統管理權限的使用者被授予登入伺服器並執行程式的能力時，伺服器會面臨更大的風險。 然而，最佳實務強烈建議您制止這種行為。

**更新的作用何在？**  
更新程式會修改傳遞訊息至分配的緩衝區之前，DirectShow 驗證訊息長度的方法，進而移除此項弱點。

**當安全性公告發行時，這項弱點是否已揭發出來？**  
否。 Microsoft 是經由可靠的來源接獲有關這項弱點的訊息。 當本安全性公告初次發行時，Microsoft 尚未接獲任何有關此弱點已被發佈出來的消息。

**當本安全性公告發行時，Microsoft 是否已接獲任何消息，指出這項弱點已遭有心人士惡用？**  
否。 當本安全性公告初次發行時，Microsoft 並未接到任何有關本弱點已成為公開攻擊媒介的消息，也沒有發現任何以此概念發展的程式碼公開範例。

**重新開機需求**

此更新程式不需要重新開機。 安裝程式會停止所需服務，然後套用更新，再重新啟動服務。 不過，如果必要的服務無法停止，或是必要的檔案正在使用中，更新程式就會要求重新開機。 在此情況下，系統會出現訊息提示您重新開機。 有關系統可能會提示您重新開機的原因的更多資訊，請參閱 [Microsoft 知識庫文件編號 887012](http://support.microsoft.com/kb/887012)。
