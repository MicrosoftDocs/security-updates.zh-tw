Security Bulletin

Microsoft Security Bulletin MS05-048 - 重大
===========================================

Microsoft Collaboration Data Objects 中的弱點可能會允許遠端執行程式碼 (907245)
------------------------------------------------------------------------------

發行: 2005年10月12日

**版本:** 1.0

#### 摘要

**應該閱讀此文件的對象：** 使用 Microsoft Windows 或 Microsoft Exchange Server 的客戶

**弱點的影響：** 遠端執行程式碼

**最高的嚴重性等級：** 重要

**建議：** 客戶應該儘快套用此更新程式。

**安全性更新取代資訊：** 無

**警告：** 無

弱點詳細資料  
------------
  
<span></span>
#### Collaboration Data Objects 弱點 - CAN-2005-1987
  
Collaboration Data Objects 弱點中存在的遠端執行程式碼弱點，可以讓成功利用此弱點的攻擊者取得受影響系統的完整控制權。
  
#### Collaboration Data Objects 弱點 - CAN-2005-1987 的緩和因素：
  
-   依照預設，Microsoft Internet Information Services (IIS) 5.0 簡易郵件傳送通訊協定 (SMTP) 及 Exchange 2000 SMTP 服務不會使用可能使用 Cdosys.dll 或 Codex.dll 的事件接收。  
-   依照預設，Windows 2003 不會啟用 IIS 6.0。  
-   依照預設，當 IIS 6.0 啟用時，不會啟用 SMTP。
  
#### Collaboration Data Objects 弱點 - CAN-2005-1987 的因應措施：
  
Microsoft 已經測試過以下的因應措施。 這些因應措施並不能徹底解決弱點，但是有助於封鎖已知的攻擊行為。 如果因應措施會降低功能，以下小節將會描述功能降低的情況。
  
-   **停用 Exchange 2000 Server 及執行 IIS 的伺服器上啟用的所有事件接收：**
  
    停用所有事件接收有助於保護受影響的系統，避免這項弱點遭到利用。 若要停用任何自訂或協力廠商的事件接收，請執行下列步驟：
  
    1.  使用 **cscript.exe smtpreg.vbs /enum** 命令列命令列舉所有已登錄的事件接收。 檢查此命令的輸出結果，尋找自訂或非 Microsoft 的軟體事件接收項目。  
    2.  確定要停用的事件接收之後，在命令提示字元下使用下列命令：  
        **cscript.exe smtpreg.vbs /disable**
  
    **因應措施的影響：** 直到事件接收重新啟用之前，已經在 SMTP 服務中建立事件接收的自動或協力廠商應用程式將無法正常運作。
  
-   **在 Exchange 2000 Server 解除 Cdoex.dll 檔案與 Cdosys.dll 檔案的登錄，在執行 IIS 的伺服器上解除 Cdosys.dll 檔案的登錄：**
  
    解除 Cdoex.dll 檔案及 Cdosys.dll 檔案的登錄有助於保護受影響的系統，避免這項弱點遭到利用。
  
    若要停用 Exchange 2000 Server 的 Cdoex.dll 檔案與 Cdosys.dll 檔案，請依照下列步驟進行。  
    **重要：**您必須依照下列順序操作這些步驟。
  
    1.  在命令列字元提示下執行下列命令：  
        **Regsvr32.exe “C:\\Program Files\\Common Files\\Microsoft Shared\\CDO\\cdoex.dll” /u**  
    2.  在命令列字元提示下執行下列命令：  
        **Regsvr32.exe %windir%\\system32\\cdosys.dll /u**
  
    若要在執行 IIS 的伺服器上停用 Cdosys.dll 檔案，請在命令提示字元中鍵入下列命令：
  
    1.  **Regsvr32.exe %windir%\\system32\\cdosys.dll /u**
  
    **因應措施的影響：** 解除 COM 物件的登錄會中斷依賴 Cdosys.dll 檔案或 Cdoex.dll 檔案的程式。
  
#### Collaboration Data Objects 弱點 - CAN-2005-1987 的常見問題集：
  
**這個弱點的範圍為何？**  
這是遠端執行程式碼的弱點。 成功利用此弱點的攻擊者可以從遠端取得受影響系統的完整控制權。 攻擊者接下來將能安裝程式，檢視、變更或刪除資料，或建立具有完整使用者權限的新帳戶。
  
**造成這個弱點的原因為何？**  
Collaboration Data Objects (CDO) 中存在未檢查的緩衝區。
  
**什麼是 Collaboration Data Objects？**  
Collaboration Data Objects (CDO) 是一個元件物件模型 (COM) 元件，在編寫建立或操控網際網路郵件訊息的程式時，此元件主要用在簡化程式的編寫工作。
  
**攻擊者可能會利用這項弱點採取什麼行動？**  
成功利用此弱點的攻擊者可以取得受影響系統的完整控制權。
  
**什麼人可以利用此弱點？**  
在 Microsoft Windows 或 Microsoft Exchange 2000 Server 上，任何匿名使用者只要能傳送蓄意製作的訊息至受影響的系統，都可能嘗試利用此弱點。
  
**攻擊者如何利用這項弱點？**  
攻擊者可能會建立蓄意製作的訊息，在受影響的系統上觸發事件接收，嘗試利用這項弱點。 攻擊者可能利用這種方式取得執行 SMTP 的系統之控制權。
  
**因為這個弱點而承受風險的主要系統有哪些？**  
如果呼叫 CDOSYS 或 CDOEX 中函式的應用程式用來處理訊息，則 Microsoft Windows 和 Microsoft Exchange 2000 Server 是受此弱點影響的主要系統。
  
**更新的作用何在？**  
更新程式會修改 CDO 將訊息傳遞至配置的緩衝區之前，驗證訊息長度的方法，進而移除此項弱點。
  
**當安全性公告發行時，這項弱點是否已揭發出來？**  
否。 Microsoft 是經由可靠的來源接獲有關這項弱點的訊息。 當本安全性公告初次發行時，Microsoft 尚未接獲任何有關此弱點已被發佈出來的消息。
  
**當本安全性公告發行時，Microsoft 是否已接獲任何消息，指出這項弱點已遭有心人士惡用？**  
否。 當本安全性公告初次發行時，Microsoft 並未接到任何有關本弱點已成為公開攻擊媒介的消息，也沒有發現任何以此概念發展的程式碼公開範例。

**重新開機需求：**

Microsoft Systems Management Server (SMS) 提供了深具彈性的企業解決方案，能夠對更新程式進行方便的管理。 利用 SMS，系統管理員能判斷有哪些 Windows 系統需要安全性更新，並控制更新程式在企業中的部署，同時將對使用者造成的干擾降到最低。 如需更多關於系統管理員如何使用 SMS 2003 部署安全性更新的資訊，請造訪 [SMS 2003 的安全性補充程式管理網站](http://www.microsoft.com/taiwan/smserver/evaluation/capabilities/patch.htm)。 SMS 2.0 使用者也可以利用 [SMS 軟體更新服務功能套件](http://go.microsoft.com/fwlink/?linkid=33340)來協助部署安全性更新。 如需關於 SMS 的詳細資訊，請造訪 [SMS 網站](http://www.microsoft.com/taiwan/smserver/)。

**注意：**SMS 使用 Microsoft Baseline Security Analyzer、Microsoft Office Detection Tool 以及企業更新掃描工具，為安全性公告的更新偵測及部署作業提供相當廣泛的支援。 不過這些工具可能無法偵測部分的軟體更新。 在這些情況中，系統管理員可以利用 SMS 的清查功能，判斷特定系統所需要的更新程式。 如需這個程序的詳細資訊，請造訪這個[網站](http://go.microsoft.com/fwlink/?linkid=33341)。 某些安全性更新程式在電腦重新啟動之後，會需要系統管理員的權限。 系統管理員可以用 Elevated Rights Deployment Tool (隨 [SMS 2003 Administration Feature Pack](http://www.microsoft.com/taiwan/smserver/downloads/2003/adminpack.htm) 和 [SMS 管理功能套件](http://www.microsoft.com/taiwan/smserver/downloads/20/featurepacks/adminpack/default.htm) 提供) 來安裝這些更新。
