Microsoft 資訊安全摘要報告 2905247
==================================

不安全的 ASP.NET 網站設定可能會允許權限提高
-------------------------------------------

發行日期： 2013 年 12 月 10 日 | 已更新： 2014 年 9 月 9 日

**版本：** 2.0

一般資訊
--------

### 提要

Microsoft 宣佈提供 Microsoft ASP.NET 更新，以解決透過組態設定停用機器驗證碼 (MAC) 驗證時，ASP.NET 檢視狀態中的一項資訊安全風險。該資訊安全風險可能會允許權限提高，並影響 Microsoft .NET Framework 1.1 Service Pack 1、Microsoft .NET Framework 2.0 Service Pack 2、Microsoft .NET Framework 3.5、Microsoft .NET Framework 3.5.1、Microsoft .NET Framework 4 以及 Microsoft .NET Framework 4.5/4.5.1。

任何 ASP.NET 網站的檢視狀態 MAC 在透過組態設定停用後，都會受到攻擊。成功利用此資訊安全風險的攻擊者，可利用蓄意製作的 HTTP 內容，將程式碼注入 ASP.NET 伺服器上的服務帳戶內容並加以執行。Microsoft 已發現這項遭到公開，可用來利用此資訊安全風險的一般資訊，但尚未發現任何主動攻擊行動。

**緩和因素**：

-   檢視狀態 MAC 在 ASP.NET 網站中預設為啟用。

**建議**。 Microsoft 建議客戶套用建議的動作，以確保 ASP.NET 檢視狀態 MAC 在 ASP.NET 網站上保持啟用。請參閱本摘要報告的＜建議動作＞一節以取得更多資訊。

**摘要報告的範圍為何？**   
此摘要報告的目的是要通知客戶 Microsoft 發布了更新，讓系統管理員設定其 ASP.NET 伺服器，以確保檢視狀態 MAC 隨時保持啟用狀態，並提供如何在 IIS 伺服器上啟用檢視狀態 MAC 的一般指南。
  
**什麼是檢視狀態？   
**檢視狀態是 ASP.NET 的一項功能，可讓網頁開發人員維護網頁狀態，並在 POST 要求或網頁更新及變更時，將資料持續保存於網頁表單。檢視狀態由 ASP.NET 開發人員廣為使用，因此在 ASP.NET 網站上隨處可見。檢視狀態一律經過剖析，即使 EnableViewState 屬性設為 False 亦然。如需更多資訊，請參閱[瞭解 ASP.NET 檢視狀態](http://msdn.microsoft.com/library/ms972976.aspx)。
  
**停用檢視狀態能否緩和此資訊安全風險？   
**否。檢視狀態一律由 ASP.NET 伺服器剖析，即使 EnableViewState 設為 False 亦然，不論屬性是否在 web.config、@Page 指令或 ASP.NET 標籤中設定。攻擊者可能會略過 EnableViewState 設定，在用戶端 Post 注入檢視狀態屬性。
  
**什麼是檢視狀態 MAC 驗證？   
**檢視狀態 MAC (機器驗證碼) 驗證是一項功能，會使 ASP.NET 在網頁產生時產生檢視狀態資料的雜湊。雜湊稍後會用於比較之後回傳的檢視狀態，允許伺服器驗證檢視狀態是否已遭竄改。這項技術可確保回傳資料未遭任何不當修改，並緩和此摘要報告中所述之資訊安全風險。
  
**攻擊者可能會利用此資訊安全風險採取什麼行動？**   
在多數案例中，成功利用此資訊安全風險的攻擊者，可能會在有資訊安全風險的 ASP.NET 網站上 (有不當設定的檢視狀態 MAC)，提高服務帳戶的權限層級。
  
**攻擊者如何利用這項資訊安全風險？**   
未經驗證的攻擊者可能會將蓄意製作的 HTTP 內容傳送到目標伺服器，讓攻擊者在 ASP.NET 網站上的服務帳戶內容中執行程式碼。
  
**這項更新有何作用？   
**此更新可使檢視狀態 MAC 隨時保持在啟用狀態，移除在伺服器上將其停用的功能，以解決此資訊安全風險。
  
**客戶在安裝更新後必須採取哪些額外動作？   
**此修正需要某些客戶採取其他動作，尤其是在 Web 伺服陣列中使用 ASP.NET 的客戶，以確保他們的 ASP.NET 網站有一致的可用性。其他設定步驟，請參閱下方的＜建議動作＞一節。
  
**我該如何分辨已安裝的 Microsoft .NET Framework 版本？**  
您可以在一個系統上安裝和執行多個版本的 .NET Framework，且能以任何順序安裝各版本。有數種方法能分辨目前已安裝的 .NET Framework 版本。如需更多資訊，請參閱 [Microsoft 知識庫文件編號 318785](http://support.microsoft.com/kb/318785)。****
  
**.NET Framework 4 與 .NET Framework 4 Client Profile 之間有何差異？**  
可在兩個設定檔中使用 .NET Framework 第 4 版可轉散發套件： .NET Framework 4 和 .NET Framework 4 Client Profile。.NET Framework 4 Client Profile 是 .NET Framework 4 設定檔的子集，其已針對用戶端應用程式進行最佳化。它會為多數用戶端應用程式提供功能，包括 Windows Presentation Foundation (WPF)、Windows Forms、Windows Communication Foundation (WCF) 和 ClickOnce 功能。這可為以 .NET Framework 4 Client Profile 為目標的應用程式進行更快速的部署及使用更小的安裝套件。如需詳細資訊，請參閱 MSDN 文章：[.NET Framework Client Profile](http://msdn.microsoft.com/en-us/library/cc656912.aspx) (英文)。****
  
**我已安裝 .NET Framework 3.0 Service Pack 2；此版本未在本公告內列入受影響的軟體。我必須安裝更新嗎？ **  
這個公告說明影響 .NET Framework 2.0 功能層的一項資訊安全風險。.NET Framework 3.0 Service Pack 2 安裝程式連結了 .NET Framework 2.0 Service Pack 2 安裝，因此安裝前者時，後者也會一併安裝。所以，已安裝 .NET Framework 3.0 Service Pack 2 的客戶也必須安裝 .NET Framework 2.0 Service Pack 2 的資訊安全更新。
  
**我已安裝 .NET Framework 3.5 Service Pack 1。我必須安裝任何更新嗎？**   
這個公告說明影響 .NET Framework 2.0 功能層的一項資訊安全風險。.NET Framework 3.5 Service Pack 1 安裝程式連結了 .NET Framework 2.0 Service Pack 2 安裝與 .NET Framework 3.0 Service Pack 2 安裝。因此，已安裝 .NET Framework 3.5 Service Pack 1 的客戶也必須安裝 .NET Framework 2.0 Service Pack 2 的資訊安全更新。
