ActiveX 刪除位元 3118753 的更新
===============================

發行日期：2016 年 1 月 12 日

**版本：** 1.0

提要
----

<span id="sectionToggle0"></span>
Microsoft 會以本摘要報告發行一套新的 ActiveX [刪除位元](https://technet.microsoft.com/zh-tw/library/security/dn848375.aspx)。這些 ActiveX 刪除位元隨附在 2016 年 1 月 12 日所發行的 Internet Explorer 累積更新中。

這項更新會針對下列協力廠商軟體設定 Kill Bit (刪除位元)：

-   **IBM Endpoint Manager for Remote Control (9.0.1 版和更新版本) 和 IBM Assist On-site (4.0.0 版)**。下列類別識別項與 IBM 為設定受影響 ActiveX 控制項刪除位元的要求有關。此 ActiveX 控制項的類別識別項 (CLSID) 為：

    {D4C0DB38-B682-42A8-AF62-DB9247543354}

**建議**。請參閱本摘要報告的**＜建議動作＞**一節，以取得適用於 Internet Explorer 特定版本更新的套用說明。

摘要報告常見問題集
------------------

<span id="sectionToggle3"></span>
**此項更新是否可取代 ActiveX 刪除位元的累積安全性更新 (2900986)？**  
否。此更新不會取代 Microsoft 資訊安全公告 [MS13-090](https://technet.microsoft.com/zh-tw/library/security/ms13-090) 中所描述的 ActiveX 刪除位元累積安全性更新 (2900986)。自動更新可能仍會提供 MS13-090 更新給客戶，不論客戶是否已安裝 Internet Explorer 累積更新。安裝了累積更新的客戶還需要安裝 MS13-090 更新，才能獲得 MS13-090 中所設定的所有刪除位元的保護。

**Microsoft 以往的刪除位元更新都是透過資訊安全公告發行，為什麼這次是透過資訊安全摘要報告宣佈這些新的 ActiveX 刪除位元？**  
＜提要＞中所描述的新的刪除位元皆為協力廠商的軟體，因此 Microsoft 才會透過摘要報告宣佈這些新的 ActiveX 刪除位元。

**Internet Explorer 累積更新會以什麼方式設定刪除位元?**  
此更新會變更登錄以防止在 Internet Explorer 中產生控制項。

**如果我沒有安裝 IBM Endpoint Manager for Remote Control (9.0.1 版和更新版本) 或 IBM Assist On-site (4.0.0 版)，是否應該安裝此更新？**  
是的。安裝此更新可防止 Internet Explorer 執行易受攻擊的控制項，保護系統不致暴露 [MS16-001](http://go.microsoft.com/fwlink/?linkid=717999) 中所描述的弱點。

**此更新是否包含之前在 Internet Explorer 安全性更新中發行的刪除位元？**  
是的。Internet Explorer 安全性更新是會累積的。此更新包含之前在 Internet Explorer 安全性更新中發行的刪除位元。

**本次摘要報告為什麼沒有與之相關的安全性等級？**  
此更新包含了適用於協力廠商控制項的新刪除位元。Microsoft 不提供易受影響的協力廠商控制項之安全性等級。
